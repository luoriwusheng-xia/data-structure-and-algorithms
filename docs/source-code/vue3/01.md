### pnpm-workspace.yaml

- 它告诉 pnpm 在 packages/ 目录下的所有包都是工作空间的一部分
- 同样 packages-private/ 目录下的所有包也包含在工作空间中
- 这种结构允许在一个代码仓库中管理多个相关的包（monorepo）

```yaml
packages:
  - 'packages/*'
  - 'packages-private/*'

catalog:
  '@babel/parser': ^7.25.3
  '@babel/types': ^7.25.2
  'estree-walker': ^2.0.2
  'magic-string': ^0.30.11
  'source-map-js': ^1.2.0
  'vite': ^5.4.0
  '@vitejs/plugin-vue': ^5.1.2
```

- catalog 这部分定义了项目中使用的核心依赖及其版本：
- 包含了 Babel 相关工具用于代码解析
- estree-walker 用于 AST 遍历
- magic-string 用于源码操作
- source-map-js 用于源码映射
- Vite 相关工具用于构建和开发

这个配置文件的主要好处是：
1. 实现了统一的依赖管理
2. 允许包之间的相互引用
3. 确保所有包使用相同版本的依赖

优化安装过程，避免重复安装相同的依赖
这是一个典型的大型项目使用的 monorepo 结构，可以更好地管理多个相关包的开发和发布。


### monorepo 解决跨包引入的问题

在 `packages` 的 A 包中需要引入 `shared `中提供的工具方法， 则需要调整 `typescript` 配置

tsconfig.json

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@vue/compat": ["packages/vue-compat/src"],
      "@vue/*": ["packages/*/src"],
      "vue": ["packages/vue/src"]
    },
  }
}
```

实现了  `@vue/compat` 和 `@vue/*` 的路径映射， `vue` 是一个单独的包，需要单独处理。

在 `packages/vue-compat` 源码中使用  `packages/shared/src` 中的工具方法

```javascript
import { isArray } from '@vue/shared'
```

模块安装

```shell
pnpm install @vue/shared --workspace --filter @vue/compat
```

将当前工作目录下的  `@vue/shared` 安装给 `@vue/compat`

@vue/compa 目录的 package.json 文件中添加依赖

```json
{
  "dependencies": {
    "@vue/shared": "workspace:*"
  }
}
```


### 小结

如果从构建脚本入手学习，大致步骤如下：

1. 使用 pnpm 初始化仓库， 初始化package.json, 并修改 `"type": "module"`
2. 设置pnpm 的 monorepo 配置, 配置 pnpm-workspace.yaml 文件
3. 创建对用的 目录结构， packages, packages/vue, packages/shared, packages/reativity 等目录， 调整对应package.json中的`name`
4. 源码中具备相同的 目录结构， src下面写对应的代码; 在构建打包的时候，比较有用
5. 初始化 typescript 的配置文件，防止 跨包引入的问题
6. 编写打包命令
7. 测试脚本
