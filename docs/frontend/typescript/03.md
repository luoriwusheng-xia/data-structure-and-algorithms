# TypeScript 面试题

## 基础类型相关问题

### 1. any、unknown、never 的区别

| 类型 | 说明 | 可赋值给任何类型 | 任何值可赋值给它 |
|------|------|----------------|----------------|
| `any` | 关闭类型检查 | ✅ | ✅ |
| `unknown` | 类型安全的 any | ❌ | ✅ |
| `never` | 永不存在的值 | ✅ | ❌ |

```typescript
let a: any = 1;
a = "hello";     // OK
a.toFixed();     // OK，编译不报错（运行时可能报错）

let u: unknown = 1;
u = "hello";     // OK
// u.toFixed();  // Error: 需要先进行类型收窄
if (typeof u === "number") {
  u.toFixed();   // OK
}

// never 通常用于穷尽检查
function exhaustiveCheck(x: never): never {
  throw new Error("Unhandled case: " + x);
}

type Shape = { kind: "circle"; radius: number } | { kind: "square"; side: number };

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "square":
      return shape.side ** 2;
    default:
      return exhaustiveCheck(shape); // 如果新增类型未处理，编译报错
  }
}
```

### 2. interface 和 type 的区别

```typescript
// 1. 声明合并 — interface 支持，type 不支持
interface Animal {
  name: string;
}
interface Animal {
  age: number;
}
// 合并为 { name: string; age: number }

// 2. 扩展方式不同
interface Dog extends Animal {
  breed: string;
}

type Cat = Animal & {
  color: string;
};

// 3. type 可以定义联合类型、元组等，interface 不行
type Status = "success" | "failure";
type Point = [number, number];

// 4. type 支持条件类型、映射类型
// interface 不支持直接使用 in 关键字
```

## 泛型相关问题

### 3. 泛型中的 extends 和约束

```typescript
// extends 在泛型中表示约束
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];
};

interface Person {
  name: string;
  age: number;
  address: string;
}

type PersonName = MyPick<Person, "name" | "age">;
// { name: string; age: number }

// extends 在条件类型中表示条件判断
type IsString<T> = T extends string ? true : false;
type A = IsString<"hello">;  // true
type B = IsString<123>;      // false
```

### 4. 逆变与协变

```typescript
// 协变（Covariance）：子类型可以赋值给父类型
let animals: Animal[] = [];
let dogs: Dog[] = [];
animals = dogs;  // OK，数组是协变的

// 逆变（Contravariance）：函数参数是逆变的
let fn1: (animal: Animal) => void = (a) => {};
let fn2: (dog: Dog) => void = (d) => {};
fn2 = fn1;  // OK，fn1 的参数更宽泛
// fn1 = fn2; // Error：fn2 不能接受任意 Animal

// TypeScript 开启 strictFunctionTypes 后，函数参数是逆变的
```

## 类型工具相关问题

### 5. 实现 DeepPartial

```typescript
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object
    ? T[P] extends Function
      ? T[P]
      : DeepPartial<T[P]>
    : T[P];
};

interface Company {
  name: string;
  employees: {
    name: string;
    department: {
      name: string;
      budget: number;
    };
  }[];
}

type PartialCompany = DeepPartial<Company>;
// 所有属性及嵌套属性都变为可选
```

### 6. 实现 RequiredKeys / OptionalKeys

```typescript
type RequiredKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? never : K;
}[keyof T];

type OptionalKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? K : never;
}[keyof T];

interface Example {
  a: string;
  b?: number;
  c?: boolean;
}

type Req = RequiredKeys<Example>;  // "a"
type Opt = OptionalKeys<Example>;  // "b" | "c"
```

## 类型守卫与收窄

### 7. typeof / instanceof / in / 自定义类型守卫

```typescript
class Cat {
  meow() {}
}
class Dog {
  bark() {}
}

// typeof — 用于基本类型
function isString(x: unknown): x is string {
  return typeof x === "string";
}

// instanceof — 用于类实例
function makeSound(animal: Cat | Dog) {
  if (animal instanceof Cat) {
    animal.meow();
  } else {
    animal.bark();
  }
}

// in — 检查属性是否存在
interface Bird {
  fly(): void;
}
interface Fish {
  swim(): void;
}

function move(animal: Bird | Fish) {
  if ("fly" in animal) {
    animal.fly();
  } else {
    animal.swim();
  }
}

// 自定义类型守卫 — 最灵活
function isFish(animal: Bird | Fish): animal is Fish {
  return (animal as Fish).swim !== undefined;
}
```

## 高级类型挑战

### 8. 实现 curry 的类型

```typescript
type Curry<T extends any[]> = T extends [infer First, ...infer Rest]
  ? Rest extends []
    ? (arg: First) => void
    : (arg: First) => Curry<Rest>
  : never;

function curry<T extends any[]>(fn: (...args: T) => any): Curry<T> {
  // 实现省略...
  return null as any;
}

const add = (a: number, b: string, c: boolean) => {};
const curriedAdd = curry(add);
// curriedAdd 类型为: (arg: number) => (arg: string) => (arg: boolean) => void
```

### 9. 实现 PromiseAll 的类型

```typescript
declare function PromiseAll<T extends readonly unknown[]>(
  values: readonly [...T]
): Promise<{
  [P in keyof T]: T[P] extends Promise<infer U> ? U : T[P];
}>;

const promise1 = Promise.resolve(3);
const promise2 = 42;
const promise3 = new Promise<string>((resolve) => setTimeout(resolve, 100, "foo"));

const p = PromiseAll([promise1, promise2, promise3]);
// p 类型为 Promise<[number, number, string]>
```

## 常见面试题总结

### 10. const 断言 (as const)

```typescript
const arr = [1, 2, 3] as const;
// arr 类型为 readonly [1, 2, 3]，不是 number[]

const obj = { x: 1, y: 2 } as const;
// obj 类型为 { readonly x: 1; readonly y: 2 }

// 使用场景：枚举值、配置对象
const config = {
  api: "https://api.example.com",
  timeout: 5000,
} as const;
```

### 11. 枚举（enum）vs 联合类型

```typescript
// enum 有运行时开销
enum Color {
  Red = "RED",
  Green = "GREEN",
  Blue = "BLUE",
}

// 更推荐使用 const enum 或联合类型
const enum ConstColor {
  Red = "RED",
  Green = "GREEN",
  Blue = "BLUE",
}

// 联合类型 — 零运行时开销
type ColorUnion = "RED" | "GREEN" | "BLUE";
```

### 12. 模块增强（Module Augmentation）

```typescript
// 给第三方库扩展类型
declare module "vue" {
  interface ComponentCustomProperties {
    $myProperty: string;
  }
}

// 或扩展全局接口
declare global {
  interface Window {
    myLib: any;
  }
}
```

## tsconfig.json 中 extends 与 references 的区别

两者都用于在多个 `tsconfig.json` 之间建立关系，但定位不同。

| 维度 | `extends` | `references` |
|------|-----------|--------------|
| 本质 | 配置继承 | 项目引用 |
| 是否同一项目 | 是 | 多个独立子项目 |
| 典型场景 | 统一团队基础编译配置 | 大型 monorepo 增量构建 |
| 是否需要 `composite` | 否 | 是（被引用的子项目） |
| 构建命令 | `tsc` | `tsc -b` |

### extends

子配置继承父配置，并可以覆盖其中部分选项。最终仍然是一个 TypeScript 项目。

```json
// base.json
{
  "compilerOptions": {
    "target": "ES2020",
    "strict": true
  }
}

// app/tsconfig.json
{
  "extends": "../base.json",
  "compilerOptions": {
    "outDir": "dist"
  },
  "include": ["src"]
}
```

### references

把代码库拆成多个子项目，每个子项目有自己的 `tsconfig.json`，被引用的项目需要开启 `composite: true`。配合 `tsc -b` 可实现增量编译。

```json
// packages/a/tsconfig.json
{
  "compilerOptions": {
    "composite": true,
    "outDir": "dist"
  },
  "include": ["src"]
}

// 根 tsconfig.json
{
  "references": [
    { "path": "./packages/a" }
  ]
}
```

### 一句话总结

- `extends` 解决“配置怎么复用”。
- `references` 解决“项目怎么拆分与增量构建”。
