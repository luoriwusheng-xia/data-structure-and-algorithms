# Chrome 扩展开发入门指南

> 本文面向有前端开发经验的工程师，系统梳理 Chrome 扩展（Chrome Extension / Manifest V3）的完整开发链路。

---

## 一、核心概念与架构

### 1.1 什么是 Chrome 扩展

Chrome 扩展是基于 Web 技术（HTML/CSS/JS）构建的浏览器功能增强程序，通过 Chrome 提供的特权 API 与浏览器深度交互，可以：

- 修改/拦截页面内容（Content Script）
- 与后台保持通信（Service Worker）
- 操作浏览器标签页、书签、历史记录
- 监听网络请求、修改请求响应
- 在工具栏弹出交互面板（Popup）
- 添加右键菜单、快捷键、桌面通知

### 1.2 Manifest V3 vs V2（关键差异）

Chrome 从 88 版本开始推行 Manifest V3，**V2 已于 2023 年 6 月停止支持**，新项目必须使用 V3。

| 维度 | Manifest V2 | Manifest V3 |
|------|------------|-------------|
| 后台脚本 | 持久后台页（Background Page） | 非持久 Service Worker |
| 网络请求 | `webRequest` API 可修改请求 | `declarativeNetRequest` 声明式规则 |
| 远程代码 | 允许加载远程 JS/CSS | **完全禁止**，所有代码必须打包 |
| CSP | 相对宽松 | 更严格，默认禁止 `eval`/`new Function` |
| 权限声明 | 运行时申请或预声明 | 更严格的权限模型 |
| 性能 | 常驻内存 | Service Worker 按需唤醒，更省内存 |

> ⚠️ **重要**：V3 的 Service Worker 是事件驱动、非持久的，每次事件处理后可能休眠，这意味着不能依赖全局变量存储状态。

### 1.4 各组件详解

#### Service Worker（后台脚本）

```javascript
// background.js —— 扩展的事件中枢

// 安装时触发（通常用于初始化）
chrome.runtime.onInstalled.addListener((details) => {
  // details.reason: 'install' | 'update' | 'chrome_update'
  if (details.reason === 'install') {
    // 初始化默认配置
    chrome.storage.sync.set({ enabled: true, theme: 'dark' });
  }
});

// 接收来自 Content Script 或 Popup 的消息
chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
  // request: 发送方传递的数据对象
  // sender: 包含 tab.id, frameId, url 等信息
  // sendResponse: 异步回调，注意：如果要异步响应，必须返回 true

  if (request.action === 'fetchData') {
    fetch(request.url)
      .then(res => res.json())
      .then(data => sendResponse({ success: true, data }))
      .catch(err => sendResponse({ success: false, error: err.message }));

    // ⚠️ 必须返回 true 才能异步调用 sendResponse
    return true;
  }
});

// 监听标签页更新，注入 Content Script（可选方案）
chrome.tabs.onUpdated.addListener((tabId, changeInfo, tab) => {
  if (changeInfo.status === 'complete' && tab.url?.includes('example.com')) {
    chrome.tabs.sendMessage(tabId, { action: 'pageLoaded' });
  }
});
```

> **生命周期陷阱**：Service Worker 空闲 30 秒后会休眠，所有全局变量丢失。状态持久化请使用 `chrome.storage`。

#### Content Script（内容脚本）

```javascript
// content.js —— 注入到目标页面的脚本

// 可读写页面 DOM，但运行在一个隔离的 "world" 中
// 无法直接访问页面 JS 变量，页面也无法访问 content script 的变量

// 向页面注入脚本（用于与页面 JS 通信）
const script = document.createElement('script');
script.src = chrome.runtime.getURL('injected.js');
script.onload = () => script.remove();
(document.head || document.documentElement).appendChild(script);

// 监听页面通过 DOM 事件发送的消息（双向通信桥梁）
window.addEventListener('message', (event) => {
  // 验证消息来源，防止被恶意页面利用
  if (event.source !== window) return;

  if (event.data.type === 'FROM_PAGE') {
    // 转发给 Service Worker
    chrome.runtime.sendMessage({
      action: 'proxyToBackground',
      data: event.data.payload
    });
  }
});

// 向 Service Worker 发送消息
chrome.runtime.sendMessage(
  { action: 'getUserConfig' },
  (response) => {
    if (chrome.runtime.lastError) {
      console.error('通信失败:', chrome.runtime.lastError.message);
      return;
    }
    applyConfig(response.config);
  }
);
```

#### Popup（弹出面板）

```html
<!-- popup.html -->
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <link rel="stylesheet" href="popup.css">
</head>
<body>
  <div id="app">
    <h1>我的扩展</h1>
    <label>
      <input type="checkbox" id="toggle">
      启用功能
    </label>
    <button id="actionBtn">执行操作</button>
  </div>
  <script src="popup.js"></script>
</body>
</html>
```

```javascript
// popup.js —— 注意：每次点击图标都会重新加载 popup.js

document.getElementById('toggle').addEventListener('change', async (e) => {
  // Popup 关闭后脚本停止运行，但 storage 数据会持久化
  await chrome.storage.sync.set({ enabled: e.target.checked });

  // 通知当前标签页的 Content Script
  const [tab] = await chrome.tabs.query({ active: true, currentWindow: true });
  if (tab?.id) {
    chrome.tabs.sendMessage(tab.id, {
      action: 'toggle',
      enabled: e.target.checked
    });
  }
});

// 初始化时读取配置
(async () => {
  const { enabled = false } = await chrome.storage.sync.get('enabled');
  document.getElementById('toggle').checked = enabled;
})();
```

### 关键字段解析

| 字段 | 说明 |
|------|------|
| `action` | V3 中替代 V2 的 `browser_action`，定义工具栏图标行为 |
| `background.service_worker` | 后台脚本，V3 强制使用 Service Worker |
| `content_scripts.matches` | URL 匹配模式，决定注入哪些页面 |
| `run_at` | `document_start` / `document_end` / `document_idle` |
| `permissions` | 运行时权限，安装时必须声明 |
| `optional_permissions` | 可选权限，运行时动态申请 |
| `host_permissions` | 跨域请求权限（V3 从 permissions 中分离出来） |
| `web_accessible_resources` | 允许网页访问的扩展内资源 |

## 三、权限系统详解

### 3.1 权限分类

```javascript
// ========== 必备基础权限 ==========
"storage"           // 本地存储（chrome.storage.local/sync）
"tabs"              // 操作标签页（读取 URL、标题、注入脚本）
"activeTab"         // 临时获取当前活动标签页权限（用户触发）
"scripting"         // 动态执行/注入脚本（V3 新增）

// ========== 页面内容权限 ==========
"host_permissions": ["https://*/*"]  // 访问特定域名

// ========== 浏览器功能权限 ==========
"bookmarks"         // 读写书签
"history"           // 读写历史记录
"downloads"         // 管理下载
"cookies"           // 读写 Cookie
"contextMenus"      // 右键菜单
"notifications"     // 桌面通知
"alarms"            // 定时任务
"idle"              // 检测用户空闲状态

// ========== 敏感权限（审核严格）==========
"<all_urls>"        // 访问所有网站（审核时会要求详细说明）
"webRequest"        // V3 中受限，主要用于观察而非修改
"declarativeNetRequestWithHostAccess"  // 修改网络请求
"background"        // 保持后台运行（特殊申请）
```

### 3.2 动态权限申请

```javascript
// 运行时申请可选权限（不会阻断安装流程）
async function requestOptionalPermissions() {
  try {
    const granted = await chrome.permissions.request({
      permissions: ['bookmarks', 'history'],
      origins: ['https://*.example.com/*']
    });

    if (granted) {
      console.log('权限已授予');
      // 执行需要权限的操作
    } else {
      console.log('用户拒绝授权');
      // 降级处理：提供有限功能
    }
  } catch (err) {
    console.error('权限申请失败:', err);
  }
}

// 检查当前是否拥有某项权限
const hasPermission = await chrome.permissions.contains({
  permissions: ['bookmarks']
});
```

### 3.3 activeTab 权限的智慧用法

`activeTab` 是一个特殊权限：仅在用户通过点击扩展图标、使用快捷键或右键菜单触发时，临时授予当前活动标签页权限。这是**最安全、最容易过审**的权限模式。

```javascript
// 配合 action 使用，无需声明 host_permissions
chrome.action.onClicked.addListener(async (tab) => {
  // 此时已获得该标签页的临时权限
  await chrome.scripting.executeScript({
    target: { tabId: tab.id },
    func: () => {
      document.body.style.filter = 'invert(1)';
    }
  });
});
```

## 四、组件间通信机制

### 4.1 通信方式总览

```
┌────────────────┬──────────────────┬─────────────────────────────┐
│ 通信方向        │ API              │ 适用场景                     │
├────────────────┼──────────────────┼─────────────────────────────┤
│ CS → SW        │ runtime.sendMessage│ 单次请求响应                │
│ SW → CS        │ tabs.sendMessage │ 向指定标签页发送消息         │
│ Popup → SW     │ runtime.sendMessage│ Popup 与后台通信            │
│ Popup → CS     │ tabs.sendMessage │ 通过 SW 或直接发送           │
│ CS ↔ 页面      │ window.postMessage│ 与页面 JS 交换数据           │
│ 全局广播        │ runtime.onMessage + broadcast │ 多组件同步状态    │
│ 跨扩展通信      │ runtime.sendMessage(extId)    │ 扩展间协作        │
└────────────────┴──────────────────┴─────────────────────────────┘
```

### 4.2 完整通信示例

```javascript
// ==================== 消息协议设计 ====================
// 建议统一消息格式，便于维护和调试

const MessageType = {
  GET_DATA: 'GET_DATA',
  SET_DATA: 'SET_DATA',
  TAB_UPDATED: 'TAB_UPDATED',
  BROADCAST: 'BROADCAST'
};

// ==================== Service Worker ====================
// background.js

// 维护一个简单的内存缓存（注意：Service Worker 休眠后丢失）
const cache = new Map();

chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
  const { type, payload, id } = request;

  switch (type) {
    case MessageType.GET_DATA:
      // 优先从 storage 读取持久化数据
      chrome.storage.local.get(payload.key)
        .then(result => sendResponse({ success: true, data: result[payload.key] }))
        .catch(err => sendResponse({ success: false, error: err.message }));
      return true; // 异步响应

    case MessageType.SET_DATA:
      chrome.storage.local.set({ [payload.key]: payload.value })
        .then(() => {
          // 广播给所有 Content Script
          broadcastToAllTabs({ type: MessageType.BROADCAST, key: payload.key, value: payload.value });
          sendResponse({ success: true });
        })
        .catch(err => sendResponse({ success: false, error: err.message }));
      return true;

    default:
      sendResponse({ success: false, error: '未知消息类型' });
  }
});

// 广播到所有标签页
async function broadcastToAllTabs(message) {
  const tabs = await chrome.tabs.query({});
  for (const tab of tabs) {
    try {
      await chrome.tabs.sendMessage(tab.id, message);
    } catch {
      // 忽略无 Content Script 的标签页
    }
  }
}

// ==================== Content Script ====================
// content.js

// 接收来自 Service Worker 的消息
chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
  if (request.type === MessageType.BROADCAST) {
    // 处理广播消息，例如同步配置
    applySetting(request.key, request.value);
  }
});

// 向 Service Worker 发送请求
async function getConfig(key) {
  return chrome.runtime.sendMessage({
    type: MessageType.GET_DATA,
    payload: { key }
  });
}

// ==================== Popup ====================
// popup.js

async function saveAndSync(key, value) {
  const response = await chrome.runtime.sendMessage({
    type: MessageType.SET_DATA,
    payload: { key, value }
  });

  if (response.success) {
    // 同时更新当前 Popup UI
    updateUI(key, value);
  }
}
```

### 4.3 长连接通信（Port）

对于需要持续通信的场景（如实时数据推送），使用 `runtime.connect`：

```javascript
// 建立端连接
const port = chrome.runtime.connect({ name: 'devtools-page' });

// 发送消息
port.postMessage({ type: 'INIT', tabId: chrome.devtools.inspectedWindow.tabId });

// 接收消息
port.onMessage.addListener((msg) => {
  console.log('收到长连接消息:', msg);
});

// 断开监听
port.onDisconnect.addListener(() => {
  console.log('连接已断开');
});
```

### 5.4 自定义 Vite 方案（适合已有项目集成）

```javascript
import { defineConfig } from 'vite';

import { crx } from '@crxjs/vite-plugin';
import manifest from './manifest.json';

export default defineConfig({

        popup: resolve(__dirname, 'src/popup/index.html'),
        options: resolve(__dirname, 'src/options/index.html'),
        background: resolve(__dirname, 'src/background.js'),
        content: resolve(__dirname, 'src/content.js'),
      },
      output: {
        entryFileNames: 'js/[name].js',
        chunkFileNames: 'js/[name].js',
        assetFileNames: (assetInfo) => {
          const info = assetInfo.name.split('.');
          const ext = info[info.length - 1];
          return `assets/[name][extname]`;
        },
      },
    },
  },
  // Content Script 不能共享 chunk，需单独打包
  build: {
```

### 6.3 Service Worker 调试要点

```javascript
// background.js —— 添加日志追踪生命周期

// 标记 Service Worker 启动（每次事件唤醒都会执行）
console.log('[SW] 启动时间:', new Date().toISOString());

// 使用 keep-alive 技巧延长生命周期（谨慎使用）
let keepAliveInterval;

function startKeepAlive() {
  keepAliveInterval = setInterval(() => {
    console.log('[SW] keep-alive ping');
  }, 20000); // 每 20 秒 ping 一次
}

chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
  startKeepAlive(); // 处理消息时保持存活

  // ... 业务逻辑

  // 5 分钟后停止 keep-alive
  setTimeout(() => {
    clearInterval(keepAliveInterval);
  }, 300000);

  return true;
});
```

```javascript
// scripts/zip.js
import { createWriteStream } from 'fs';
import { readdir, stat } from 'fs/promises';
import archiver from 'archiver';
import path from 'path';

async function zipDirectory(source, out) {
  const archive = archiver('zip', { zlib: { level: 9 } });
  const stream = createWriteStream(out);

  return new Promise((resolve, reject) => {
    archive
      .directory(source, false)
      .on('error', reject)
      .pipe(stream);

    stream.on('close', () => {
      const bytes = archive.pointer();
      console.log(`打包完成: ${out} (${(bytes / 1024).toFixed(2)} KB)`);
      resolve();
    });

    archive.finalize();
  });
}

// Chrome 商店要求 ZIP 包
```

### 9.3 版本自动递增

```javascript
// scripts/bump-version.js
import { readFile, writeFile } from 'fs/promises';

async function bumpVersion(type = 'patch') {
  const pkg = JSON.parse(await readFile('package.json', 'utf8'));
  const [major, minor, patch] = pkg.version.split('.').map(Number);

  switch (type) {
    case 'major': pkg.version = `${major + 1}.0.0`; break;
    case 'minor': pkg.version = `${major}.${minor + 1}.0`; break;
    case 'patch': pkg.version = `${major}.${minor}.${patch + 1}`; break;
  }

  await writeFile('package.json', JSON.stringify(pkg, null, 2));

  // 同步更新 manifest.json
  const manifest = JSON.parse(await readFile('src/manifest.json', 'utf8'));
  manifest.version = pkg.version;
  await writeFile('src/manifest.json', JSON.stringify(manifest, null, 2));

  console.log(`版本已更新至 ${pkg.version}`);
}

bumpVersion(process.argv[2]);
```

## 十、常见坑与解决方案

### 10.1 Service Worker 休眠导致状态丢失

```javascript
// ❌ 错误：依赖全局变量
let counter = 0;
chrome.runtime.onMessage.addListener((req, sender, res) => {
  counter++;
  res({ counter }); // counter 会在 SW 休眠后归零
});

// ✅ 正确：使用 chrome.storage
chrome.runtime.onMessage.addListener(async (req, sender, res) => {
  const { counter = 0 } = await chrome.storage.local.get('counter');
  const newCounter = counter + 1;
  await chrome.storage.local.set({ counter: newCounter });
  res({ counter: newCounter });
  return true;
});
```

### 10.2 Content Script 与页面 JS 的隔离

```javascript
// ❌ 错误：Content Script 无法直接访问页面变量
// 页面有 window.jQuery，但 content script 中访问不到
console.log(window.jQuery); // undefined

// ✅ 正确：通过注入脚本桥接
const script = document.createElement('script');
script.textContent = `
  // 这个函数运行在页面上下文中，可以访问页面变量
  const data = {
    userToken: window.localStorage.getItem('token'),
    jQueryVersion: jQuery?.fn?.jquery
  };
  window.postMessage({
    source: 'MY_EXTENSION_BRIDGE',
    payload: data
  }, '*');
`;
document.documentElement.appendChild(script);
script.remove();

// Content Script 监听页面消息
window.addEventListener('message', (event) => {
  if (event.data?.source === 'MY_EXTENSION_BRIDGE') {
    chrome.runtime.sendMessage({ type: 'PAGE_DATA', data: event.data.payload });
  }
});
```

### 10.3 CSP 限制导致内联脚本无法执行

```javascript
const div = document.createElement('div');
div.innerHTML = '<script>alert(1)</script>'; // 不会执行

// ✅ 正确：使用外部脚本文件
const script = document.createElement('script');
script.src = chrome.runtime.getURL('injected.js');
document.head.appendChild(script);

// 或在 manifest 中声明（仅限特定场景）
// manifest.json 中无法放宽 CSP，这是设计上的
```

### 10.4 跨域请求问题

```javascript
// ❌ 错误：Content Script 中直接 fetch 跨域
// Content Script 受页面 CSP 限制，不能随意跨域

// ✅ 正确：在 Service Worker 中发起请求
// background.js
chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
  if (request.type === 'API_CALL') {
    fetch(request.url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(request.body)
    })
    .then(res => res.json())
    .then(data => sendResponse({ success: true, data }))
    .catch(err => sendResponse({ success: false, error: err.message }));

    return true;
  }
});

// content.js 中调用
const result = await chrome.runtime.sendMessage({
  type: 'API_CALL',
  url: 'https://api.example.com/data',
  body: { query: 'search' }
});
```

### 10.5 异步消息通信忘记 return true

```javascript
// ❌ 错误：异步响应但忘记 return true
chrome.runtime.onMessage.addListener((req, sender, res) => {
  fetch('/api/data').then(data => res(data));
  // sendResponse 在监听器返回后才调用，但连接已关闭
});

// ✅ 正确：返回 true 保持消息通道开启
chrome.runtime.onMessage.addListener((req, sender, sendResponse) => {
  fetch('/api/data')
    .then(data => sendResponse({ success: true, data }))
    .catch(err => sendResponse({ success: false, error: err.message }));

  return true; // ← 关键！保持消息通道开启直到 sendResponse 被调用
});
```

### 10.7 Offscreen Document 的使用场景

V3 的 Service Worker 不支持 DOM API（如 `document`、`window`），如果需要用 DOMParser、播放音频等，需要创建 Offscreen Document：

```javascript
// background.js —— 创建离屏文档
let creatingOffscreen;

async function setupOffscreenDocument(path) {
  // 检查是否已存在
  if (await hasOffscreenDocument()) return;

  if (creatingOffscreen) await creatingOffscreen;
  else {
    creatingOffscreen = chrome.offscreen.createDocument({
      url: path,
      reasons: ['DOM_PARSER', 'AUDIO_PLAYBACK'],
      justification: '需要 DOM API 解析 HTML / 播放音频',
    });
    await creatingOffscreen;
    creatingOffscreen = null;
  }
}

async function hasOffscreenDocument() {
  const clients = await self.clients.matchAll();
  return clients.some(client => client.url.includes('offscreen.html'));
}

// 使用离屏文档解析 HTML
async function parseHtml(html) {
  await setupOffscreenDocument('offscreen.html');
  return chrome.runtime.sendMessage({
    target: 'offscreen',
    type: 'PARSE_HTML',
    html
  });
}
```

```javascript
// offscreen.js —— 离屏文档脚本
chrome.runtime.onMessage.addListener((msg) => {
  if (msg.target !== 'offscreen') return;

  if (msg.type === 'PARSE_HTML') {
    const parser = new DOMParser();
    const doc = parser.parseFromString(msg.html, 'text/html');

    // 提取数据后返回
    const title = doc.querySelector('h1')?.textContent;
    chrome.runtime.sendMessage({
      target: 'background',
      result: { title }
    });
  }
});
```
