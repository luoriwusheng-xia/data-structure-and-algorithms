## 一、为什么选择 SVG.js

在 [SVG 企业级实战](./01) 中，我们系统梳理了 SVG 的核心概念与工程化方案。如果说原生 SVG 是“手动挡”，那么 **SVG.js** 就是一辆“自动挡跑车”：

- **链式 API**：创建、移动、填充、动画一气呵成。
- **现代化架构**：v3 完全基于 ES Module，支持 TypeScript 类型提示。
- **事件与交互**：每个图形元素天然支持 `mouseenter`、`click` 等事件绑定。
- **动画系统**：内置 `animate()`，支持缓动、延迟、顺序编排。
- **零依赖原型**：通过 CDN 即可在 HTML/VitePress/Vue 中快速验证想法。

适用场景：仪表盘、拓扑图、流程编辑器、图标动效、轻量级数据可视化。

## 二、CDN 引入与项目初始化

SVG.js 3.x 推荐通过 ES Module CDN 引入，无需 npm install 即可运行。

### 2.1 浏览器原生使用

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>SVG.js Demo</title>
</head>
<body>
  <div id="drawing"></div>

  <script type="module">
    import { SVG } from 'https://cdn.jsdelivr.net/npm/@svgdotjs/svg.js@3.2.4/+esm'

    const draw = SVG().addTo('#drawing').size(400, 300)
    draw.rect(100, 80).move(50, 50).fill('#3b82f6').radius(8)
    draw.circle(60).move(220, 60).fill('#ef4444')
  </script>
</body>
</html>
```

### 2.2 Vue 3 / VitePress 中使用

在 `.vue` 文件里同样可以直接从 CDN import：

```vue
<script setup>
import { onMounted } from 'vue'

onMounted(async () => {
  const { SVG } = await import('https://cdn.jsdelivr.net/npm/@svgdotjs/svg.js@3.2.4/+esm')
  const draw = SVG().addTo('#chart').size(400, 300)
  // ...
})
</script>
```

注意：`import(/* @vite-ignore */ url)` 中的 `@vite-ignore` 注释可防止 Vite 在构建阶段误将完整 URL 当作裸模块处理。

## 三、核心 API 速览

### 3.1 创建画布与基础图形

```javascript
import { SVG } from 'https://cdn.jsdelivr.net/npm/@svgdotjs/svg.js@3.2.4/+esm'

const draw = SVG().addTo('#canvas').size(600, 400)

// 基础形状
const rect = draw.rect(100, 60).move(20, 20).fill('#3b82f6').radius(8)
const circle = draw.circle(50).move(150, 25).fill('#ef4444')
const line = draw.line(50, 120, 250, 120).stroke({ width: 3, color: '#f59e0b' })

// 路径
const path = draw.path('M 50 200 Q 150 100 250 200 T 450 200')
  .fill('none')
  .stroke({ width: 3, color: '#10b981' })
```

### 3.2 渐变与滤镜

```javascript
const gradient = draw.gradient('linear', (add) => {
  add.stop(0, '#3b82f6')
  add.stop(1, '#8b5cf6')
})

rect.fill(gradient)
```

### 3.3 动画

```javascript
rect.animate(1000, '<')
  .move(300, 20)
  .fill('#10b981')
  .after(() => {
    console.log('动画结束')
  })
```

### 3.4 事件绑定

```javascript
circle.on('click', function () {
  this.animate().radius(this.attr('r') > 40 ? 25 : 50)
})
```

### 3.5 Group 与坐标系

```javascript
const group = draw.group().move(100, 100)
group.rect(50, 50).fill('#3b82f6')
group.circle(30).fill('#ef4444')
// group 内元素的坐标相对于 group 的原点
```

### 5.2 关键实现拆解

#### 1）CDN 动态加载

```javascript
const SVG_CDN = 'https://cdn.jsdelivr.net/npm/@svgdotjs/svg.js@3.2.4/+esm'

async function loadSvgJs() {
  const mod = await import(/* @vite-ignore */ SVG_CDN)
  return mod.SVG
}
```

这种方式的好处是：

- 不污染 `package.json`。
- 演示环境可以秒级启动。
- 生产项目如需使用，再迁移到 `npm install @svgdotjs/svg.js` 即可。

#### 2）环形图路径生成

SVG.js 没有内置“带内孔的扇形”，我们通过 `path` 命令手动描述：

```javascript
function describeArc(cx, cy, outerR, innerR, start, end) {
  const startOuter = polar(cx, cy, outerR, end)
  const endOuter = polar(cx, cy, outerR, start)
  const startInner = polar(cx, cy, innerR, end)
  const endInner = polar(cx, cy, innerR, start)
  const largeArc = end - start > Math.PI ? 1 : 0

  return [
    'M', startOuter.x, startOuter.y,
    'A', outerR, outerR, 0, largeArc, 0, endOuter.x, endOuter.y,
    'L', endInner.x, endInner.y,
    'A', innerR, innerR, 0, largeArc, 1, startInner.x, startInner.y,
    'Z',
  ].join(' ')
}
```

这是 SVG.js 企业实战中最常见的“补位”思路：库没有提供的高级图形，用 `path` + 数学公式补齐。

#### 3）Tooltip 层级管理

SVG 的渲染顺序由 DOM 顺序决定，后添加的元素会覆盖先添加的元素。因此 Tooltip 必须在最后创建：

```javascript
tooltip = createTooltip(draw) // 放在 initBarChart / initRingChart 之后
```

同时通过 `show()` / `hide()` 控制显隐，而不是反复创建销毁，避免 GC 抖动。

#### 4）事件委托 vs 直接绑定

本案例每个柱子和节点都单独绑定了事件。如果元素数量超过 1000，建议改用 **事件委托**：

```javascript
const chart = draw.group()
chart.on('mouseover', (e) => {
  const target = e.target
  if (target.tagName === 'rect') {
    // 命中检测后的统一处理
  }
})
```

## 六、性能优化与最佳实践

### 6.1 减少 DOM 节点数量

SVG.js 每个元素都是真实 DOM，节点过多会拖慢渲染。常用优化：

- 复用 `group` 和 `use`。
- 大数据量（>5k）考虑切到 Canvas / WebGL。
- 只渲染视口内元素。

### 6.2 动画避免修改布局属性

SVG 中 `x`、`y`、`width`、`height` 的动画会触发重排；优先使用 `transform`：

```javascript
// 推荐：transform 只触发合成
el.transform({ translateX: 100, translateY: 50 })

// 不推荐：频繁修改布局属性
el.animate().attr({ x: 100, y: 50 })
```

### 6.3 生命周期管理

在 Vue / React 组件中卸载时，必须清理：

```javascript
onUnmounted(() => {
  clearInterval(animationTimer)
  draw?.clear()
  draw = null
})
```

### 6.4 响应式容器

SVG.js 支持 `size('100%', 'auto')` 配合 `viewbox()` 实现响应式：

```javascript
const draw = SVG()
  .addTo('#chart')
  .size('100%', 420)
  .viewbox(0, 0, 800, 420)
```

内部按 800x420 坐标系编写，页面自动缩放。

### 6.5 生产环境建议

| 场景 | 建议 |
|------|------|
| 原型验证 | CDN + 动态 import |
| 正式项目 | `npm install @svgdotjs/svg.js` |
| 图标系统 | 配合 Symbol + SVGO，不直接用 SVG.js |
| 大数据可视化 | SVG.js + Canvas/WebGL 混合，或直接用 D3/AntV |
| 复杂交互 | 使用 TypeScript + SVG.js 类型声明 |

## 七、总结

SVG.js 3.x 是一个现代化、轻量、表达能力强的 SVG 操作库。通过本文案例可以看到：

- **快速搭建**：几行代码即可创建专业级图表。
- **交互友好**：事件绑定与动画 API 让数据可视化“活”起来。
- **工程可扩展**：原型阶段用 CDN，生产阶段切到 npm，迁移成本极低。
- **互补而非替代**：SVG.js 适合中小规模、强交互的矢量图形；超大规模数据仍需结合 Canvas / WebGL。

> 当你需要在 SVG 上快速做“可交互动画、数据绑定、拓扑关系”时，SVG.js 是值得优先考虑的瑞士军刀。
