## 一、SVG 是什么

SVG（Scalable Vector Graphics）是一种基于 XML 的矢量图形格式，由 W3C 制定标准。与位图不同，SVG 图形由数学描述（点、线、曲线、形状）构成，因此：

- **无限缩放**：任意分辨率下都清晰，不会失真。
- **文件体积小**：简单图形通常比同等质量的 PNG/JPG 更小。
- **可编程**：可以通过 CSS、JavaScript 动态控制样式、属性、动画。
- **可搜索/可访问**：SVG 中的文本可以被搜索引擎索引，也支持 ARIA 属性。

在前端工程中，SVG 最常见的形态是：图标、插画、数据可视化、地图、复杂动画。

## 二、SVG vs Canvas vs DOM

| 维度 | SVG | Canvas | DOM（HTML/CSS） |
|------|-----|--------|----------------|
| 图形类型 | 矢量 | 位图 | 盒子模型 |
| 渲染方式 | 保留对象模型，可单独操作元素 | 每帧重绘像素 | 由浏览器布局渲染 |
| 交互性 | 天然支持事件绑定 | 需手动计算命中检测 | 原生事件体系 |
| 元素数量 | 成千上万个元素会拖慢 DOM | 适合大量粒子/像素 | 大量节点性能差 |
| 动画 | CSS/JS/SMIL 均可 | 需要手动逐帧绘制 | CSS 动画、Transition |
| 缩放 | 无损 | 会模糊 | 依赖 CSS |
| 适用场景 | 图标、图表、地图、插画 | 游戏、视频处理、粒子特效 | 页面布局、表单、组件 |

一句话总结：

- **SVG**：需要“可交互、可缩放、保留结构”的矢量图形。
- **Canvas**：需要“高性能像素级控制”的位图渲染。
- **DOM**：需要“标准页面布局和组件化”的 UI。

## 三、SVG 核心用法

### 3.1 基础形状

```html
<svg width="200" height="200" viewBox="0 0 200 200" xmlns="http://www.w3.org/2000/svg">
  <rect x="10" y="10" width="80" height="60" rx="8" fill="#3b82f6" />
  <circle cx="150" cy="50" r="30" fill="#ef4444" />
  <ellipse cx="50" cy="150" rx="40" ry="20" fill="#10b981" />
  <line x1="120" y1="120" x2="180" y2="180" stroke="#f59e0b" stroke-width="4" />
  <polygon points="100,10 120,80 80,80" fill="#8b5cf6" />
</svg>
```

### 3.2 viewBox 与坐标系

`viewBox="x y width height"` 是 SVG 最重要的属性之一，它定义了用户坐标系：

```html
<svg width="400" height="200" viewBox="0 0 200 100">
  <circle cx="100" cy="50" r="40" />
</svg>
```

上述 SVG 显示区域是 400x200，但内部坐标系是 200x100，因此圆会被等比例放大 2 倍。

掌握 `viewBox`、`width`、`height`、`preserveAspectRatio` 是做好响应式 SVG 的基础。

### 3.3 Path 路径

`path` 是 SVG 最强大的元素，通过 `d` 属性描述任意形状：

```html
<path d="M 10 10 H 90 V 90 H 10 Z" fill="none" stroke="black" />
```

常用命令：

| 命令 | 含义 |
|------|------|
| M/m | 移动到 |
| L/l | 直线到 |
| H/h | 水平线 |
| V/v | 垂直线 |
| C/c | 三次贝塞尔曲线 |
| Q/q | 二次贝塞尔曲线 |
| A/a | 椭圆弧 |
| Z/z | 闭合路径 |

### 3.4 渐变与滤镜

```html
<svg width="200" height="100">
  <defs>
    <linearGradient id="grad" x1="0%" y1="0%" x2="100%" y2="0%">
      <stop offset="0%" stop-color="#3b82f6" />
      <stop offset="100%" stop-color="#8b5cf6" />
    </linearGradient>
  </defs>
  <rect width="200" height="100" fill="url(#grad)" rx="12" />
</svg>
```

常用滤镜：

- `feGaussianBlur`：高斯模糊
- `feDropShadow`：投影
- `feColorMatrix`：颜色矩阵
- `feBlend` / `feComposite`：混合与合成

### 3.5 动画方式

SVG 动画有三种主要方式：

**CSS 动画（推荐）**：

```css
.spinner {
  transform-origin: center;
  animation: rotate 2s linear infinite;
}

@keyframes rotate {
  to {
    transform: rotate(360deg);
  }
}
```

**SMIL 动画（原生，但兼容性下降）**：

```html
<circle cx="50" cy="50" r="40">
  <animate attributeName="r" values="40;20;40" dur="2s" repeatCount="indefinite" />
</circle>
```

**JavaScript 动画（可控性最强）**：

```javascript
const circle = document.querySelector('circle')
let r = 40
function animate() {
  r = 40 + Math.sin(Date.now() / 500) * 10
  circle.setAttribute('r', r)
  requestAnimationFrame(animate)
}
animate()
```

## 四、企业实战场景

### 4.1 场景一：企业级图标系统

大型企业项目通常有数百个图标，最佳实践是：

- **统一设计规范**：尺寸基础网格（16x16 / 24x24 / 32x32）、描边宽度、端点样式。
- **SVG Symbol 雪碧图**：通过 `symbol` + `use` 减少 HTTP 请求和 DOM 重复。

```html
<svg style="display: none;">
  <defs>
    <symbol id="icon-home" viewBox="0 0 24 24">
      <path d="M3 12l2-2v7a2 2 0 002 2h10a2 2 0 002-2v-7l2 2" />
      <path d="M12 3l-9 9h3v8h6v-6h6v6h3l-9-9z" />
    </symbol>
  </defs>
</svg>

<svg width="24" height="24">
  <use href="#icon-home" />
</svg>
```

- **图标库工程化**：使用 `svgo` 压缩，`vite-plugin-svg-icons` / `unplugin-icons` 自动注入 Symbol。
- **按需加载**：按页面/模块拆分包，避免一次性加载全部图标。

### 4.2 场景二：数据可视化图表

SVG 是 ECharts、D3.js、AntV 等可视化库的重要底层渲染方式。

SVG 图表的优势：

- 每个数据点对应一个 DOM 元素，便于绑定事件、tooltip、选中态。
- 支持 CSS 控制样式，主题切换方便。
- 导出矢量图（PDF/SVG）不会失真。

```html
<svg width="600" height="300" viewBox="0 0 600 300">
  <g class="bar-chart">
    <rect x="20" y="180" width="40" height="120" fill="#3b82f6" />
    <rect x="80" y="120" width="40" height="180" fill="#3b82f6" />
    <rect x="140" y="60" width="40" height="240" fill="#3b82f6" />
  </g>
</svg>
```

对于数据量极大的场景（>10k 节点），可以切换为 Canvas 或 WebGL 渲染。

### 4.3 场景三：拓扑图与流程图

BPMN 流程图、ER 图、组织架构图、网络拓扑图等企业应用非常适合 SVG：

- 节点和连边都是独立元素，支持拖拽、连线、对齐。
- 可以无限缩放，适合大屏展示。
- 支持导出为 SVG/PDF，便于打印和二次编辑。

典型库：AntV X6、React Flow、GoJS、D3-force。

### 4.4 场景四：地图与地理信息

SVG 地图可以精确控制每个区域的路径、颜色、交互：

```html
<svg viewBox="0 0 1000 800">
  <path
    class="region"
    d="M200,300 L400,250 ..."
    data-name="上海"
    fill="#e5e7eb"
    stroke="#9ca3af"
  />
</svg>
```

```javascript
document.querySelectorAll('.region').forEach((region) => {
  region.addEventListener('mouseenter', () => {
    region.setAttribute('fill', '#3b82f6')
  })
  region.addEventListener('mouseleave', () => {
    region.setAttribute('fill', '#e5e7eb')
  })
})
```

### 4.5 场景五：响应式插画与品牌图形

Landing Page、空状态插画、加载动画、品牌 Logo 等适合用 SVG 实现：

- 一份资源适配移动端和桌面端。
- 可以通过 CSS 变量动态换肤。
- 支持 Lottie 导出，但纯 SVG + CSS/JS 通常更小、更可控。

## 五、主流 SVG 库与框架实战

### 5.1 D3.js（数据驱动文档）

D3 是企业可视化领域的王者，提供数据绑定、比例尺、选择器、过渡动画、地理投影等全套工具。

```javascript
import * as d3 from 'd3'

const data = [
  { name: 'A', value: 30 },
  { name: 'B', value: 80 },
  { name: 'C', value: 55 },
]

const svg = d3.select('svg')

svg
  .selectAll('rect')
  .data(data)
  .join('rect')
  .attr('x', (d, i) => i * 60)
  .attr('y', (d) => 100 - d.value)
  .attr('width', 50)
  .attr('height', (d) => d.value)
  .attr('fill', '#3b82f6')
  .transition()
  .duration(800)
  .attr('y', (d) => 100 - d.value)
```

**企业用法**：

- 自定义折线图、柱状图、桑基图、力导向图。
- 与 React/Vue 结合时，把 D3 作为计算引擎（scale、layout），渲染交给框架。
- 大量节点时，用 D3 + Canvas / WebGL 混合渲染。

### 5.2 React 中使用 SVG

React 中可以直接写 JSX 形式的 SVG：

```jsx
function ProgressCircle({ percent }) {
  const radius = 50
  const circumference = 2 * Math.PI * radius
  const offset = circumference - (percent / 100) * circumference

  return (
    <svg width="120" height="120" viewBox="0 0 120 120">
      <circle
        cx="60"
        cy="60"
        r={radius}
        fill="none"
        stroke="#e5e7eb"
        strokeWidth="10"
      />
      <circle
        cx="60"
        cy="60"
        r={radius}
        fill="none"
        stroke="#3b82f6"
        strokeWidth="10"
        strokeDasharray={circumference}
        strokeDashoffset={offset}
        strokeLinecap="round"
        transform="rotate(-90 60 60)"
      />
    </svg>
  )
}
```

**企业实战建议**：

- 复杂 SVG 组件建议拆分为 `Icon`、`Chart`、`Diagram` 等独立组件。
- 图标使用 `vite-plugin-svgr` 把 SVG 当 React 组件引入。
- 大数据量 SVG 在 React 中要注意避免频繁重渲染，可用 `useMemo`、`React.memo` 优化。

### 5.3 Vue 中使用 SVG

Vue 中可以使用 SVG 作为组件或 Symbol：

```vue
<template>
  <svg class="icon" aria-hidden="true">
    <use :xlink:href="`#icon-${name}`" />
  </svg>
</template>

<script setup>
defineProps({ name: String })
</script>

<style scoped>
.icon {
  width: 1em;
  height: 1em;
  fill: currentColor;
  vertical-align: middle;
}
</style>
```

配合 `vite-plugin-svg-icons` 自动注入雪碧图：

```javascript
// vite.config.js
import { createSvgIconsPlugin } from 'vite-plugin-svg-icons'
import path from 'path'

export default {
  plugins: [
    createSvgIconsPlugin({
      iconDirs: [path.resolve(process.cwd(), 'src/assets/icons')],
      symbolId: 'icon-[name]',
    }),
  ],
}
```

### 5.4 GSAP + SVG 动画

GSAP 是业界最强的动画库，配合 SVG 可以实现复杂的路径动画、形变、描边动画。

```javascript
import gsap from 'gsap'

// 描边动画
const path = document.querySelector('#animated-path')
const length = path.getTotalLength()

gsap.set(path, {
  strokeDasharray: length,
  strokeDashoffset: length,
})

gsap.to(path, {
  strokeDashoffset: 0,
  duration: 2,
  ease: 'power2.inOut',
})
```

```javascript
// 路径跟随动画
const follower = document.querySelector('#follower')

gsap.to(follower, {
  motionPath: {
    path: '#animated-path',
    align: '#animated-path',
    alignOrigin: [0.5, 0.5],
  },
  duration: 3,
  repeat: -1,
})
```

企业应用：复杂 Loading、数据流动效、Logo 演绎、交互引导动画。

### 5.5 SVG.js / Snap.svg

如果需要更轻量的 SVG 操作库，可以考虑：

- **SVG.js**：链式 API，现代化，TypeScript 支持好。
- **Snap.svg**：Adobe 出品，兼容性好，API 类似 jQuery。

```javascript
import { SVG } from '@svgdotjs/svg.js'

const draw = SVG().addTo('#canvas').size(300, 300)
draw.rect(100, 100).move(50, 50).fill('#3b82f6').radius(10)
draw.circle(50).move(200, 200).fill('#ef4444')
```

### 5.6 SVGO：SVG 压缩与优化

企业级项目必须做 SVG 优化，SVGO 可以：

- 移除无用属性、注释、隐藏元素。
- 合并路径、简化小数精度。
- 移除编辑器生成的冗余元数据。

```bash
npx svgo input.svg -o output.svg
```

配合 Vite 使用：

```javascript
import svgo from 'vite-plugin-svgo'

export default {
  plugins: [svgo()],
}
```

## 六、性能优化与工程化

### 6.1 减少 DOM 节点数量

SVG 中每个元素都是 DOM 节点，节点过多会导致重排和重绘变慢。

优化手段：

- 对静态图形使用 `use` 复用 `symbol`。
- 合并相邻的简单路径，减少元素数量。
- 大数据量时切换为 Canvas / WebGL。

### 6.2 使用 `transform` 代替频繁修改 `x/y`

```javascript
// 不推荐：频繁修改 x/y 会触发布局
circle.setAttribute('cx', newX)
circle.setAttribute('cy', newY)

// 推荐：使用 transform，只触发合成
circle.style.transform = `translate(${newX}px, ${newY}px)`
```

### 6.3 避免在 SVG 中大量使用滤镜

`filter` 会创建离屏缓冲，对 GPU 开销较大。动画中慎用动态滤镜。

### 6.4 按需加载与懒加载

- 大型 SVG 地图按区域拆分，按需加载。
- 不在首屏的插画使用 `loading="lazy"` 或 Intersection Observer。
- 图标雪碧图按业务模块拆分。

### 6.5 工程化配置

- 使用 `svgo` 压缩所有 SVG。
- 使用 `vite-plugin-svgr` / `unplugin-icons` / `vite-plugin-svg-icons` 提升开发体验。
- 在 CI 中加入 SVG 规范检查（尺寸、命名、颜色规范）。

## 七、常见坑与最佳实践

### 7.1 颜色继承问题

SVG 中 `fill` 和 `stroke` 默认继承自父元素，使用图标时建议：

```css
.icon {
  fill: currentColor;
}
```

这样图标颜色会跟随父元素的 `color` 属性变化。

### 7.2 `viewBox` 与 `width/height`

如果只设置 `viewBox` 而不设置 `width/height`，SVG 会按默认尺寸 300x150 渲染。建议始终显式声明 `width` / `height` 或使用 CSS 控制。

### 7.3 路径性能

复杂路径（`path` 数据点过多）会影响渲染和动画性能。设计稿导出后，用 SVGO 简化路径。

### 7.4 跨域与 CORS

使用 `img` 标签加载外部 SVG 时，SVG 中的脚本和外部资源会被禁用。如果需要交互，应使用 `object`、`embed` 或内联 SVG。

### 7.5 可访问性

```html
<svg role="img" aria-label="描述文字">...</svg>
```

对于装饰性图标，使用 `aria-hidden="true"` 避免干扰屏幕阅读器。

### 7.6 响应式 SVG

```css
.responsive-svg {
  width: 100%;
  height: auto;
  max-width: 100%;
}
```

配合 `viewBox` 可以实现任意容器内的自适应缩放。

## 八、总结

SVG 是企业前端工程中不可或缺的矢量图形技术：

- **想做图标**：用 Symbol 雪碧图 + `use`，配合 `svgo` 压缩。
- **想做图表**：D3.js / AntV / ECharts，SVG 负责可交互部分。
- **想做拓扑/流程图**：AntV X6 / React Flow，SVG 是底层载体。
- **想做动画**：GSAP + SVG，实现路径动画、形变、描边动效。
- **想做响应式图形**：用好 `viewBox` 和 CSS，适配所有屏幕。

> 当你需要“清晰、可交互、可缩放、可导出”的图形时，SVG 通常是比 Canvas 和位图更好的选择。
