## 一、什么是 PWA

PWA（Progressive Web App，渐进式 Web 应用）是利用现代 Web 技术提供接近原生应用体验的一类 Web 应用。它不需要用户通过应用商店下载安装，却能实现：

- **离线访问**：没有网络时也能展示核心页面。
- **可安装**：支持添加到主屏幕，拥有独立启动图标和全屏体验。
- **推送通知**：即使页面关闭，也能接收服务端推送。
- **后台同步**：网络恢复后自动补发之前失败的操作。
- **性能优越**：通过缓存减少加载时间，提升首屏体验。

PWA 的目标不是完全替代原生 App，而是在 Web 生态内提供“无需安装、即点即用、弱网可用”的体验。

---

## 二、PWA 核心技术栈

| 技术 | 作用 |
|------|------|
| **Service Worker** | 运行在浏览器后台的代理脚本，拦截网络请求、管理缓存、处理推送 |
| **Web App Manifest** | JSON 配置文件，描述应用名称、图标、主题色、启动方式 |
| **Cache Storage** | 浏览器提供的缓存接口，Service Worker 用它存储静态资源 |
| **Push API / Notification API** | 接收服务端推送并展示系统通知 |
| **Background Sync** | 网络恢复后自动执行延迟任务 |
| **Web Share API** | 调用系统原生分享面板 |

## 三、Service Worker 实战

### 3.1 注册 Service Worker

Service Worker 是 PWA 的核心，需要先注册才能生效。

```javascript
// main.js：应用入口文件中注册 Service Worker
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker
      .register('/sw.js')
      .then((registration) => {
        // 注册成功，打印作用域
        console.log('Service Worker 注册成功:', registration.scope)
      })
      .catch((error) => {
        // 注册失败，常见原因：sw.js 不在根目录或 HTTPS 未启用
        console.error('Service Worker 注册失败:', error)
      })
  })
}
```

### 3.2 Service Worker 生命周期

```javascript
// sw.js：Service Worker 脚本

const CACHE_NAME = 'app-v1'

// install 事件：首次安装时触发，适合缓存核心静态资源
self.addEventListener('install', (event) => {
  console.log('[SW] install 事件触发')

  // 等待缓存写入完成后再进入下一阶段
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => {
      // 预缓存关键资源：HTML、CSS、JS、图标等
      return cache.addAll([
        '/',
        '/index.html',
        '/styles.css',
        '/app.js',
        '/offline.html',
        '/icons/icon-192x192.png',
      ])
    })
  )

  // 跳过等待，立即激活新版本
  self.skipWaiting()
})

// activate 事件：Service Worker 被激活时触发，适合清理旧缓存
self.addEventListener('activate', (event) => {
  console.log('[SW] activate 事件触发')

  event.waitUntil(
    caches.keys().then((cacheNames) => {
      return Promise.all(
        cacheNames.map((name) => {
          // 删除不属于当前版本的旧缓存
          if (name !== CACHE_NAME) {
            console.log('[SW] 删除旧缓存:', name)
            return caches.delete(name)
          }
        })
      )
    })
  )

  // 立即接管所有客户端页面
  self.clients.claim()
})

// fetch 事件：拦截所有网络请求
self.addEventListener('fetch', (event) => {
  console.log('[SW] 拦截请求:', event.request.url)

  event.respondWith(
    caches.match(event.request).then((response) => {
      // 命中缓存则直接返回，否则走网络请求
      return response || fetch(event.request)
    })
  )
})
```

### 3.3 缓存优先 + 网络回退策略

```javascript
// sw.js

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.open(CACHE_NAME).then((cache) => {
      // 第一步：优先从缓存中匹配请求
      return cache.match(event.request).then((cachedResponse) => {
        // 第二步：同时发起网络请求，用于更新缓存
        const fetchPromise = fetch(event.request)
          .then((networkResponse) => {
            // 网络请求成功，把新响应写入缓存
            cache.put(event.request, networkResponse.clone())
            return networkResponse
          })
          .catch(() => {
            // 网络失败时静默处理，后续由缓存兜底
            console.warn('[SW] 网络请求失败，使用缓存:', event.request.url)
          })

        // 缓存命中则先返回缓存，否则等待网络结果
        return cachedResponse || fetchPromise
      })
    })
  )
})
```

## 四、缓存策略与离线方案

### 4.1 不同资源的缓存策略

企业项目中，不同资源类型需要不同的缓存策略：

```javascript
// sw.js

// 定义不同资源的缓存策略
const strategies = {
  // 静态资源：缓存优先，长期有效
  static: {
    pattern: /\.(js|css|png|jpg|svg|woff2?)$/,
    handler: cacheFirst,
  },
  // HTML 页面：网络优先，保证内容最新
  html: {
    pattern: /\/[^.]*$|\.(html)$/,
    handler: networkFirst,
  },
  // API 接口：网络优先，离线时回退到缓存
  api: {
    pattern: /\/api\//,
    handler: networkFirst,
  },
}

// 缓存优先策略
async function cacheFirst(request) {
  const cached = await caches.match(request)
  if (cached) return cached

  try {
    const networkResponse = await fetch(request)
    const cache = await caches.open(CACHE_NAME)
    cache.put(request, networkResponse.clone())
    return networkResponse
  } catch (error) {
    // 彻底离线且缓存不存在时返回离线页面
    return caches.match('/offline.html')
  }
}

// 网络优先策略
async function networkFirst(request) {
  try {
    const networkResponse = await fetch(request)
    const cache = await caches.open(CACHE_NAME)
    cache.put(request, networkResponse.clone())
    return networkResponse
  } catch (error) {
    const cached = await caches.match(request)
    return cached || caches.match('/offline.html')
  }
}

self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url)

  // 仅处理同源请求，避免跨域问题
  if (url.origin !== self.location.origin) return

  // 根据 URL 匹配策略
  for (const { pattern, handler } of Object.values(strategies)) {
    if (pattern.test(url.pathname)) {
      event.respondWith(handler(event.request))
      return
    }
  }
})
```

## 六、推送通知实战

### 6.1 申请通知权限并订阅 Push

```javascript
// main.js

/**
 * 订阅推送服务
 * @param {ServiceWorkerRegistration} registration - Service Worker 注册对象
 */
async function subscribePush(registration) {
  // 第一步：请求用户授权显示通知
  const permission = await Notification.requestPermission()
  if (permission !== 'granted') {
    console.warn('用户未授权通知权限')
    return
  }

  // 第二步：向推送服务申请订阅
  // applicationServerKey 是后端 VAPID 公钥的 Base64 URL Safe 编码
  const subscription = await registration.pushManager.subscribe({
    userVisibleOnly: true,
    applicationServerKey: urlBase64ToUint8Array(
      'BPxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'
    ),
  })

  // 第三步：把订阅信息发送给后端，后端用此信息推送消息
  await fetch('/api/push/subscribe', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(subscription),
  })

  console.log('推送订阅成功:', subscription)
}

/**
 * 将 Base64 URL Safe 字符串转换为 Uint8Array
 * @param {string} base64String - VAPID 公钥
 */
function urlBase64ToUint8Array(base64String) {
  const padding = '='.repeat((4 - (base64String.length % 4)) % 4)
  const base64 = (base64String + padding).replace(/-/g, '+').replace(/_/g, '/')
  const rawData = atob(base64)
  return Uint8Array.from(rawData, (char) => char.charCodeAt(0))
}
```

### 6.2 Service Worker 中处理推送

```javascript
// sw.js

// push 事件：收到服务端推送时触发
self.addEventListener('push', (event) => {
  // 解析推送数据
  const data = event.data ? event.data.json() : {}

  const title = data.title || '新消息'
  const options = {
    body: data.body || '您有一条新的通知',
    icon: '/icons/icon-192x192.png',
    badge: '/icons/badge-72x72.png',
    tag: data.tag || 'default',
    requireInteraction: false,
    data: {
      url: data.url || '/',
    },
  }

  // 展示系统通知
  event.waitUntil(self.registration.showNotification(title, options))
})

// notificationclick 事件：用户点击通知时触发
self.addEventListener('notificationclick', (event) => {
  event.notification.close()

  const urlToOpen = event.notification.data?.url || '/'

  event.waitUntil(
    self.clients
      .matchAll({ type: 'window', includeUncontrolled: true })
      .then((clientList) => {
        // 如果已有窗口打开，则聚焦；否则打开新窗口
        for (const client of clientList) {
          if (client.url === urlToOpen && 'focus' in client) {
            return client.focus()
          }
        }
        if (self.clients.openWindow) {
          return self.clients.openWindow(urlToOpen)
        }
      })
  )
})
```

## 七、企业落地场景

### 7.1 场景一：B 端管理系统离线可用

B 端系统用户经常遇到网络波动。PWA 可以让核心表单、列表、仪表盘在网络不稳定时仍可查看和填写。

```javascript
// 提交表单时优先网络，失败则存入 IndexedDB 等待同步
async function submitForm(data) {
  try {
    await fetch('/api/form', {
      method: 'POST',
      body: JSON.stringify(data),
    })
  } catch (error) {
    // 网络失败时保存到本地数据库
    await saveToPendingQueue(data)
    // 注册后台同步任务
    const registration = await navigator.serviceWorker.ready
    await registration.sync.register('sync-form')
  }
}
```

```javascript
// sw.js
self.addEventListener('sync', (event) => {
  if (event.tag === 'sync-form') {
    event.waitUntil(syncPendingForms())
  }
})

async function syncPendingForms() {
  const pendingForms = await getPendingForms()
  for (const form of pendingForms) {
    try {
      await fetch('/api/form', {
        method: 'POST',
        body: JSON.stringify(form),
      })
      await removeFromPendingQueue(form.id)
    } catch (error) {
      console.error('后台同步失败:', error)
    }
  }
}
```

### 7.2 场景二：移动端 H5 添加到主屏幕

电商、工具类 H5 页面通过 PWA 实现“类原生 App”体验：

- 用户访问页面时弹出“添加到主屏幕”提示。
- 全屏运行，隐藏浏览器地址栏。
- 启动时显示 Splash Screen，品牌感更强。

```javascript
// main.js：监听安装提示事件
let deferredPrompt

window.addEventListener('beforeinstallprompt', (event) => {
  // 阻止浏览器自动弹出安装提示
  event.preventDefault()
  deferredPrompt = event

  // 在合适的时机（如用户点击按钮）主动触发安装
  document.getElementById('install-btn').addEventListener('click', async () => {
    if (!deferredPrompt) return
    deferredPrompt.prompt()

    const choiceResult = await deferredPrompt.userChoice
    if (choiceResult.outcome === 'accepted') {
      console.log('用户同意安装应用')
    } else {
      console.log('用户拒绝安装应用')
    }
    deferredPrompt = null
  })
})
```

### 7.3 场景三：内容型站点离线阅读

博客、文档、新闻类站点可以缓存文章列表和正文，用户离线时也能继续阅读。

```javascript
// sw.js：缓存文章详情页
self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url)

  // 文章详情页使用 stale-while-revalidate 策略
  if (url.pathname.startsWith('/article/')) {
    event.respondWith(staleWhileRevalidate(event.request))
  }
})

async function staleWhileRevalidate(request) {
  const cache = await caches.open(CACHE_NAME)
  const cached = await cache.match(request)

  // 同时发起网络请求更新缓存
  const fetchPromise = fetch(request).then((networkResponse) => {
    cache.put(request, networkResponse.clone())
    return networkResponse
  })

  // 优先返回缓存，网络数据下次生效
  return cached || fetchPromise
}
```

## 八、调试与最佳实践

### 8.1 Chrome DevTools 调试

- 打开 `Application` 面板 → `Service Workers`，查看注册状态、作用域、是否跳过等待。
- `Cache Storage` 中查看已缓存资源。
- `Manifest` 中检查 Manifest 配置是否正确。
- `Push` / `Notifications` 中可模拟推送测试。

### 8.2 最佳实践

1. **始终使用 HTTPS**：Service Worker 只能在 HTTPS 或 localhost 环境下运行。
2. **缓存版本管理**：每次发版更新 `CACHE_NAME`，并在 `activate` 中清理旧缓存。
3. **避免缓存过多**：预缓存只放核心资源，其他资源按需缓存。
4. **谨慎处理 POST 请求**：默认不要缓存 POST 请求，避免脏数据。
5. **注册时机**：在 `window.onload` 后注册 Service Worker，避免阻塞首屏资源加载。
6. **友好降级**：在不支持 PWA 的浏览器中保持正常 Web 体验。

### 8.3 常见坑

- **Service Worker 作用域**：`sw.js` 的位置决定了它能拦截哪些请求，建议放在根目录。
- **缓存不更新**：旧版本缓存未清理会导致用户看不到新版本，务必更新 `CACHE_NAME`。
- **跨域资源无法缓存**：第三方 CDN 资源默认无法缓存，需要配置 CORS。
- **Manifest 路径错误**：路径必须以 `/` 开头，且图标尺寸要符合规范。

## 九、总结

PWA 是提升 Web 应用体验和稳定性的重要手段：

- **离线可用**：通过 Service Worker + Cache Storage 缓存核心资源。
- **可安装**：通过 Web App Manifest 让应用拥有原生般的启动体验。
- **消息触达**：通过 Push API + Notification API 实现推送通知。
- **后台同步**：通过 Background Sync 补发离线期间的操作。
- **生产落地**：B 端系统、移动 H5、内容站点是最适合 PWA 改造的场景。

> 当你希望 Web 应用在弱网、离线、移动端场景下依然稳定可用时，PWA 是首选方案。
