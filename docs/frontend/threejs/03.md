## 一、先给结论：三者是什么关系

```
┌─────────────────────────────────────────────────────────────┐
│                      浏览器 / JavaScript                       │
├─────────────────────────────────────────────────────────────┤
│   WebGIS (Cesium/Mapbox/Leaflet)                            │
│   ┌─────────────────────────────────────────────────────┐   │
│   │   Three.js                                          │   │
│   │   ┌─────────────────────────────────────────────┐   │   │
│   │   │   WebGL / WebGPU                              │   │   │
│   │   │   ┌─────────────────────────────────────┐   │   │   │
│   │   │   │   OpenGL ES / DirectX / Metal / Vulkan│   │   │   │
│   │   │   └─────────────────────────────────────┘   │   │   │
│   │   └─────────────────────────────────────────────┘   │   │
│   └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
        GPU 驱动
            │
            ▼
           GPU
```

**一句话概括：**

| 技术 | 本质 | 类比 |
|------|------|------|
| **WebGL** | GPU 的 JavaScript API | 汇编语言 — 直接操作硬件 |
| **Three.js** | WebGL 的高级封装库 | React/Vue — 工程化框架 |
| **WebGIS** | 地理信息可视化平台 | 钉钉/飞书 — 垂直领域应用层 |

**关键认知**：Three.js 和 WebGIS 是同一层级的东西吗？**不是**。Three.js 可以跑在 WebGIS 里面，WebGIS 也可以用 Three.js 做 3D 地球。它们是**可以交叉**的关系，不是**并列替代**的关系。

## 二、WebGL：GPU 的 JavaScript 接口

### 2.1 技术本质

WebGL（Web Graphics Library）是 Khronos Group 制定的标准，基于 OpenGL ES。它给 JavaScript 提供了直接调用 GPU 的能力。

WebGL 程序的核心就做两件事：
1. **创建 Shader**（顶点着色器 + 片段着色器）
2. **准备数据**（顶点数据、纹理、Uniform 变量）然后提交给 GPU 绘制

### 2.2 用 WebGL 画一个三角形（原教旨主义写法）

```javascript
// 获取 WebGL 上下文
const canvas = document.getElementById('canvas')
const gl = canvas.getContext('webgl2')

// === 顶点着色器 ===
const vertexShaderSource = `#version 300 es
in vec2 a_position;
void main() {
  gl_Position = vec4(a_position, 0.0, 1.0);
}
`

// === 片段着色器 ===
const fragmentShaderSource = `#version 300 es
precision highp float;
out vec4 outColor;
void main() {
  outColor = vec4(1.0, 0.0, 0.0, 1.0); // 红色
}
`

// 编译 Shader
function createShader(gl, type, source) {
  const shader = gl.createShader(type)
  gl.shaderSource(shader, source)
  gl.compileShader(shader)
  return shader
}

const vs = createShader(gl, gl.VERTEX_SHADER, vertexShaderSource)
const fs = createShader(gl, gl.FRAGMENT_SHADER, fragmentShaderSource)

// 链接着色器程序
const program = gl.createProgram()
gl.attachShader(program, vs)
gl.attachShader(program, fs)
gl.linkProgram(program)
gl.useProgram(program)

// 准备顶点数据
const positions = new Float32Array([
  0.0,  0.5,   // 顶点1
 -0.5, -0.5,   // 顶点2
  0.5, -0.5    // 顶点3
])

const positionBuffer = gl.createBuffer()
gl.bindBuffer(gl.ARRAY_BUFFER, positionBuffer)
gl.bufferData(gl.ARRAY_BUFFER, positions, gl.STATIC_DRAW)

// 告诉 Shader 如何读取顶点数据
const positionLocation = gl.getAttribLocation(program, 'a_position')
gl.enableVertexAttribArray(positionLocation)
gl.vertexAttribPointer(positionLocation, 2, gl.FLOAT, false, 0, 0)

// 绘制！
gl.drawArrays(gl.TRIANGLES, 0, 3)
```

**看到了吗？** 画一个红色三角形需要 40+ 行代码。如果要画一个带纹理、有光照、能旋转的立方体，需要 500+ 行。如果要画一个完整的 3D 场景（相机、控制器、阴影、后处理），需要 3000+ 行。

### 2.3 WebGL 的能力边界

| 能力 | WebGL 1.0 | WebGL 2.0 |
|------|-----------|-----------|
| GLSL 版本 | 1.0 | 3.0 |
| 3D 纹理 | ❌ | ✅ |
| 实例化渲染 | ❌ | ✅ |
| MRT（多渲染目标） | ❌ | ✅ |
| Transform Feedback | ❌ | ✅ |
| 最大纹理尺寸 | 4096 | 16384 |
| Compute Shader | ❌ | ❌ |

**WebGL 2.0 是目前浏览器的主流支持版本。** Three.js v0.184 的 WebGLRenderer 默认使用 WebGL 2.0，如果不可用则降级到 1.0。

### 2.4 什么时候直接写 WebGL？

| 场景 | 建议 |
|------|------|
| 特殊视觉效果（流体、粒子、后处理） | 直接写 Shader |
| 极致性能优化（自定义剔除、GPU Driven） | 直接写 WebGL |
| 学习 GPU 编程原理 | 必须写 WebGL |
| 常规 3D 应用 | **不要写**，用 Three.js |
| 团队多人协作 | **不要写**，维护成本极高 |

## 三、Three.js：WebGL 的工程化封装

### 3.1 Three.js 做了什么封装

```
WebGL 原语          Three.js 封装
─────────────────────────────────────
Shader Program  →   Material（材质）
Vertex Buffer   →   Geometry（几何体）
Uniform         →   Material 属性
Texture         →   TextureLoader
FrameBuffer     →   RenderTarget
Matrix Math     →   Vector3/Matrix4/Quaternion
Camera Matrix   →   Camera/Projection
Depth Buffer    →   ShadowMap
Blending        →   Material.blending
Stencil Test    →   Material.stencil*系列
```

### 3.2 同样的三角形，Three.js 写法

```javascript
import * as THREE from 'three'

const scene = new THREE.Scene()
const camera = new THREE.PerspectiveCamera(75, 1, 0.1, 1000)
const renderer = new THREE.WebGLRenderer({ canvas })

// 定义三角形几何体
const geometry = new THREE.BufferGeometry()
const vertices = new Float32Array([
   0.0,  0.5, 0.0,
  -0.5, -0.5, 0.0,
   0.5, -0.5, 0.0
])
geometry.setAttribute('position', new THREE.BufferAttribute(vertices, 3))

// 红色材质
const material = new THREE.MeshBasicMaterial({ color: 0xff0000 })

// 创建 Mesh
const triangle = new THREE.Mesh(geometry, material)
scene.add(triangle)

camera.position.z = 2
renderer.render(scene, camera)
```

**5 行核心代码 vs 40 行 WebGL 代码。** 这就是工程化封装的价值。

### 3.4 Three.js 的不可替代性

为什么不用 Babylon.js、PlayCanvas 或其他引擎？

| 维度 | Three.js | Babylon.js | PlayCanvas |
|------|----------|------------|------------|
| 社区生态 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 学习资源 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 灵活性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 内置功能 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 包体积 | ⭐⭐⭐⭐⭐ (tree-shaking) | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 编辑器 | 无 (代码优先) | 有 | 有 (在线) |
| 商业化支持 | 无 | 有 (Microsoft) | 有 |

**选型建议：**
- **Three.js**：需要深度定制、与前端框架深度集成、对包体积敏感
- **Babylon.js**：需要开箱即用的完整功能（物理、音频、GUI）、Microsoft 生态
- **PlayCanvas**：需要浏览器内的可视化编辑器、快速原型

## 四、WebGIS：地理空间的 Web 可视化

### 4.1 WebGIS 的本质

WebGIS（Web Geographic Information System）不是一项"图形技术"，而是一个**垂直领域解决方案**。它的核心能力是：

1. **地理数据管理**：坐标系转换、投影、瓦片切分
2. **地图服务协议**：WMS、WMTS、TMS、矢量瓦片 (MVT)
3. **空间分析**：缓冲区分析、叠加分析、路径规划
4. **多源数据融合**：卫星影像、DEM 高程、矢量数据、POI

### 4.2 WebGIS 技术栈

```
┌─────────────────────────────────────────────┐
│            应用层 (Application)               │
│   智慧城市  物流追踪  气象监控  国土规划       │
├─────────────────────────────────────────────┤
│            SDK 层 (JavaScript SDK)           │
│   CesiumJS  MapboxGL  ArcGIS API  OpenLayers │
├─────────────────────────────────────────────┤
│            渲染层 (Renderer)                  │
│   WebGL (2D/3D)  Canvas 2D  SVG             │
├─────────────────────────────────────────────┤
│            数据层 (Data Source)               │
│   瓦片服务  WMS/WMTS  GeoJSON  3DTiles      │
├─────────────────────────────────────────────┤
│            服务层 (Backend)                   │
│   GeoServer  MapServer  PostGIS  ArcGIS      │
└─────────────────────────────────────────────┘
```

### 4.3 主流 WebGIS 引擎对比

| 引擎 | 渲染器 | 3D 能力 | 坐标系 | 开源 | 适用场景 |
|------|--------|---------|--------|------|----------|
| **CesiumJS** | WebGL | ⭐⭐⭐⭐⭐ | WGS84/ECEF | ✅ | 地球级 3D、卫星、航测 |
| **Mapbox GL JS** | WebGL | ⭐⭐⭐ | Web Mercator | ✅(v1) | 矢量地图、个性化样式 |
| **ArcGIS API** | WebGL/Canvas | ⭐⭐⭐⭐ | 全支持 | ❌ | 企业级 GIS、政府项目 |
| **OpenLayers** | Canvas/WebGL | ⭐⭐ | 全支持 | ✅ | 传统 GIS、打印输出 |
| **Deck.gl** | WebGL | ⭐⭐⭐ | Web Mercator | ✅ | 大数据可视化、轨迹 |
| **mars3D** | WebGL(Cesium) | ⭐⭐⭐⭐⭐ | 全支持 | ✅ | 国内项目、国产化适配 |

### 4.4 CesiumJS 示例：加载 3D 地球

```javascript
import * as Cesium from 'cesium'

// Cesium 的"Hello World"是加载一个 3D 地球
const viewer = new Cesium.Viewer('cesiumContainer', {
  terrainProvider: Cesium.createWorldTerrain(),
  imageryProvider: new Cesium.IonImageryProvider({ assetId: 2 }),
  baseLayerPicker: true,
  geocoder: true
})

// 添加 3D 建筑瓦片
const tileset = viewer.scene.primitives.add(
  new Cesium.Cesium3DTileset({
    url: Cesium.IonResource.fromAssetId(96188)
  })
)

// 飞到指定位置
viewer.camera.flyTo({
  destination: Cesium.Cartesian3.fromDegrees(116.3974, 39.9093, 1000),
  orientation: {
    heading: Cesium.Math.toRadians(0),
    pitch: Cesium.Math.toRadians(-45)
  }
})
```

**注意**：CesiumJS 内部也是用 WebGL 渲染的。它有自己的 Scene、Camera、Renderer —— 和 Three.js 类似。但 Cesium 的 Camera 是**以地球为中心**的，所有坐标都要经过**地理投影变换**。

## 七、融合场景：三者如何协同工作

### 7.1 Three.js 嵌入 Cesium：3D 地球 + 自定义效果

```javascript
// 在 Cesium 地球表面渲染 Three.js 粒子效果
import * as Cesium from 'cesium'
import * as THREE from 'three'

// Cesium 场景
const viewer = new Cesium.Viewer('cesiumContainer')

// Three.js 场景（叠加在 Cesium 之上）
const scene = new THREE.Scene()
const camera = new THREE.PerspectiveCamera()
const renderer = new THREE.WebGLRenderer({
  alpha: true,
  canvas: viewer.scene.canvas
})

// 同步相机：将 Cesium 相机矩阵转换为 Three.js
cesiumToThreejsCamera(viewer.camera, camera)

function cesiumToThreejsCamera(cesiumCamera, threeCamera) {
  // Cesium 使用列主序矩阵，Three.js 使用行主序
  const cvm = cesiumCamera.viewMatrix
  const pem = cesiumCamera.frustum.projectionMatrix

  // 转换视图矩阵
  threeCamera.matrixWorldInverse.fromArray([
    cvm[0], cvm[4], cvm[8],  cvm[12],
    cvm[1], cvm[5], cvm[9],  cvm[13],
    cvm[2], cvm[6], cvm[10], cvm[14],
    cvm[3], cvm[7], cvm[11], cvm[15]
  ])

  // 转换投影矩阵
  threeCamera.projectionMatrix.fromArray([
    pem[0], pem[4], pem[8],  pem[12],
    pem[1], pem[5], pem[9],  pem[13],
    pem[2], pem[6], pem[10], pem[14],
    pem[3], pem[7], pem[11], pem[15]
  ])
}

// 在地球上添加 Three.js 效果（如台风粒子）
function addTyphoonEffect(longitude, latitude) {
  const position = Cesium.Cartesian3.fromDegrees(longitude, latitude, 0)

  // 创建粒子系统
  const particleCount = 10000
  const geometry = new THREE.BufferGeometry()
  const positions = new Float32Array(particleCount * 3)

  for (let i = 0; i < particleCount; i++) {
    const angle = Math.random() * Math.PI * 2
    const radius = Math.random() * 50000 // 50km
    positions[i * 3] = position.x + Math.cos(angle) * radius
    positions[i * 3 + 1] = position.y + Math.random() * 20000
    positions[i * 3 + 2] = position.z + Math.sin(angle) * radius
  }

  geometry.setAttribute('position', new THREE.BufferAttribute(positions, 3))

  const material = new THREE.PointsMaterial({
    color: 0x88ccff,
    size: 2000,
    transparent: true,
    opacity: 0.6,
    blending: THREE.AdditiveBlending
  })

  scene.add(new THREE.Points(geometry, material))
}

// 渲染循环：先 Cesium 后 Three.js
viewer.scene.postRender.addEventListener(() => {
  cesiumToThreejsCamera(viewer.camera, camera)
  renderer.state.reset() // 重置 WebGL 状态
  renderer.render(scene, camera)
})
```

### 7.3 Three.js + 地理数据：自定义 3D 地图

如果不需要完整的 WebGIS，只是想在 Three.js 中展示地理数据：

```javascript
import * as THREE from 'three'

// 墨卡托投影（Web Mercator）
function mercatorProject(lng, lat) {
  const x = (lng + 180) / 360
  const y = (1 - Math.log(Math.tan(lat * Math.PI / 180) +
    1 / Math.cos(lat * Math.PI / 180)) / Math.PI) / 2
  return { x, y }
}

// 加载 GeoJSON 数据并渲染
async function loadGeoJSON(url) {
  const response = await fetch(url)
  const geojson = await response.json()

  const group = new THREE.Group()

  geojson.features.forEach(feature => {
    if (feature.geometry.type === 'Polygon') {
      const shape = new THREE.Shape()

      feature.geometry.coordinates[0].forEach(([lng, lat], i) => {
        const { x, y } = mercatorProject(lng, lat)
        if (i === 0) shape.moveTo(x * 1000, y * 1000)
        else shape.lineTo(x * 1000, y * 1000)
      })

      const geometry = new THREE.ExtrudeGeometry(shape, {
        depth: Math.random() * 50 + 10,
        bevelEnabled: false
      })

      const material = new THREE.MeshStandardMaterial({
        color: new THREE.Color().setHSL(Math.random() * 0.2 + 0.5, 0.8, 0.5)
      })

      group.add(new THREE.Mesh(geometry, material))
    }
  })

  return group
}
```

## 八、常见误区与陷阱

### 误区 1："WebGL 比 Three.js 性能好"

**错误。** Three.js 的渲染开销几乎为零。它生成的 WebGL 命令和你手写的一样。性能差异来自：
- 你的 Shader 写得比 Three.js 好（ unlikely ）
- Three.js 的默认设置不够激进（可以通过配置优化）
- 你不需要 Three.js 的通用性，可以做专门优化

### 误区 2："WebGIS 可以替代 Three.js"

**错误。** CesiumJS 擅长地球级大场景，但在以下方面不如 Three.js：
- 精细的 PBR 材质和光照
- 复杂的粒子特效
- 骨骼动画和蒙皮
- 自定义 Shader 的灵活性
- 包体积（Cesium 是 Three.js 的 3-5 倍）

### 误区 3："选了一个就不能换"

**错误。** 三者可以混合使用：
- Mapbox 做底图 + Three.js 做 3D 数据层
- Cesium 做地球 + Three.js 做特效
- Three.js 做场景 + 手写 Shader 做特殊效果

### 误区 4："WebGL 已经被 WebGPU 取代了"

**错误。** WebGPU 目前（2026年）的浏览器支持率约为 75%，且 Safari 支持有限。WebGL 2.0 仍然是最安全的生产选择。Three.js v0.184 同时支持 WebGL 和 WebGPU，可以无缝切换。

### 误区 5："3D 一定比 2D 好"

**错误。** 3D 的代价：
- 开发成本高 3-5 倍
- 性能要求更高
- 移动端续航下降
- 部分用户会晕 3D

以下场景**不要**用 3D：
- 纯数据表格展示
- 简单的趋势图表
- 信息密度极高的仪表盘
- 目标用户主要是低配手机

## 总结

| 问题 | 答案 |
|------|------|
| 学哪个？ | **先学 Three.js**，需要时再补 WebGL/WebGIS |
| 做地理项目？ | **CesiumJS/Mapbox** 打底，**Three.js** 做特效补充 |
| 性能瓶颈在哪？ | 不在 WebGL vs Three.js 的选择，在** draw call 数量**和**纹理内存** |
| 未来趋势？ | WebGPU 逐步替代 WebGL，TSL 统一 Shader 开发，WebGIS 与游戏引擎边界模糊 |

> **核心建议**：作为前端工程师，掌握 Three.js 是性价比最高的选择。它覆盖了 80% 的 3D 业务场景，且可以向下延伸（写 Shader）或向上整合（嵌入 WebGIS）。不要从零写 WebGL，也不要把 WebGIS 当作万能方案。
