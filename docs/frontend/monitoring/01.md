# 前端异常监控实战指南

---

## 一、为什么需要前端异常监控

用户侧的前端故障往往具有以下特点：

- **不可见**：开发环境难以复现，只有真实用户才能触发。
- **链路长**：涉及浏览器版本、网络环境、CDN、后端接口、第三方脚本等多方因素。
- **影响大**：一次 JS 报错可能导致整页白屏、按钮失效、交易中断。

异常监控的目标不是消灭所有报错，而是：

1. **第一时间发现**线上问题；
2. **快速定位**报错位置和上下文；
3. **衡量影响范围**（用户数、页面数、版本号）；
4. **建立修复闭环**（告警 → 排查 → 修复 → 验证）。

---

## 二、前端错误类型全景

| 类型 | 典型场景 | 捕获方式 |
|------|---------|---------|
| JS 运行时错误 | 空指针、类型错误、越界访问 | `window.onerror` |
| Promise 未捕获异常 | `async/await` 未加 `try/catch` | `unhandledrejection` |
| 资源加载失败 | 图片、JS、CSS、字体 404 / 跨域 | `addEventListener('error', ..., true)` |
| 接口错误 | HTTP 4xx / 5xx、超时、网络断开 | 请求拦截器 |
| WebSocket 异常 | 连接断开、消息解析失败 | `ws.onerror` / `ws.onclose` |
| 框架级错误 | Vue/React 组件渲染异常 | `errorCaptured` / `Error Boundary` |
| 业务自定义异常 | 表单校验失败、权限不足 | 主动 `tracker.capture()` |
| 跨域脚本错误 | `Script error.` | CORS + `crossorigin` |

## 三、全局错误捕获

### 3.1 JS 运行时错误：`window.onerror`

```javascript
window.onerror = function (message, source, lineno, colno, error) {
  tracker.report({
    type: 'js-error',
    message,
    source,
    lineno,
    colno,
    stack: error?.stack,
    timestamp: Date.now()
  })

  // 返回 true 可阻止浏览器默认报错打印
  return true
}
```

> 注意：通过 `new Function` 或 `setTimeout` 抛出的错误，部分浏览器不会携带完整堆栈。

### 3.2 Promise 未捕获异常：`unhandledrejection`

```javascript
window.addEventListener('unhandledrejection', (event) => {
  tracker.report({
    type: 'promise-error',
    reason: event.reason?.message || String(event.reason),
    stack: event.reason?.stack,
    timestamp: Date.now()
  })

  // 阻止控制台输出未被处理的 Promise 错误提示
  event.preventDefault()
})
```

### 3.3 资源加载错误

```javascript
window.addEventListener('error', (event) => {
  const target = event.target
  if (!target || !target.tagName) return

  const tag = target.tagName.toLowerCase()
  if (['img', 'script', 'link', 'video', 'audio'].includes(tag)) {
    tracker.report({
      type: 'resource-error',
      tag,
      src: target.src || target.href,
      timestamp: Date.now()
    })
  }
}, true) // 捕获阶段才能监听到资源错误
```

### 3.4 跨域脚本错误处理

当加载跨域 JS 文件时，浏览器会隐藏详细错误信息，只报 `Script error.`。

```html
<!-- 脚本标签添加 crossorigin -->
<script src="https://cdn.example.com/app.js" crossorigin="anonymous"></script>
```

## 四、框架级错误边界

### 4.1 Vue 3：`onErrorCaptured`

```vue
<script setup>
import { onErrorCaptured } from 'vue'

onErrorCaptured((err, instance, info) => {
  tracker.report({
    type: 'vue-error',
    message: err.message,
    stack: err.stack,
    component: instance?.$options?.name || 'anonymous',
    info
  })

  // 返回 false 阻止错误继续向上传播
  return false
})
</script>
```

### 4.2 React：Error Boundary

```jsx
import React from 'react'

class ErrorBoundary extends React.Component {
  state = { hasError: false }

  static getDerivedStateFromError() {
    return { hasError: true }
  }

  componentDidCatch(error, info) {
    tracker.report({
      type: 'react-error',
      message: error.message,
      stack: error.stack,
      componentStack: info.componentStack
    })
  }

  render() {
    if (this.state.hasError) {
      return <div>模块加载失败，请刷新重试</div>
    }
    return this.props.children
  }
}
```

### 4.3 Vue 全局错误处理器

```javascript
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)

app.config.errorHandler = (err, instance, info) => {
  tracker.report({
    type: 'vue-global-error',
    message: err.message,
    stack: err.stack,
    info
  })
}

app.mount('#app')
```

## 五、接口与业务错误监控

### 5.1 封装请求拦截器

```javascript
// http.js
class HttpClient {
  constructor() {
    this.baseURL = '/api'
  }

  async request(url, options = {}) {
    const start = performance.now()
    const fullUrl = `${this.baseURL}${url}`

    try {
      const response = await fetch(fullUrl, {
        ...options,
        headers: {
          'Content-Type': 'application/json',
          ...options.headers
        }
      })

      const duration = Math.round(performance.now() - start)

      if (!response.ok) {
        tracker.report({
          type: 'api-error',
          url: fullUrl,
          method: options.method || 'GET',
          status: response.status,
          statusText: response.statusText,
          duration
        })
      }

      return response
    } catch (error) {
      tracker.report({
        type: 'api-error',
        url: fullUrl,
        method: options.method || 'GET',
        message: error.message,
        duration: Math.round(performance.now() - start)
      })
      throw error
    }
  }
}
```

### 5.2 WebSocket 错误监控

```javascript
class TrackedWebSocket {
  constructor(url) {
    this.ws = new WebSocket(url)

    this.ws.onerror = (event) => {
      tracker.report({
        type: 'websocket-error',
        url,
        readyState: this.ws.readyState
      })
    }

    this.ws.onclose = (event) => {
      if (!event.wasClean) {
        tracker.report({
          type: 'websocket-abnormal-close',
          url,
          code: event.code,
          reason: event.reason
        })
      }
    }
  }
}
```

## 六、Sourcemap 与堆栈还原

生产环境代码经过压缩混淆后，错误堆栈无法直接定位源码。

```javascript
// vite.config.js
export default {
  build: {
    sourcemap: 'hidden' // 生成 .map 但不暴露引用
  }
}
```

### 6.1 上报时收集关键信息

```javascript
tracker.report({
  type: 'js-error',
  message: error.message,
  stack: error.stack,
  filename: error.filename || source,
  lineno,
  colno,
  appVersion: __APP_VERSION__,
  commitHash: __COMMIT_HASH__
})
```

### 6.2 监控平台还原流程

```
前端上报 (压缩堆栈 + 版本号)
    │
    ▼
监控平台根据版本号拉取对应 sourcemap
    │
    ▼
使用 source-map-js / mozilla/source-map 解析原始位置
    │
    ▼
展示源码文件、行号、列号、对应函数名
```

> 安全注意：sourcemap 文件**不要部署到公网**，只上传到内部监控平台或私有存储。

## 七、设计一个轻量级异常监控 SDK

```javascript
class ExceptionTracker {
  constructor(options = {}) {
    this.reportUrl = options.reportUrl || '/api/log'
    this.sampleRate = options.sampleRate ?? 1.0
    this.maxBreadcrumbs = options.maxBreadcrumbs ?? 20
    this.queue = []
    this.breadcrumbs = []
    this.timer = null
    this.batchSize = options.batchSize ?? 10
    this.flushInterval = options.flushInterval ?? 5000

    this.init()
  }

  init() {
    this._hookWindowError()
    this._hookPromiseError()
    this._hookResourceError()
    this._startFlushTimer()
  }

  // 添加用户行为面包屑
  addBreadcrumb(data) {
    this.breadcrumbs.push({
      ...data,
      time: Date.now()
    })
    if (this.breadcrumbs.length > this.maxBreadcrumbs) {
      this.breadcrumbs.shift()
    }
  }

  // 主动捕获异常
  capture(error, context = {}) {
    this.report({
      type: 'manual',
      message: error.message,
      stack: error.stack,
      ...context
    })
  }

  report(data) {
    if (Math.random() > this.sampleRate) return

    const payload = {
      ...data,
      url: location.href,
      userAgent: navigator.userAgent,
      breadcrumbs: [...this.breadcrumbs],
      timestamp: Date.now()
    }

    this.queue.push(payload)

    if (this.queue.length >= this.batchSize) {
      this.flush()
    }
  }

  async flush() {
    if (!this.queue.length) return

    const batch = this.queue.splice(0, this.batchSize)

    try {
      if (navigator.sendBeacon && this.reportUrl) {
        navigator.sendBeacon(this.reportUrl, JSON.stringify(batch))
      } else {
        await fetch(this.reportUrl, {
          method: 'POST',
          body: JSON.stringify(batch),
          headers: { 'Content-Type': 'application/json' },
          keepalive: true
        })
      }
    } catch (e) {
      // 上报失败时避免递归报错
      console.warn('异常上报失败', e)
    }
  }

  _hookWindowError() {
    window.onerror = (message, source, lineno, colno, error) => {
      this.report({
        type: 'js-error',
        message,
        source,
        lineno,
        colno,
        stack: error?.stack
      })
      return true
    }
  }

  _hookPromiseError() {
    window.addEventListener('unhandledrejection', (event) => {
      this.report({
        type: 'promise-error',
        reason: event.reason?.message || String(event.reason),
        stack: event.reason?.stack
      })
      event.preventDefault()
    })
  }

  _hookResourceError() {
    window.addEventListener('error', (event) => {
      const target = event.target
      if (!target?.tagName) return
      const tag = target.tagName.toLowerCase()
      if (['img', 'script', 'link', 'video', 'audio'].includes(tag)) {
        this.report({
          type: 'resource-error',
          tag,
          src: target.src || target.href
        })
      }
    }, true)
  }

  _startFlushTimer() {
    this.timer = setInterval(() => this.flush(), this.flushInterval)
  }

  destroy() {
    clearInterval(this.timer)
    window.onerror = null
  }
}

// 使用
const tracker = new ExceptionTracker({
  reportUrl: '/api/errors',
  sampleRate: 1.0,
  batchSize: 5,
  flushInterval: 3000
})

export default tracker
```

### 场景 2：React 项目完整接入

```jsx
// ErrorBoundary.jsx
import React from 'react'
import tracker from './monitor/tracker'

class ErrorBoundary extends React.Component {
  state = { hasError: false }

  static getDerivedStateFromError() {
    return { hasError: true }
  }

  componentDidCatch(error, info) {
    tracker.capture(error, {
      type: 'react-error',
      componentStack: info.componentStack
    })

  render() {
    return this.state.hasError
      ? this.props.fallback || <div>出错了，请刷新</div>
      : this.props.children
  }
}
```

### 场景 3：过滤第三方脚本噪声

```javascript
function isThirdPartyError(source = '') {
  const thirdPartyDomains = [
    'cdn.example.com',
    'ad.network.com',
    'chat-widget.js'
  ]
  return thirdPartyDomains.some(domain => source.includes(domain))
}

window.onerror = (message, source, lineno, colno, error) => {
  if (isThirdPartyError(source)) return true

  tracker.capture(error || new Error(message), {
    type: 'js-error',
    source,
    lineno,
    colno
  })
  return true
}
```

### 场景 4：大屏 7×24 小时运行告警

```javascript
// 关键错误立即上报，不等待批量
function criticalReport(data) {
  fetch('/api/alarm', {
    method: 'POST',
    body: JSON.stringify(data),
    keepalive: true
  })
}

window.onerror = (message, source, lineno, colno, error) => {
  const payload = {
    type: 'js-error',
    message,
    source,
    lineno,
    colno,
    stack: error?.stack,
    page: location.href

    criticalReport({ ...payload, level: 'critical' })
  }

  tracker.report(payload)
  return true
}
```

## 九、最佳实践与自检清单

| 实践 | 说明 |
|------|------|
| 采样上报 | 高流量场景设置 `sampleRate`，避免压垮服务端 |
| 批量与异步 | 使用队列 + 定时器批量上报，优先使用 `sendBeacon` |
| 错误去重 | 同一错误短时间内只上报一次，减少噪音 |
| 版本关联 | 上报携带版本号、commit hash、构建时间 |
| Sourcemap 管理 | 不暴露公网，监控平台私有存储 |
| 用户隐私 | 不上报敏感字段（密码、token、手机号） |
| 降级策略 | 上报服务不可用时不影响业务主流程 |
| 面包屑 | 保留用户操作路径，辅助复现问题 |
