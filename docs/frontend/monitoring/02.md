# 前端性能监控实战指南

> 系统梳理前端性能指标体系（Web Vitals、自定义指标）、采集方案、RUM 实现与典型场景下的性能治理经验。

---

## 一、为什么需要前端性能监控

性能问题往往具备以下特征：

- **用户感知强**：首屏慢、卡顿、布局抖动都会直接影响转化率。
- **环境差异大**：低端机、弱网、长距离 CDN 都会放大性能问题。
- **本地难复现**：开发机性能通常优于用户真实设备。

性能监控的核心价值：

1. **量化体验**：把“页面有点卡”变成可衡量的指标。
2. **发现瓶颈**：定位是资源加载慢、JS 执行慢，还是渲染卡顿。
3. **指导优化**：用数据验证优化效果，避免盲目重构。
4. **建立基线**：持续跟踪核心指标，防止劣化。

## 二、核心性能指标体系

### 2.1 Web Vitals（Google 推荐核心指标）

| 指标 | 全称 | 含义 | 优秀阈值 | 测量工具 |
|------|------|------|---------|---------|
| **LCP** | Largest Contentful Paint | 最大内容元素渲染时间 | ≤ 2.5s | PerformanceObserver |
| **INP** | Interaction to Next Paint | 交互到下一帧绘制时间 | ≤ 200ms | PerformanceObserver |
| **CLS** | Cumulative Layout Shift | 累积布局偏移 | ≤ 0.1 | PerformanceObserver |
| **FCP** | First Contentful Paint | 首次内容绘制 | ≤ 1.8s | PerformanceObserver |
| **TTFB** | Time to First Byte | 首字节时间 | ≤ 800ms | Navigation Timing |

### 2.2 传统与自定义指标

| 指标 | 含义 | 计算方式 |
|------|------|---------|
| **FP** | First Paint | `performance.getEntriesByName('first-paint')[0]` |
| **FMP** | First Meaningful Paint | 自定义标记或 heuristics |
| **TTI** | Time to Interactive | 可交互时间，Long Task 结束后 |
| **DCL** | DOMContentLoaded | `domContentLoadedEventEnd - navigationStart` |
| **Load** | 完全加载 | `loadEventEnd - navigationStart` |
| **TBT** | Total Blocking Time | Long Task 阻塞总时长 |
| **FIP** | First Input Delay | 首次输入延迟（已被 INP 取代） |

### 2.3 性能指标评级参考

```
LCP:  good ≤ 2.5s,  needs-improvement ≤ 4.0s,  poor > 4.0s
INP:  good ≤ 200ms, needs-improvement ≤ 500ms, poor > 500ms
CLS:  good ≤ 0.1,   needs-improvement ≤ 0.25,  poor > 0.25
FCP:  good ≤ 1.8s,  needs-improvement ≤ 3.0s,  poor > 3.0s
TTFB: good ≤ 800ms, needs-improvement ≤ 1.8s,  poor > 1.8s
```

### 3.2 Resource Timing

```javascript
function getSlowResources(threshold = 1000) {
  const resources = performance.getEntriesByType('resource')
  return resources
    .filter(r => r.duration > threshold)
    .map(r => ({
      name: r.name,
      type: r.initiatorType,
      duration: Math.round(r.duration),
      transferSize: r.transferSize,
      startTime: Math.round(r.startTime)
    }))
}
```

### 3.3 PerformanceObserver 采集核心指标

```javascript
function observeWebVitals(report) {
  // LCP
  new PerformanceObserver((list) => {
    const entries = list.getEntries()
    const last = entries[entries.length - 1]
    report('LCP', last.startTime)
  }).observe({ type: 'largest-contentful-paint', buffered: true })

  // FCP
  new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      report('FCP', entry.startTime)
    }
  }).observe({ type: 'paint', buffered: true })

  // CLS
  let clsValue = 0
  new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      if (!entry.hadRecentInput) {
        clsValue += entry.value
      }
    }
    report('CLS', clsValue)
  }).observe({ type: 'layout-shift', buffered: true })

  // INP
  let inpValue = 0
  new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      if (entry.interactionId && entry.duration > inpValue) {
        inpValue = entry.duration
      }
    }
    report('INP', inpValue)
  }).observe({ type: 'event', buffered: true, durationThreshold: 0 })

  // Long Tasks
  new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      report('LongTask', entry.duration)
    }
  }).observe({ type: 'longtask', buffered: true })
}
```

## 四、自定义性能 SDK 设计

```javascript
class PerformanceTracker {
  constructor(options = {}) {
    this.reportUrl = options.reportUrl || '/api/performance'
    this.sampleRate = options.sampleRate ?? 1.0
    this.queue = []
    this.timer = null
    this.flushInterval = options.flushInterval ?? 10000

    this.init()
  }

  init() {
    this._observeWebVitals()
    this._observeLongTasks()
    this._observeResources()
    this._observeNavigation()
    this._measureFPS()
    this._startFlushTimer()
  }

  report(type, value, extra = {}) {
    if (Math.random() > this.sampleRate) return

    this.queue.push({
      type,
      value,
      url: location.href,
      userAgent: navigator.userAgent,
      timestamp: Date.now(),
      ...extra
    })
  }

  mark(name) {
    performance.mark(name)
  }

  measure(name, startMark, endMark) {
    performance.measure(name, startMark, endMark)
    const [entry] = performance.getEntriesByName(name, 'measure')
    if (entry) {
      this.report('custom', entry.duration, { name })
    }
  }

  async flush() {
    if (!this.queue.length) return
    const batch = this.queue.splice(0, this.queue.length)

    try {
      if (navigator.sendBeacon) {
        navigator.sendBeacon(this.reportUrl, JSON.stringify(batch))
      } else {
        await fetch(this.reportUrl, {
          method: 'POST',
          body: JSON.stringify(batch),
          headers: { 'Content-Type': 'application/json' },
          keepalive: true
        })
      }
    } catch (e) {
      console.warn('性能数据上报失败', e)
    }
  }

  _observeWebVitals() {
    // LCP
    new PerformanceObserver((list) => {
      const entries = list.getEntries()
      const last = entries[entries.length - 1]
      this.report('LCP', last.startTime)
    }).observe({ type: 'largest-contentful-paint', buffered: true })

    // CLS
    let clsValue = 0
    new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (!entry.hadRecentInput) clsValue += entry.value
      }
      this.report('CLS', clsValue)
    }).observe({ type: 'layout-shift', buffered: true })

    // INP
    let inpValue = 0
    new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (entry.interactionId && entry.duration > inpValue) {
          inpValue = entry.duration
        }
      }
      this.report('INP', inpValue)
    }).observe({ type: 'event', buffered: true, durationThreshold: 0 })

    // FCP
    new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (entry.name === 'first-contentful-paint') {
          this.report('FCP', entry.startTime)
        }
      }
    }).observe({ type: 'paint', buffered: true })
  }

  _observeLongTasks() {
    if (!('PerformanceLongTaskTiming' in window)) return

    new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        this.report('LongTask', entry.duration, {
          attribution: entry.attribution?.map(a => a.name).join(',')
        })
      }
    }).observe({ type: 'longtask', buffered: true })
  }

  _observeResources() {
    new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (entry.duration > 1000) {
          this.report('SlowResource', entry.duration, {
            name: entry.name,
            initiatorType: entry.initiatorType
          })
        }
      }
    }).observe({ type: 'resource', buffered: true })
  }

  _observeNavigation() {
    window.addEventListener('load', () => {
      setTimeout(() => {
        const [nav] = performance.getEntriesByType('navigation')
        if (!nav) return

        this.report('TTFB', nav.responseStart - nav.startTime)
        this.report('DCL', nav.domContentLoadedEventEnd - nav.startTime)
        this.report('Load', nav.loadEventEnd - nav.startTime)
      }, 0)
    })
  }

  _measureFPS() {
    let frameCount = 0
    let lastTime = performance.now()

    const loop = () => {
      frameCount++
      const now = performance.now()
      if (now - lastTime >= 1000) {
        const fps = frameCount
        frameCount = 0
        lastTime = now

        if (fps < 30) {
          this.report('FPS_LOW', fps)
        }
      }
      requestAnimationFrame(loop)
    }

    requestAnimationFrame(loop)
  }

  _startFlushTimer() {
    this.timer = setInterval(() => this.flush(), this.flushInterval)
  }

  destroy() {
    clearInterval(this.timer)
  }
}

const tracker = new PerformanceTracker({
  reportUrl: '/api/performance',
  sampleRate: 1.0,
  flushInterval: 10000
})

export default tracker
```

```javascript
window.addEventListener('load', () => {
```

### 场景 2：SPA 路由切换性能

```javascript
// router.js
let routeStartTime = 0

router.beforeEach((to, from) => {
  routeStartTime = performance.now()
  performance.mark('route-start')
})

router.afterEach((to, from) => {
  requestAnimationFrame(() => {
    requestAnimationFrame(() => {
      const duration = performance.now() - routeStartTime
      tracker.report('route-change', duration, {
        from: from.path,
        to: to.path
      })
    })
  })
})
```

### 场景 3：关键 API 耗时监控

```javascript
async function fetchUserProfile(userId) {
  performance.mark('api-user-start')

  try {
    const res = await fetch(`/api/users/${userId}`)
    const data = await res.json()

    performance.mark('api-user-end')
    performance.measure('api-user', 'api-user-start', 'api-user-end')

    const [measure] = performance.getEntriesByName('api-user', 'measure')
    tracker.report('api-duration', measure.duration, {
      api: '/api/users/:id',
      status: res.status
    })

    return data
  } catch (error) {
    tracker.report('api-error', 0, {
      api: '/api/users/:id',
      message: error.message
    })
    throw error
  }
}
```

### 场景 5：图片懒加载效果验证

```javascript
const imageObserver = new IntersectionObserver((entries) => {
  for (const entry of entries) {
    if (entry.isIntersecting) {
      const img = entry.target
      const src = img.dataset.src

      performance.mark(`img-load-start-${src}`)

      img.onload = () => {
        performance.mark(`img-load-end-${src}`)
        performance.measure(`img-load-${src}`, `img-load-start-${src}`, `img-load-end-${src}`)
        const [measure] = performance.getEntriesByName(`img-load-${src}`, 'measure')
        tracker.report('img-load', measure.duration, { src })
      }

      img.src = src
      imageObserver.unobserve(img)
    }
  }
})
```

### 场景 6：内存增长监控（大屏 / 长时间运行）

```javascript
function watchMemory() {
  if (!performance.memory) return

  setInterval(() => {
    const { usedJSHeapSize, totalJSHeapSize, jsHeapSizeLimit } = performance.memory
    const usedMB = usedJSHeapSize / 1024 / 1024
    const totalMB = totalJSHeapSize / 1024 / 1024

    tracker.report('memory', usedMB, {
      usedMB: usedMB.toFixed(2),
      totalMB: totalMB.toFixed(2),
      limitMB: (jsHeapSizeLimit / 1024 / 1024).toFixed(2)
    })

    if (usedMB / totalMB > 0.85) {
      tracker.report('memory-warning', usedMB)
    }
  }, 30000)
}

watchMemory()
```

### 场景 7：FPS 实时监控

```javascript
function monitorFPS() {
  let frameCount = 0
  let lastTime = performance.now()

  function tick() {
    frameCount++
    const now = performance.now()

    if (now - lastTime >= 1000) {
      const fps = frameCount

        tracker.report('fps-low', fps)
      }
    }

    requestAnimationFrame(tick)
  }

}
```

### 6.2 聚合与分位值

原始性能数据波动大，建议服务端按页面、设备、网络类型聚合，输出 **P50 / P75 / P90 / P95 / P99**。

| 指标 | 关注分位 | 说明 |
|------|---------|------|
| LCP | P75 | 75% 用户达到该值 |
| INP | P95 | 交互体验看尾部用户 |
| CLS | P75 | 布局偏移看大部分用户 |

### 6.3 告警阈值建议

```javascript
const thresholds = {
  LCP: { warning: 2500, critical: 4000 },
  INP: { warning: 200, critical: 500 },
  CLS: { warning: 0.1, critical: 0.25 },
  FPS: { warning: 30, critical: 20 },
  LongTask: { warning: 100, critical: 200 }
}
```

## 七、最佳实践与自检清单

| 实践 | 说明 |
|------|------|
| 优先关注 Web Vitals | LCP / INP / CLS 是 Google 排名的核心体验指标 |
| 采样 + 批量上报 | 高流量场景避免压垮服务端 |
| 使用 `web-vitals` 库 | 兼容性好，自动处理复杂场景 |
| 结合实验室数据 | Lighthouse + CI 防止回归 |
| 关注 P75 / P95 | 平均值会掩盖尾部用户的糟糕体验 |
| 设备分级 | 低端机、弱网环境单独看指标 |
| 版本对比 | 每次发版后对比核心指标变化 |
| 自定义业务指标 | 如首屏可交互时间、关键 API 耗时 |
