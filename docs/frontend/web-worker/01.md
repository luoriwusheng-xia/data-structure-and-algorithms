## 一、为什么需要 Web Worker

JavaScript 在浏览器中运行在单线程模型下，任何耗时操作都会阻塞主线程，导致页面卡顿、动画掉帧、交互无响应。

Web Worker 提供了一套在后台线程运行脚本的机制，核心收益是：

- **不阻塞 UI**：复杂计算、大数据处理、音视频编解码等任务放到 Worker 中执行。
- **利用多核 CPU**：现代设备多为多核，Worker 可以并行处理任务。
- **提升首屏体验**：把非关键初始化逻辑后置到 Worker，减少主线程负担。

**适用场景**：

| 场景 | 典型问题 | Worker 方案 |
|------|---------|------------|
| 大文件解析 | CSV/Excel/JSON 解析阻塞输入响应 | Worker 中解析，分批返回 |
| 图像处理 | 滤镜、压缩、水印导致动画卡顿 | 在 Worker 中操作 `ImageBitmap` / `OffscreenCanvas` |
| 复杂计算 | 路径规划、排序、聚类、密码学 | 多 Worker 并行，利用多核 |
| 日志埋点 | 大批量日志序列化与压缩占用主线程 | Worker 聚合、压缩、批量上报 |
| 音视频处理 | 解码、转码、波形分析 | Worker + WebAssembly / WebCodec |
| 离屏 Canvas 渲染 | 复杂可视化每帧重绘掉帧 | `transferControlToOffscreen` 交给 Worker |

**不适用场景**：

- 需要直接访问 DOM 的操作（Worker 中没有 `window` / `document`）。
- 大量小任务频繁通信（序列化/反序列化开销可能得不偿失）。
- 对实时性要求极高且数据量极小的计算。

## 二、核心 API 与生命周期

### 2.1 创建 Worker

```javascript
// 同源策略：worker.js 必须与当前页面同源
const worker = new Worker('./worker.js')
```

Worker 脚本有独立的运行时环境：

- 全局对象是 `self`（不是 `window`）。
- 没有 DOM、BOM（没有 `document`、`window`、`alert`）。
- 可以使用 `fetch`、`WebSocket`、`IndexedDB`、`setTimeout`、`setInterval` 等。
- 可以加载其他脚本：`importScripts('a.js', 'b.js')`（经典 Worker）或通过 `type: 'module'` 使用 ES Module。

### 2.2 使用 ES Module Worker（推荐）

```javascript
// 主线程
const worker = new Worker('./worker.js', { type: 'module' })
```

```javascript
// worker.js
import { heavyCompute } from './compute.js'

self.onmessage = async (event) => {
  const result = heavyCompute(event.data)
  self.postMessage(result)
}
```

> 注意：ES Module Worker 的兼容性正在提升，旧版浏览器需回退到 `importScripts`。

### 2.3 终止 Worker

```javascript
// 主线程主动终止
worker.terminate()

// Worker 内部主动关闭
self.close()
```

- `terminate()` 会立即终止，不会等待正在执行的任务完成。
- Worker 中抛出的未捕获错误不会阻塞主线程，但会通过 `worker.onerror` 上报。

### 2.4 错误处理

```javascript
worker.onerror = (error) => {
  console.error('Worker error:', error.message, error.filename, error.lineno)
}

worker.addEventListener('error', (error) => {
  console.error('Worker error event:', error)
})
```

## 三、主线程与 Worker 的通信

### 3.1 基础消息通信

```javascript
// main.js
const worker = new Worker('./worker.js')

worker.onmessage = (event) => {
  console.log('收到 Worker 结果:', event.data)
}

worker.postMessage({ type: 'COMPUTE', payload: { n: 35 } })
```

```javascript
// worker.js
self.onmessage = (event) => {
  const { type, payload } = event.data

  if (type === 'COMPUTE') {
    const result = fibonacci(payload.n)
    self.postMessage({ type: 'COMPUTE_RESULT', payload: result })
  }
}

function fibonacci(n) {
  return n <= 1 ? n : fibonacci(n - 1) + fibonacci(n - 2)
}
```

### 3.2 Transferable Objects：零拷贝传输

默认情况下，`postMessage` 会结构化克隆数据，对 `ArrayBuffer`、`ImageBitmap`、`OffscreenCanvas` 等大数据拷贝开销很高。

Transferable Objects 可以把所有权从发送方转移到接收方，**实现零拷贝**。

```javascript
// 主线程创建 buffer
const buffer = new ArrayBuffer(1024 * 1024 * 10) // 10MB

worker.postMessage({ buffer }, [buffer])

// transfer 后，主线程不能再访问 buffer
console.log(buffer.byteLength) // 0
```

```javascript
// worker.js
self.onmessage = (event) => {
  const { buffer } = event.data
  // Worker 现在拥有这块内存
  const view = new Uint8Array(buffer)
  // 处理完后再 transfer 回去
  self.postMessage({ buffer }, [buffer])
}
```

常见 Transferable 类型：

- `ArrayBuffer`
- `MessagePort`
- `ImageBitmap`
- `OffscreenCanvas`
- `ReadableStream`、`WritableStream`、`TransformStream`

### 3.3 MessageChannel：双向独立通道

当主线程需要与 Worker 建立多条独立通信线路时，可以使用 `MessageChannel`。

```javascript
const worker = new Worker('./worker.js')
const channel = new MessageChannel()

// 把 port2 交给 Worker
worker.postMessage({ port: channel.port2 }, [channel.port2])

// 主线程通过 port1 通信
channel.port1.onmessage = (event) => {
  console.log('通过 channel 收到:', event.data)
}
channel.port1.postMessage('hello from main')
```

```javascript
// worker.js
self.onmessage = (event) => {
  const { port } = event.data
  port.onmessage = (e) => {
    port.postMessage(`echo: ${e.data}`)
  }
}
```

### 3.4 封装一个 Promise 化的 Worker 调用

```javascript
class WorkerPool {
  constructor(scriptUrl, poolSize = navigator.hardwareConcurrency || 4) {
    this.scriptUrl = scriptUrl
    this.poolSize = poolSize
    this.workers = []
    this.queue = []
    this.taskId = 0

    for (let i = 0; i < poolSize; i++) {
      this.workers.push(this.createWorker())
    }
  }

  createWorker() {
    const worker = new Worker(this.scriptUrl)
    worker.busy = false
    worker.onmessage = (event) => {
      const { id, result, error } = event.data
      const task = worker.currentTask
      if (task) {
        if (error) {
          task.reject(new Error(error))
        } else {
          task.resolve(result)
        }
        worker.currentTask = null
        worker.busy = false
        this.processQueue()
      }
    }
    return worker
  }

  execute(payload) {
    return new Promise((resolve, reject) => {
      this.queue.push({ id: ++this.taskId, payload, resolve, reject })
      this.processQueue()
    })
  }

  processQueue() {
    if (this.queue.length === 0) return
    const worker = this.workers.find((w) => !w.busy)
    if (!worker) return

    const task = this.queue.shift()
    worker.busy = true
    worker.currentTask = task
    worker.postMessage({ id: task.id, payload: task.payload })
  }

  terminate() {
    this.workers.forEach((w) => w.terminate())
    this.workers = []
  }
}

// 使用
const pool = new WorkerPool('./compute-worker.js', 4)
const result = await pool.execute({ n: 40 })
```

## 四、典型实战场景

### 4.1 场景一：大文件分片解析

业务场景：上传前需要解析 100MB 级别的 CSV/Excel，计算行数、列校验、预览前 100 行。

```javascript
// main.js
const input = document.getElementById('fileInput')
input.addEventListener('change', async (e) => {
  const file = e.target.files[0]
  const worker = new Worker('./csv-worker.js')

  worker.onmessage = (event) => {
    const { type, payload } = event.data
    if (type === 'PROGRESS') {
      updateProgress(payload.percent)
    }
    if (type === 'DONE') {
      renderPreview(payload.preview)
    }
  }

  // 使用 File 的 slice 分片读取
  const chunkSize = 1024 * 1024 // 1MB
  let offset = 0
  const reader = file.stream().getReader()

  while (true) {
    const { done, value } = await reader.read()
    if (done) {
      worker.postMessage({ type: 'END' })
      break
    }
    worker.postMessage({ type: 'CHUNK', payload: value.buffer }, [value.buffer])
  }
})
```

```javascript
// csv-worker.js
let textDecoder = new TextDecoder('utf-8')
let buffer = ''
let totalRows = 0
let preview = []

self.onmessage = (event) => {
  const { type, payload } = event.data

  if (type === 'CHUNK') {
    const chunk = textDecoder.decode(payload, { stream: true })
    buffer += chunk

    // 按行切分，保留最后一行不完整部分
    const lines = buffer.split('\n')
    buffer = lines.pop()

    lines.forEach((line) => {
      totalRows++
      if (preview.length < 100) {
        preview.push(line.split(','))
      }
    })
  }

  if (type === 'END') {
    if (buffer) {
      totalRows++
      preview.push(buffer.split(','))
    }
    self.postMessage({ type: 'DONE', payload: { totalRows, preview } })
    self.close()
  }
}
```

### 4.2 场景二：图像滤镜与压缩

业务场景：用户上传图片后前端生成缩略图、加水印、应用滤镜，再上传。

```javascript
// main.js
async function processImage(file) {
  const bitmap = await createImageBitmap(file)
  const worker = new Worker('./image-worker.js')

  const canvas = new OffscreenCanvas(bitmap.width, bitmap.height)
  const ctx = canvas.getContext('2d')
  ctx.drawImage(bitmap, 0, 0)

  const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height)

  return new Promise((resolve) => {
    worker.onmessage = (event) => {
      const { buffer, width, height } = event.data
      // 把处理后的 ImageData 转回 Blob
      const resultCanvas = new OffscreenCanvas(width, height)
      const resultCtx = resultCanvas.getContext('2d')
      const resultData = new ImageData(new Uint8ClampedArray(buffer), width, height)
      resultCtx.putImageData(resultData, 0, 0)
      resultCanvas.convertToBlob({ type: 'image/jpeg', quality: 0.8 }).then(resolve)
      worker.terminate()
    }

    worker.postMessage(
      { buffer: imageData.data.buffer, width, height, filter: 'grayscale' },
      [imageData.data.buffer]
    )
  })
}
```

```javascript
// image-worker.js
self.onmessage = (event) => {
  const { buffer, width, height, filter } = event.data
  const data = new Uint8ClampedArray(buffer)

  if (filter === 'grayscale') {
    for (let i = 0; i < data.length; i += 4) {
      const gray = 0.299 * data[i] + 0.587 * data[i + 1] + 0.114 * data[i + 2]
      data[i] = data[i + 1] = data[i + 2] = gray
    }
  }

  self.postMessage({ buffer: data.buffer, width, height }, [data.buffer])
}
```

### 4.3 场景三：离屏 Canvas 渲染

业务场景：数据大屏、复杂 2D 可视化，主线程需要保持 60fps 响应交互。

```javascript
// main.js
const canvas = document.getElementById('canvas')
const offscreen = canvas.transferControlToOffscreen()
const worker = new Worker('./render-worker.js')

worker.postMessage({ canvas: offscreen }, [offscreen])

// 后续向 Worker 发送数据和交互事件
worker.postMessage({ type: 'DATA', payload: dataPoints })
```

```javascript
// render-worker.js
let canvas = null
let ctx = null
let data = []

self.onmessage = (event) => {
  const { type, payload } = event.data

  if (event.data.canvas) {
    canvas = event.data.canvas
    ctx = canvas.getContext('2d')
    startRenderLoop()
  }

  if (type === 'DATA') {
    data = payload
  }
}

function startRenderLoop() {
  function render() {
    ctx.clearRect(0, 0, canvas.width, canvas.height)

    // 在 Worker 中执行重绘制，不阻塞主线程
    data.forEach((point) => {
      ctx.beginPath()
      ctx.arc(point.x, point.y, point.r, 0, Math.PI * 2)
      ctx.fill()
    })

    requestAnimationFrame(render)
  }
  render()
}
```

> 详情可参考 [Canvas 进阶实战](/docs/frontend/canvas/01) 中“离屏渲染”章节。

### 4.4 场景四：复杂数据计算并行化

业务场景：地图热力图、聚类分析、大规模排序、遗传算法、蒙特卡洛模拟。

```javascript
// main.js
const data = new Float64Array(10_000_000).map(() => Math.random())

const workerCount = navigator.hardwareConcurrency || 4
const chunkSize = Math.ceil(data.length / workerCount)
const promises = []

for (let i = 0; i < workerCount; i++) {
  const start = i * chunkSize
  const end = Math.min(start + chunkSize, data.length)
  const chunk = data.slice(start, end)

  promises.push(
    new Promise((resolve) => {
      const worker = new Worker('./stats-worker.js')
      worker.onmessage = (event) => resolve(event.data)
      worker.postMessage({ buffer: chunk.buffer }, [chunk.buffer])
    })
  )
}

const results = await Promise.all(promises)
const totalSum = results.reduce((sum, r) => sum + r.sum, 0)
const totalCount = results.reduce((sum, r) => sum + r.count, 0)
const average = totalSum / totalCount
```

```javascript
// stats-worker.js
self.onmessage = (event) => {
  const data = new Float64Array(event.data.buffer)
  let sum = 0
  for (let i = 0; i < data.length; i++) {
    sum += data[i]
  }
  self.postMessage({ sum, count: data.length }, [data.buffer])
}
```

### 4.5 场景五：日志聚合与批量上报

业务场景：前端产生大量埋点日志，需要在合适的时机压缩、批量上报，避免阻塞业务代码。

```javascript
// main.js
const logWorker = new Worker('./log-worker.js')

logWorker.onmessage = (event) => {
  const { type, payload } = event.data
  if (type === 'FLUSH') {
    navigator.sendBeacon('/api/log', payload)
  }
}

// 业务代码中直接调用
function track(event) {
  logWorker.postMessage({ type: 'LOG', payload: event })
}

// 页面卸载前触发刷盘
window.addEventListener('beforeunload', () => {
  logWorker.postMessage({ type: 'FLUSH' })
})
```

```javascript
// log-worker.js
let buffer = []
const MAX_SIZE = 50
const FLUSH_INTERVAL = 5000

self.onmessage = (event) => {
  const { type, payload } = event.data

  if (type === 'LOG') {
    buffer.push({ ...payload, ts: Date.now() })
    if (buffer.length >= MAX_SIZE) {
      flush()
    }
  }

  if (type === 'FLUSH') {
    flush()
  }
}

function flush() {
  if (buffer.length === 0) return
  const batch = buffer.splice(0)
  const compressed = JSON.stringify(batch)
  self.postMessage({ type: 'FLUSH', payload: compressed })
}

setInterval(flush, FLUSH_INTERVAL)
```

## 五、Shared Worker 与 Service Worker 的区别

很多同学会把 Worker 家族搞混，这里做一个明确对比：

| 特性 | Web Worker | Shared Worker | Service Worker |
|------|-----------|---------------|----------------|
| 运行线程 | 独立后台线程 | 独立后台线程 | 独立后台线程 |
| 作用范围 | 单个页面/iframe | 同源的多个页面共享 | 同源的所有页面/请求 |
| 生命周期 | 随页面关闭 | 最后一个连接关闭 | 事件驱动，可长期驻留 |
| 主要用途 | 复杂计算 | 跨页通信、共享状态 | 离线缓存、推送、拦截请求 |
| 通信方式 | `postMessage` | `port.postMessage` | `fetch` 事件、`postMessage` |
| 访问 DOM | 否 | 否 | 否 |

### 5.1 Shared Worker 示例

```javascript
// shared-worker.js
const connections = new Set()

self.onconnect = (event) => {
  const port = event.ports[0]
  connections.add(port)

  port.onmessage = (e) => {
    // 广播给所有连接
    connections.forEach((conn) => {
      if (conn !== port) {
        conn.postMessage(e.data)
      }
    })
  }

  port.start()
}
```

```javascript
// main.js
const worker = new SharedWorker('./shared-worker.js')
worker.port.start()

worker.port.postMessage({ from: 'page-A', text: 'hello' })
worker.port.onmessage = (event) => {
  console.log('收到广播:', event.data)
}
```

## 六、最佳实践与常见坑

### 6.1 通信数据要尽量小

`postMessage` 会触发结构化克隆，大数据对象会消耗 CPU 和内存。优先使用：

- `Transferable Objects` 传输 `ArrayBuffer`。
- `ImageBitmap` 代替 `ImageData` 传输图像。
- 分片处理，避免一次性塞入整个大文件。

### 6.2 Worker 不是越多越好

每个 Worker 都会占用内存和线程资源。建议：

- 计算密集型任务使用 `navigator.hardwareConcurrency` 控制并发数。
- 对于一次性小任务，任务结束后立即 `terminate()`。
- 复用 Worker 时使用 Worker Pool，避免频繁创建销毁。

### 6.3 注意 Worker 中的模块加载

经典 Worker 使用 `importScripts`：

```javascript
importScripts('https://cdn.example.com/lodash.min.js', './utils.js')
```

ES Module Worker 使用 `import`：

```javascript
import { debounce } from './utils.js'
```

混用可能导致报错，建议新项目直接启用 `{ type: 'module' }`。

### 6.4 错误隔离

Worker 内部抛出的错误不会导致主线程崩溃，但一定要监听：

```javascript
worker.onerror = (error) => {
  console.error('Worker failed:', error)
}
```

### 6.5 Worker 中的内存泄漏

- 长期运行的 Worker 要定期清理缓存和事件监听器。
- 使用 `WeakMap` / `WeakRef` 管理临时对象。
- 页面卸载前 terminate 不再需要的 Worker。

### 6.6 调试离屏 Canvas 时的常见问题

- `transferControlToOffscreen()` 后主线程不能再调用 `getContext`。
- Worker 中 `canvas.width / height` 需要手动维护，不会随 CSS 尺寸自动变化。
- 某些 Canvas API（如 `measureText`、部分合成模式）在 Worker 中行为可能与主线程有差异。

## 七、调试技巧

### 7.1 Chrome DevTools

- 打开 `Sources` → `Threads`，可以看到所有 Worker 线程。
- 在 Worker 脚本中直接打断点，支持单步调试。
- `Console` 面板可以切换上下文到指定 Worker。

### 7.2 性能分析

- 使用 `Performance` 面板录制主线程，确认耗时操作已被移走。
- 在 Worker 中可以通过 `performance.now()` 测量单次任务耗时。

### 7.3 日志定位

在 Worker 中 `console.log` 会输出到主线程 DevTools 控制台，但上下文不同，建议：

```javascript
self.postMessage({ type: 'LOG', payload: message })
```

## 八、总结

Web Worker 是前端性能优化的核心工具之一：

- **会创建**：使用 `new Worker('./worker.js', { type: 'module' })`。
- **会通信**：掌握 `postMessage`、Transferable Objects、`MessageChannel`。
- **会实战**：大文件解析、图像处理、离屏 Canvas、并行计算、日志聚合。
- **会管理**：Worker Pool、错误处理、内存释放、性能调试。

> 当你发现主线程被复杂计算或数据处理拖慢时，第一反应应该是：这件事能不能放到 Web Worker 里做？
