# knip

## 一、项目概况

Knip 是一个用于清理 JavaScript 和 TypeScript 项目的开源静态分析工具，旨在帮助开发者发现并移除项目中未使用的文件、依赖和导出。它的核心理念是"Knip it before you ship it"（发布前先剪掉冗余），通过减少代码和依赖来提升性能、降低维护成本，并让重构更加轻松。

- **项目地址**：[github.com/webpro-nl/knip](https://github.com/webpro-nl/knip)
- **官方网站**：[knip.dev](https://knip.dev)
- **开源协议**：ISC
- **npm 包**：`knip`
- **VS Code 扩展**：可在 VS Code Marketplace 或 Open VSX Registry 获取

## 二、主要功能

Knip 能够检测以下问题：

- **未使用的文件**：项目中存在但未被任何地方引用的文件。
- **未使用的依赖**：`package.json` 中已声明但代码中从未使用的 npm 包（包括 `dependencies` 和 `devDependencies`）。
- **未列出的依赖**：代码中引用了但在 `package.json` 中未声明的依赖。
- **未使用的导出**：导出的函数、类、变量、类型等未被其他文件使用。
- **未使用的枚举成员、类成员**。
- **重复导出**。
- **配置优化提示**：帮助你收紧项目配置。

## 三、Node 版本要求

Knip v6（当前最新版本）需要 Node.js `v20.19.0` 或更高版本，或者使用 Bun 运行时。

- 早期版本（如 Knip v5）支持 Node.js v18，但 v6 已明确放弃对 Node.js v18 的支持。
- 旧版 Knip v3 曾要求 Node.js v18.6 或更高版本。

> ⚠️ **注意**：如果你仍在使用 Node.js v18，请务必使用 Knip v5（例如 `npm install knip@5`）；若需使用最新功能，请升级至 Node.js v20.19.0+。

## 四、基础使用

### 方式一：使用官方初始化脚本（推荐）

```bash
npm init @knip.config
```

该命令会：

1. 自动安装 Knip 到项目的开发依赖中；
2. 在 `package.json` 中添加运行脚本。

配置完成后，运行以下命令即可开始分析：

```bash
npm run knip
```

### 方式二：手动安装与运行

```bash
npm install -D knip
npx knip
```

也可以使用 yarn、pnpm 等包管理器：

```bash
# yarn
yarn add -D knip
yarn knip

# pnpm
pnpm add -D knip
pnpm run knip
```

或者直接通过 npx / pnpm dlx / bunx 临时运行（无需安装）：

```bash
npx knip
pnpm dlx knip
bunx knip
```

> 📌 Knip 兼容 yarn、pnpm 等主流包管理器，并原生支持 monorepo 结构。

## 五、核心优势与生态支持

- **广泛的插件生态**：Knip 内置了约 150 个插件，能够识别各种流行框架和工具的配置文件，如 Astro、Next.js、Remix、Svelte、Jest、Vitest、Cypress、ESLint、Webpack、Vite、GitHub Actions、Nx、Storybook 等。
- **静态分析与精确结果**：通过从项目的细粒度入口点构建依赖图进行分析，有效减少误报。
- **集成支持**：
  - **VS Code 扩展**：在编辑器中实时查看未使用代码；
  - **语言服务器**（`@knip/language-server`）：便于编辑器集成；
  - **MCP 服务器**（`@knip/mcp`）：可供 AI 编码代理调用。
- **丰富的输出格式**：支持默认、JSON、Markdown、JUnit 等多种报告格式，便于 CI 集成。
- **可自动修复**：支持移除未使用的导出和依赖。
- **在线体验**：可在 [Knip Playground](https://knip.dev) 中在线试用，无需安装。
