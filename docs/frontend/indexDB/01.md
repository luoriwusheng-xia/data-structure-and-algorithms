# IndexedDB 基础使用指南

> 本文面向有前端开发经验的工程师，系统梳理 IndexedDB 的核心概念、API 设计与基础用法。

---

## 一、什么是 IndexedDB

IndexedDB 是浏览器内置的**结构化持久化存储方案**，基于键值对存储，支持索引查询和事务操作。它是 Web Storage（localStorage / sessionStorage）的上位替代，适合存储大量结构化数据。

### 核心特征

| 特征 | 说明 |
|------|------|
| 存储容量 | 通常 50MB 起步，远超 localStorage 的 5MB |
| 数据类型 | 支持任意结构化克隆类型（对象、数组、Blob、File、ArrayBuffer 等）|
| 查询能力 | 支持索引、范围查询、游标遍历 |
| 事务支持 | 所有读写操作必须通过事务完成 |
| 异步 API | 基于事件回调，不阻塞主线程 |
| 同源策略 | 严格遵守同源限制 |

### 与其他存储方案的对比

| 特性 | Cookie | localStorage | sessionStorage | IndexedDB |
|------|--------|-------------|----------------|-----------|
| 容量 | ~4KB | ~5MB | ~5MB | 50MB+ |
| 数据类型 | 字符串 | 字符串 | 字符串 | 结构化数据 |
| 索引查询 | ❌ | ❌ | ❌ | ✅ |
| 事务支持 | ❌ | ❌ | ❌ | ✅ |
| 异步 API | ❌ | ❌ | ❌ | ✅ |
| 同源限制 | ✅ | ✅ | ✅ | ✅ |
| 服务端可读写 | ✅ | ❌ | ❌ | ❌ |
| 持久化 | 可配置 | 长期 | 会话级 | 长期 |

## 二、核心概念

### 2.1 数据库（Database）

一个域名下可创建多个数据库，通过 `indexedDB.open(name, version)` 打开。

```javascript
// 打开名为 "myApp" 的数据库，版本号为 1
const request = indexedDB.open('myApp', 1);

request.onsuccess = (event) => {
  const db = event.target.result;
  console.log('数据库打开成功', db.name, db.version);
};

request.onerror = (event) => {
  console.error('数据库打开失败', event.target.error);
};

// version 升级时触发（首次打开也会触发）
request.onupgradeneeded = (event) => {
  const db = event.target.result;
  console.log('数据库升级中...');
};
```

> **注意**：版本号必须是**大于 0 的整数**，只能递增不能回退。

### 2.2 对象仓库（Object Store）

相当于关系型数据库中的**表**，用于存储同类型数据。只能在 `onupgradeneeded` 中创建。

```javascript
request.onupgradeneeded = (event) => {
  const db = event.target.result;

  // 创建对象仓库，指定主键
  if (!db.objectStoreNames.contains('users')) {
    const store = db.createObjectStore('users', {
      keyPath: 'id',        // 主键字段名
      autoIncrement: true   // 自增主键
    });

    // 创建索引
    store.createIndex('name', 'name', { unique: false });
    store.createIndex('email', 'email', { unique: true });
  }
};
```

| 参数 | 说明 |
|------|------|
| `keyPath` | 指定对象中的哪个字段作为主键 |
| `autoIncrement` | 主键是否自增 |

**不指定 keyPath，使用外部键**：

```javascript
// 使用外部传入的键
const store = db.createObjectStore('logs', { autoIncrement: true });

// 添加数据时手动指定键
store.add({ message: '系统启动' }, 'log-001');
```

### 2.3 索引（Index）

索引用于加速非主键字段的查询。同样在 `onupgradeneeded` 中创建。

```javascript
store.createIndex('age', 'age', { unique: false });      // 普通索引
store.createIndex('email', 'email', { unique: true });    // 唯一索引
```

### 2.4 事务（Transaction）

#### 核心概念

事务（Transaction）是 IndexedDB 执行数据操作的**基本单元**。所有对对象仓库的读写都必须通过事务完成，其核心特性包括：

| 特性 | 说明 |
|------|------|
| 原子性 | 事务中的所有请求要么全部成功提交，要么任一请求失败时整体回滚 |
| 隔离性 | 不同事务之间相互隔离，读写操作不会互相干扰 |
| 自动提交 | 当事务内所有请求执行完毕且事件循环空闲时，事务会自动提交 |
| 同源性 | 事务只能在打开它的数据库实例上执行，不能跨数据库 |

#### 三种事务模式

IndexedDB 为事务定义了三种模式，创建事务时必须显式指定：

```javascript
// readonly：只读，可并发执行，性能最高
const ro = db.transaction('users', 'readonly');

// readwrite：读写，同一仓库同时只能存在一个读写事务
const rw = db.transaction('users', 'readwrite');

// versionchange：版本变更，仅在 onupgradeneeded 中由系统自动创建
const vc = db.transaction('users', 'versionchange');
```

| 模式 | 用途 | 并发能力 | 注意事项 |
|------|------|---------|---------|
| `readonly` | 查询数据 | 高，多个只读事务可并行 | 禁止写入，否则会报错 |
| `readwrite` | 增删改数据 | 低，同一仓库同时仅一个 | 会阻塞同仓库的其他读写事务 |
| `versionchange` | 修改数据库结构 | 独占 | 不允许手动创建，仅供升级使用 |

#### 基础用法

**单仓库事务**

```javascript
const transaction = db.transaction('users', 'readwrite');
const store = transaction.objectStore('users');

store.add({ id: 1, name: '张三', age: 25 });
store.put({ id: 2, name: '李四', age: 30 });
```

**多仓库事务**

传入仓库名称数组，即可在一个事务中操作多个仓库。任一仓库操作失败，整个事务都会回滚：

```javascript
const transaction = db.transaction(['orders', 'inventory'], 'readwrite');
const orderStore = transaction.objectStore('orders');
const inventoryStore = transaction.objectStore('inventory');

orderStore.add({ orderId: 'O001', total: 199 });
inventoryStore.put({ sku: 'SKU001', stock: 80 });
```

#### 事务生命周期

一个事务从创建到结束经历以下阶段：

```javascript
function runTransaction(db) {
  // 1. 创建事务
  const transaction = db.transaction('users', 'readwrite');
  const store = transaction.objectStore('users');

  // 2. 发起请求（可连续发起多个）
  store.add({ id: 1, name: 'Alice' });
  store.put({ id: 2, name: 'Bob' });

  // 3. 监听事务级事件
  transaction.oncomplete = () => {
    console.log('事务提交成功');
  };

  transaction.onerror = (event) => {
    console.error('事务失败，整体回滚:', event.target.error);
  };

  transaction.onabort = () => {
    console.warn('事务被中止');
  };
}
```

> 事务一旦创建，应尽快发起所有请求；当微任务队列清空、事件循环空闲时，事务会自动提交。

#### 错误处理和回滚

事务中的错误默认会向上冒泡，触发 `transaction.onerror` 并导致整个事务回滚。如果某个请求的错误已被单独处理且不希望影响其他请求，可调用 `event.preventDefault()` 阻止冒泡。

```javascript
const transaction = db.transaction('users', 'readwrite');
const store = transaction.objectStore('users');

const request1 = store.add({ id: 1, name: '张三' });
const request2 = store.add({ id: 1, name: '李四' }); // 主键冲突，会失败

request1.onsuccess = () => console.log('请求1成功');

request2.onerror = (event) => {
  // 阻止错误冒泡到事务，避免整体回滚
  event.preventDefault();
  console.warn('请求2失败，但不影响事务中的其他请求');
};

transaction.oncomplete = () => {
  console.log('事务完成，请求1的数据已提交');
};

transaction.onerror = (event) => {
  console.error('事务级错误:', event.target.error);
};
```

也可以手动中止事务：

```javascript
const transaction = db.transaction('users', 'readwrite');
// 当检测到业务条件不满足时，主动回滚
transaction.abort();
```

#### 实际应用场景

**场景 1：购物车结算（多仓库原子操作）**

结算时需要同时创建订单、扣减库存、清空购物车，这些操作必须作为一个原子单元：

```javascript
async function checkout(db, cartItems, userId) {
  return new Promise((resolve, reject) => {
    const transaction = db.transaction(
      ['orders', 'inventory', 'cart'],
      'readwrite'
    );

    const orderStore = transaction.objectStore('orders');
    const inventoryStore = transaction.objectStore('inventory');
    const cartStore = transaction.objectStore('cart');

    const orderId = crypto.randomUUID();
    const total = cartItems.reduce((sum, item) => sum + item.price * item.qty, 0);

    // 创建订单
    orderStore.add({
      orderId,
      userId,
      items: cartItems,
      total,
      createdAt: Date.now()
    });

    // 扣减库存
    for (const item of cartItems) {
      const getReq = inventoryStore.get(item.sku);
      getReq.onsuccess = () => {
        const record = getReq.result;
        if (!record || record.stock < item.qty) {
          transaction.abort();
          reject(new Error(`库存不足: ${item.sku}`));
          return;
        }
        inventoryStore.put({ ...record, stock: record.stock - item.qty });
      };
    }

    // 清空当前用户购物车
    cartStore.delete(userId);

    transaction.oncomplete = () => resolve(orderId);
    transaction.onerror = () => reject(transaction.error);
    transaction.onabort = () => reject(new Error('结算事务被中止'));
  });
}
```

**场景 2：游标遍历 + 批量操作**

处理大量数据时，使用游标逐条读取并在同一事务中批量更新，避免频繁创建事务：

```javascript
async function batchUpdateUsers(db) {
  return new Promise((resolve, reject) => {
    const transaction = db.transaction('users', 'readwrite');
    const store = transaction.objectStore('users');

    let count = 0;
    const cursorRequest = store.openCursor();

    cursorRequest.onsuccess = (event) => {
      const cursor = event.target.result;
      if (!cursor) {
        resolve(count);
        return;
      }

      const user = cursor.value;
      user.updatedAt = Date.now();
      cursor.update(user); // 在同一事务中更新
      count++;
      cursor.continue();
    };

    cursorRequest.onerror = () => reject(cursorRequest.error);
    transaction.onerror = () => reject(transaction.error);
  });
}
```

#### 性能最佳实践

| 实践 | 说明 |
|------|------|
| 合并请求 | 将多个操作放在同一个事务中，减少事务创建开销 |
| 优先只读 | 查询场景使用 `readonly` 模式，允许多个只读事务并发 |
| 避免长事务 | 长时间持有读写事务会阻塞其他操作，应尽快完成 |
| 批量写入 | 大量写入时，每批控制在 1000 条左右，分事务提交 |
| 减少范围 | 只打开需要操作的仓库，避免不必要的事务范围 |

```javascript
// ✅ 推荐：批量添加合并到一个事务
const transaction = db.transaction('logs', 'readwrite');
const store = transaction.objectStore('logs');
for (const log of logs) {
  store.add(log);
}

// ❌ 不推荐：每条数据单独创建事务
for (const log of logs) {
  const t = db.transaction('logs', 'readwrite');
  t.objectStore('logs').add(log);
}
```

#### 事务的限制

1. **不能嵌套事务**：IndexedDB 不支持在一个事务内部创建子事务。
2. **`versionchange` 独占**：数据库升级期间，其他页面无法同时访问该数据库。
3. **事务结束后不可复用**：事务提交或中止后，不能再向其添加请求。
4. **读写事务会互斥**：同一仓库的 `readwrite` 事务会串行执行，降低并发。
5. **请求必须同步发起**：事务创建后，所有请求应在同步代码中立即发起，避免事务自动提交。

#### 常见错误及解决

| 错误 | 原因 | 解决 |
|------|------|------|
| `TransactionInactiveError` | 事务已提交或中止后仍发起请求 | 在事务活跃期间同步发起所有请求 |
| `ReadOnlyError` | 在只读事务中执行写入 | 将模式改为 `readwrite` |
| `NotFoundError` | 访问了不存在的对象仓库 | 检查仓库名称拼写，确认在 `onupgradeneeded` 中已创建 |
| 事务意外回滚 | 某个请求失败且未阻止冒泡 | 对可预见的错误调用 `event.preventDefault()` |

#### 总结

- 事务是 IndexedDB 的**核心机制**，所有数据操作都围绕事务展开。
- 根据操作类型选择合适模式：查询用 `readonly`，写入用 `readwrite`。
- 多仓库事务可以保证跨仓库操作的原子性，非常适合购物车结算等场景。
- 合理合并请求、控制事务粒度，是提升 IndexedDB 性能的关键。

## 三、CRUD 操作

### 3.1 添加数据（add）

```javascript
function addUser(user) {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('myApp', 1);

    request.onsuccess = (event) => {
      const db = event.target.result;
      const transaction = db.transaction(['users'], 'readwrite');
      const store = transaction.objectStore('users');

      // add：仅当键不存在时才添加，键已存在则报错
      const addRequest = store.add(user);

      addRequest.onsuccess = () => {
        console.log('添加成功，主键:', addRequest.result);
        resolve(addRequest.result);
      };

      addRequest.onerror = () => {
        reject(addRequest.error);
      };
    };

    request.onerror = () => reject(request.error);
  });
}

// 使用
await addUser({ name: '张三', email: 'zhangsan@example.com', age: 25 });
```

### 3.2 读取数据（get）

```javascript
function getUser(id) {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('myApp', 1);

    request.onsuccess = (event) => {
      const db = event.target.result;
      const transaction = db.transaction(['users'], 'readonly');
      const store = transaction.objectStore('users');

      const getRequest = store.get(id);

      getRequest.onsuccess = () => {
        resolve(getRequest.result); // 找不到返回 undefined
      };

      getRequest.onerror = () => reject(getRequest.error);
    };
  });
}

// 通过索引查询
function getUserByEmail(email) {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('myApp', 1);

    request.onsuccess = (event) => {
      const db = event.target.result;
      const transaction = db.transaction(['users'], 'readonly');
      const store = transaction.objectStore('users');
      const index = store.index('email');

      const getRequest = index.get(email);
      getRequest.onsuccess = () => resolve(getRequest.result);
      getRequest.onerror = () => reject(getRequest.error);
    };
  });
}
```

## 四、范围查询与游标

### 4.1 范围对象（IDBKeyRange）

```javascript
// 精确匹配
const exact = IDBKeyRange.only(5);

// 小于 / 小于等于
const upper = IDBKeyRange.upperBound(10);       // <= 10
const upperOpen = IDBKeyRange.upperBound(10, true);  // < 10

// 大于 / 大于等于

// 范围（包含边界）
const range = IDBKeyRange.bound(5, 10);              // 5 <= x <= 10
const rangeOpen = IDBKeyRange.bound(5, 10, true, true); // 5 < x < 10
```

### 4.2 索引范围查询

```javascript
function getUsersByAgeRange(minAge, maxAge) {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('myApp', 1);

    request.onsuccess = (event) => {
      const db = event.target.result;
      const transaction = db.transaction(['users'], 'readonly');
      const store = transaction.objectStore('users');
      const ageIndex = store.index('age');

      const range = IDBKeyRange.bound(minAge, maxAge);
      const getAllRequest = ageIndex.getAll(range);

      getAllRequest.onsuccess = () => resolve(getAllRequest.result);
      getAllRequest.onerror = () => reject(getAllRequest.error);
    };
  });
}

// 获取 20-30 岁的用户
const users = await getUsersByAgeRange(20, 30);
```

### 4.3 游标遍历（Cursor）

当数据量较大时，使用游标分批读取，避免一次性加载过多数据。

```javascript
function iterateUsers(callback) {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('myApp', 1);

    request.onsuccess = (event) => {
      const db = event.target.result;
      const transaction = db.transaction(['users'], 'readonly');
      const store = transaction.objectStore('users');

      const cursorRequest = store.openCursor();

      cursorRequest.onsuccess = (event) => {
        const cursor = event.target.result;

        if (cursor) {
          callback(cursor.value, cursor.key);
          cursor.continue(); // 移动到下一个
        } else {
          resolve(); // 遍历完成
        }
      };

      cursorRequest.onerror = () => reject(cursorRequest.error);
    };
  });
}

// 使用：遍历所有用户
await iterateUsers((user, key) => {
  console.log(`用户 ${key}:`, user.name);
});
```

**索引游标**：

```javascript
// 按年龄索引遍历，从大到小
const ageIndex = store.index('age');
const cursorRequest = ageIndex.openCursor(null, 'prev'); // 'prev' 降序，'next' 升序

cursorRequest.onsuccess = (event) => {
  const cursor = event.target.result;
  if (cursor) {
    console.log(cursor.value);
    cursor.continue();
  }
};
```

### 4.4 分页查询

```javascript
function getUsersPage(page, pageSize) {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('myApp', 1);

    request.onsuccess = (event) => {
      const db = event.target.result;
      const transaction = db.transaction(['users'], 'readonly');
      const store = transaction.objectStore('users');

      const results = [];
      let skipped = 0;
      const offset = (page - 1) * pageSize;

      const cursorRequest = store.openCursor();

      cursorRequest.onsuccess = (event) => {
        const cursor = event.target.result;

        if (!cursor) {
          resolve(results);
          return;
        }

        if (skipped < offset) {
          skipped++;
          cursor.continue();
          return;
        }

        if (results.length < pageSize) {
          results.push(cursor.value);
          cursor.continue();
        } else {
          resolve(results);
        }
      };

      cursorRequest.onerror = () => reject(cursorRequest.error);
    };
  });
}

// 获取第 2 页，每页 10 条
const page2 = await getUsersPage(2, 10);
```

## 五、数据库版本升级

升级数据库版本时，可在 `onupgradeneeded` 中修改结构：

```javascript
const request = indexedDB.open('myApp', 2); // 从 1 升级到 2

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  const oldVersion = event.oldVersion;  // 旧版本号
  const newVersion = event.newVersion;  // 新版本号

  // 按版本逐步升级
  if (oldVersion < 1) {
    // v0 -> v1：创建 users 表
    const store = db.createObjectStore('users', { keyPath: 'id', autoIncrement: true });
    store.createIndex('email', 'email', { unique: true });
  }

  if (oldVersion < 2) {
    // v1 -> v2：添加新表
    const orderStore = db.createObjectStore('orders', { keyPath: 'orderId' });
    orderStore.createIndex('userId', 'userId', { unique: false });
    orderStore.createIndex('createTime', 'createTime', { unique: false });
  }

  // 删除旧索引
  if (oldVersion < 3) {
    const store = transaction.objectStore('users');
    if (store.indexNames.contains('oldIndex')) {
      store.deleteIndex('oldIndex');
    }
  }
};
```

> **注意**：`onupgradeneeded` 中**不能读取数据**（versionchange 事务不允许），只能修改结构。数据迁移需要在升级完成后的 `onsuccess` 中进行。

## 六、Promise 封装

原生 IndexedDB API 基于事件回调，建议封装为 Promise：

```javascript
class IndexedDBWrapper {
  constructor(dbName, version, onUpgrade) {
    this.dbName = dbName;
    this.version = version;
    this.onUpgrade = onUpgrade;
    this.db = null;
  }

  async open() {
    return new Promise((resolve, reject) => {
      const request = indexedDB.open(this.dbName, this.version);

      request.onupgradeneeded = (event) => {
        const db = event.target.result;
        const transaction = event.target.transaction;
        this.onUpgrade?.(db, event.oldVersion, transaction);
      };

      request.onsuccess = (event) => {
        this.db = event.target.result;
        resolve(this.db);
      };

      request.onerror = () => reject(request.error);
      request.onblocked = () => reject(new Error('数据库被阻塞，请关闭其他标签页'));
    });
  }

  async add(storeName, data, key) {
    return new Promise((resolve, reject) => {
      const transaction = this.db.transaction([storeName], 'readwrite');
      const store = transaction.objectStore(storeName);
      const request = key ? store.add(data, key) : store.add(data);

      request.onsuccess = () => resolve(request.result);
      request.onerror = () => reject(request.error);
    });
  }

  async get(storeName, key) {
    return new Promise((resolve, reject) => {
      const transaction = this.db.transaction([storeName], 'readonly');
      const store = transaction.objectStore(storeName);
      const request = store.get(key);

      request.onsuccess = () => resolve(request.result);
      request.onerror = () => reject(request.error);
    });
  }

  async put(storeName, data, key) {
    return new Promise((resolve, reject) => {
      const transaction = this.db.transaction([storeName], 'readwrite');
      const store = transaction.objectStore(storeName);
      const request = key ? store.put(data, key) : store.put(data);

      request.onsuccess = () => resolve(request.result);
      request.onerror = () => reject(request.error);
    });
  }

  async delete(storeName, key) {
    return new Promise((resolve, reject) => {
      const transaction = this.db.transaction([storeName], 'readwrite');
      const store = transaction.objectStore(storeName);
      const request = store.delete(key);

      request.onsuccess = () => resolve();
      request.onerror = () => reject(request.error);
    });
  }

  async getAll(storeName, query, count) {
    return new Promise((resolve, reject) => {
      const transaction = this.db.transaction([storeName], 'readonly');
      const store = transaction.objectStore(storeName);
      const request = store.getAll(query, count);

      request.onsuccess = () => resolve(request.result);
      request.onerror = () => reject(request.error);
    });
  }

  async close() {
    this.db?.close();
    this.db = null;
  }
}

// 使用示例
const db = new IndexedDBWrapper('myApp', 1, (db, oldVersion) => {
  if (oldVersion < 1) {
    const store = db.createObjectStore('users', { keyPath: 'id', autoIncrement: true });
    store.createIndex('email', 'email', { unique: true });
  }
});

await db.open();
const userId = await db.add('users', { name: '李四', email: 'lisi@example.com' });
const user = await db.get('users', userId);
await db.close();
```

## 七、常见问题与注意事项

### 7.1 存储空间限制

```javascript
// 查询已用空间（Chrome/Edge 支持）
async function estimateStorage() {
  if ('storage' in navigator && 'estimate' in navigator.storage) {
    const estimate = await navigator.storage.estimate();
    console.log('总配额:', (estimate.quota / 1024 / 1024).toFixed(2), 'MB');
    console.log('已使用:', (estimate.usage / 1024 / 1024).toFixed(2), 'MB');
    console.log('IndexedDB 用量:', estimate.usageDetails?.indexedDB);
  }
}
```

### 7.2 数据持久化声明

```javascript
// 请求持久化存储（防止浏览器自动清理）
async function requestPersistentStorage() {
  if (navigator.storage && navigator.storage.persist) {
    const isPersistent = await navigator.storage.persist();
    console.log('持久化状态:', isPersistent); // true 表示不会被自动清理
  }
}
```

### 7.3 事务自动提交

IndexedDB 事务会在**所有请求完成后自动提交**，无需手动调用 `commit()`（旧浏览器不支持）。事务完成后不能再发起新请求。

```javascript
// ❌ 错误：事务完成后不能再发请求
const transaction = db.transaction(['users'], 'readwrite');
const store = transaction.objectStore('users');

store.add(data1);

setTimeout(() => {
  store.add(data2); // 可能失败，事务可能已经提交
}, 1000);

// ✅ 正确：所有请求同步发起
store.add(data1);
store.add(data2);
store.add(data3);
// 所有请求完成后事务自动提交
```

### 7.4 错误处理

```javascript
// 事务级别的错误监听
transaction.onerror = (event) => {
  console.error('事务失败:', event.target.error);
  // 整个事务会回滚
};

// 事务完成监听
transaction.oncomplete = () => {
  console.log('事务成功提交');
};

// 单个请求的错误不会阻止其他请求
request1.onerror = (event) => {
  event.preventDefault(); // 阻止冒泡到事务 onerror
  console.log('请求1失败，但不影响其他请求');
};
```

## 附录：IndexedDB 与 SQLite 横向对比

IndexedDB 与 SQLite 都是本地/嵌入式数据库方案，但二者面向的运行环境、数据模型和使用方式差异明显。下面以 **IndexedDB（现代浏览器实现）** 和 **SQLite 3.53.2（2026-06-03 发布）** 为例进行对比。

### 核心差异一览

| 对比维度 | IndexedDB | SQLite |
|---------|-----------|--------|
| 运行环境 | 浏览器主线程 / Web Worker | 任意进程（Node.js、移动端、桌面端、嵌入式设备） |
| 数据模型 | NoSQL，对象仓库（键值对） | 关系型，表 + 行 + 列 |
| 查询语言 | 原生 JavaScript API（游标、索引、范围） | 标准 SQL |
| 事务支持 | 支持，基于对象仓库的事务 | 支持完整 ACID |
| 存储方式 | 浏览器内部文件（同源隔离） | 单个 `.db` / `.sqlite` 文件 |
| 典型容量 | 50MB+，具体取决于浏览器和磁盘配额 | 受磁盘空间限制，可达 TB 级 |
| 并发能力 | 同一仓库读写事务串行，只读可并发 | WAL 模式下读写可高度并发 |
| 网络访问 | 不支持 | 不支持（本身无服务器，但可借助工具暴露） |
| 是否需要服务进程 | 否 | 否 |
| 索引能力 | 支持对象仓库索引和范围查询 | 支持 B-tree 索引、复合索引、表达式索引等 |
| 跨标签页共享 | 同源标签页共享 | 同一文件可被多个进程/线程共享 |
| 部署依赖 | 浏览器内置 | 需要引入 SQLite 库（Node.js 22.5+ 已内置 `node:sqlite`） |

### 查询能力对比

| 能力 | IndexedDB | SQLite |
|------|-----------|--------|
| 精确查询 | `store.get(key)` | `SELECT * WHERE id = ?` |
| 条件过滤 | 通过索引范围 + 游标实现 | `WHERE`、`AND`、`OR` 等 SQL 条件 |
| 聚合统计 | 需手动遍历计算 | `COUNT`、`SUM`、`AVG`、`GROUP BY` |
| 多表关联 | 不支持，需手动组织数据 | `JOIN`、`LEFT JOIN`、`子查询` |
| 排序 | 索引游标支持升/降序 | `ORDER BY` |
| 分页 | 游标偏移实现 | `LIMIT`、`OFFSET` |
| 全文搜索 | 需自行实现或借助库 | 内置 FTS5 扩展 |

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 浏览器离线应用缓存 | IndexedDB | 原生内置，无需额外依赖 |
| 前端大量结构化数据本地存储 | IndexedDB | 支持对象、Blob、ArrayBuffer 等复杂类型 |
| Node.js / 桌面 / 移动端本地数据库 | SQLite | 关系模型成熟，SQL 生态丰富 |
| 需要复杂查询和报表统计 | SQLite | SQL 表达能力强，支持聚合和 JOIN |
| 多表事务和强一致性业务 | SQLite | ACID 完整，外键约束完善 |
| 浏览器内缓存图片/文件分片 | IndexedDB | 可直接存储 Blob 和 ArrayBuffer |

### 选型建议

- **浏览器端**：优先选择 IndexedDB。它是浏览器原生的结构化存储方案，与 Service Worker、PWA 离线能力天然契合。
- **非浏览器端（Node.js、桌面、移动、IoT）**：优先选择 SQLite。它是业界最成熟的嵌入式关系型数据库，SQL 能力完整，工具链丰富。
- **混合架构**：浏览器端用 IndexedDB 做离线缓存，服务端/桌面端用 SQLite 做本地持久化，通过 API 同步两端数据。

### 总结

IndexedDB 是**浏览器生态的本地 NoSQL 数据库**，适合存储结构化对象、文件分片和离线应用数据；SQLite 是**通用嵌入式关系型数据库**，适合需要 SQL 能力、复杂查询和多表事务的场景。两者并非竞争关系，而是分别服务于浏览器与非浏览器环境，实际项目中经常组合使用。
