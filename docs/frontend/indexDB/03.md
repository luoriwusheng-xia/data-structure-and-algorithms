# 前端本地存储技术方案对比

> 系统对比前端可用的本地存储方案，包括容量、API、性能、适用场景等维度，帮助技术选型决策。

---

## 一、存储方案概览

| 方案 | 容量 | 数据类型 | 同步/异步 | 同源策略 |
|------|------|---------|----------|---------|
| Cookie | ~4KB | 字符串 | 同步 | ✅ |
| localStorage | ~5-10MB | 字符串 | 同步 | ✅ |
| sessionStorage | ~5-10MB | 字符串 | 同步 | ✅ (标签页级) |
| IndexedDB | 50MB+ | 结构化数据 | 异步 | ✅ |
| Cache Storage | 磁盘空间依赖 | Request/Response | 异步 | ✅ |
| OPFS | 磁盘空间依赖 | 文件/流 | 异步 | ✅ |
| Memory (变量) | 内存限制 | 任意 | 同步 | ❌ (页面级) |

## 二、各方案详解

### 2.1 Cookie

```javascript
// 设置 Cookie（默认会话级）
document.cookie = 'token=abc123; path=/';

// 设置带过期时间的 Cookie
const expires = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000).toUTCString();
document.cookie = `token=abc123; expires=${expires}; path=/; Secure; SameSite=Strict`;

// 读取（返回所有 Cookie 的字符串）
console.log(document.cookie); // "token=abc123; theme=dark"

// 删除
const past = new Date(0).toUTCString();
document.cookie = 'token=; expires=' + past + '; path=/';
```

| 属性 | 说明 |
|------|------|
| `Secure` | 仅 HTTPS 传输 |
| `HttpOnly` | 禁止 JavaScript 读取（防 XSS）|
| `SameSite=Strict` | 禁止跨站携带（防 CSRF）|
| `Domain` | 指定可访问的域名 |
| `Path` | 指定可访问的路径 |

**适用场景**：
- 身份认证 Token（配合 `HttpOnly` + `Secure`）
- 服务端需要读取的小数据
- 跨页面共享的轻量配置

**限制**：
- 每次 HTTP 请求都会自动携带，增加请求头体积
- 容量极小（约 4KB）
- 只能存储字符串

### 2.2 localStorage / sessionStorage

```javascript
// ========== localStorage ==========
// 存储（只能存字符串）
localStorage.setItem('user', JSON.stringify({ name: '张三', age: 25 }));

// 读取
const user = JSON.parse(localStorage.getItem('user') || 'null');

// 删除单条
localStorage.removeItem('user');

// 清空全部
localStorage.clear();

// 遍历
for (let i = 0; i < localStorage.length; i++) {
  const key = localStorage.key(i);
  console.log(key, localStorage.getItem(key));
}

// ========== sessionStorage ==========
// API 与 localStorage 完全一致
// 区别在于：数据只在当前标签页会话期间保留

sessionStorage.setItem('tempData', '仅当前标签页可见');

// 标签页关闭后自动清除
// 同源新标签页无法访问
```

| 特性 | localStorage | sessionStorage |
|------|-------------|----------------|
| 生命周期 | 永久（除非手动清除）| 标签页关闭即清除 |
| 作用域 | 同源窗口共享 | 仅当前标签页 |
| 容量 | ~5-10MB | ~5-10MB |
| 同步/异步 | 同步（阻塞主线程）| 同步 |

**适用场景**：
- 用户偏好设置（主题、语言）
- 简单的表单草稿
- 客户端缓存的小型配置数据

**注意事项**：

```javascript
// ❌ 错误：存储前未 JSON 序列化
localStorage.setItem('data', { a: 1 }); // 实际存的是 "[object Object]"

// ❌ 错误：无异常处理，可能超出容量
localStorage.setItem('largeData', hugeString); // 可能抛出 QuotaExceededError

// ✅ 正确：封装工具函数
const storage = {
  set(key, value) {
    try {
      const serialized = JSON.stringify(value);
      localStorage.setItem(key, serialized);
      return true;
    } catch (e) {
      if (e.name === 'QuotaExceededError') {
        console.error('存储空间不足');
        // 清理旧数据或降级处理
      }
      return false;
    }
  },

  get(key) {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : null;
    } catch (e) {
      console.error('解析失败:', e);
      localStorage.removeItem(key);
      return null;
    }
  },

  remove(key) {
    localStorage.removeItem(key);
  }
};
```

### 2.4 Cache Storage

Cache Storage 是 Service Worker 专用的缓存 API，用于缓存网络请求的 Response。

```javascript
// ========== 基本操作 ==========

// 打开/创建缓存
const cache = await caches.open('app-cache-v1');

// 缓存单个请求
await cache.put('/api/users', new Response(JSON.stringify(users)));

// 匹配缓存
const response = await cache.match('/api/users');
if (response) {
  const data = await response.json();
}

// 删除缓存
await caches.delete('app-cache-v1');

// ========== Service Worker 中典型用法 ==========

// sw.js
const CACHE_NAME = 'app-v1';
const STATIC_ASSETS = [
  '/',
  '/index.html',
  '/app.js',
  '/styles.css'
];

// 安装时预缓存静态资源
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => {
      return cache.addAll(STATIC_ASSETS);
    })
  );
});

// 拦截请求，优先读缓存
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then((response) => {
      // 缓存命中直接返回
      if (response) return response;

      // 否则走网络请求
      return fetch(event.request).then((networkResponse) => {
        // 可选：动态缓存新资源
        if (networkResponse.ok) {
          const clone = networkResponse.clone();
          caches.open(CACHE_NAME).then((cache) => {
            cache.put(event.request, clone);
          });
        }
        return networkResponse;
      });
    })
  );
});

// ========== 缓存策略模式 ==========

// 1. Cache First（静态资源）
async function cacheFirst(request) {
  const cached = await caches.match(request);
  if (cached) return cached;
  return fetch(request);
}

// 2. Network First（实时数据）
async function networkFirst(request) {
  try {
    const network = await fetch(request);
    // 更新缓存
    const cache = await caches.open('dynamic');
    cache.put(request, network.clone());
    return network;
  } catch {
    return caches.match(request);
  }
}

// 3. Stale While Revalidate（平衡方案）
async function staleWhileRevalidate(request) {
  const cache = await caches.open('dynamic');
  const cached = await cache.match(request);

  // 发起网络请求更新缓存（不阻塞）
  const networkPromise = fetch(request).then((response) => {
    cache.put(request, response.clone());
    return response;
  });

  // 优先返回缓存，同时后台更新
  return cached || await networkPromise;
}
```

| 特性 | 说明 |
|------|------|
| 数据类型 | Request / Response 对象 |
| 生命周期 | 长期，可手动管理 |
| 容量 | 取决于可用磁盘空间 |
| 使用场景 | Service Worker 网络缓存 |
| 索引能力 | 仅通过 Request URL 匹配 |

**适用场景**：
- PWA 离线缓存
- 静态资源缓存（JS/CSS/图片）
- API 响应缓存

### 2.5 OPFS（Origin Private File System）

File System Access API 的一部分，提供类似文件系统的接口，适合处理大文件。

```javascript
// ========== 获取文件系统根目录 ==========
const root = await navigator.storage.getDirectory();

// ========== 文件操作 ==========

// 创建/获取文件
const fileHandle = await root.getFileHandle('data.txt', { create: true });

// 写入文件（需要获取写入权限）
const writable = await fileHandle.createWritable();
await writable.write('Hello, OPFS!');
await writable.write(new Blob(['更多数据']));
await writable.close();

// 读取文件
const file = await fileHandle.getFile();
const content = await file.text();
console.log(content);

// ========== 目录操作 ==========

// 创建子目录
const subDir = await root.getDirectoryHandle('uploads', { create: true });

// 遍历目录
for await (const [name, handle] of root.entries()) {
  console.log(name, handle.kind); // 'file' 或 'directory'
}

// 删除文件/目录
await root.removeEntry('data.txt');
await root.removeEntry('uploads', { recursive: true });

// ========== 大文件流式写入 ==========

async function streamWriteFile(filename, stream) {
  const fileHandle = await root.getFileHandle(filename, { create: true });
  const writable = await fileHandle.createWritable();

  const reader = stream.getReader();
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    await writable.write(value);
  }

  await writable.close();
}

// ========== 与 File System Access API 结合 ==========

// 用户选择本地文件保存位置（需要用户交互触发）
async function saveToUserDisk(blob, suggestedName) {
  const handle = await window.showSaveFilePicker({
    suggestedName,
    types: [{
      description: 'Text Files',
      accept: { 'text/plain': ['.txt'] }
    }]
  });

  const writable = await handle.createWritable();
  await writable.write(blob);
  await writable.close();
}
```

| 特性 | 说明 |
|------|------|
| 数据类型 | 文件、流 |
| 容量 | 受磁盘空间限制 |
| 路径结构 | 支持目录层级 |
| 访问方式 | 异步，基于句柄 |
| 用户可见性 | OPFS 对用户不可见；File System Access 可访问用户文件 |

**适用场景**：
- 视频/音频编辑（大文件处理）
- 本地文件缓存
- 需要文件系统语义的操作

### 2.6 Memory Storage

纯内存变量存储，页面刷新即丢失。

```javascript
// 简单的内存缓存实现
class MemoryCache {
  constructor(maxSize = 100) {
    this.cache = new Map();
    this.maxSize = maxSize;
  }

  get(key) {
    const item = this.cache.get(key);
    if (!item) return undefined;

    // LRU：访问后移到末尾
    this.cache.delete(key);
    this.cache.set(key, item);

    return item.value;
  }

  set(key, value, ttl) {
    // 超出容量时淘汰最久未使用
    if (this.cache.size >= this.maxSize) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }

    const item = { value, expireAt: ttl ? Date.now() + ttl : null };
    this.cache.set(key, item);
  }

  has(key) {
    const item = this.cache.get(key);
    if (!item) return false;
    if (item.expireAt && item.expireAt < Date.now()) {
      this.cache.delete(key);
      return false;
    }
    return true;
  }

  clear() {
    this.cache.clear();
  }
}
```

**适用场景**：
- 组件级状态（React Context、Vue provide/inject）
- 高频访问的临时计算结果
- 路由跳转间的数据传递

## 三、方案选型决策树

```
是否需要服务端读取？
├── 是 → Cookie（注意大小限制和安全性）
└── 否 → 数据量多大？
    ├── < 5KB → Cookie（如需跨请求携带）或 localStorage
    ├── < 5MB → localStorage / sessionStorage
    │   └── 是否需要标签页隔离？
    │       ├── 是 → sessionStorage
    │       └── 否 → localStorage
    ├── 结构化数据 + 查询需求 → IndexedDB
    │   └── 需要更友好的 API？
    │       ├── 是 → Dexie.js / localForage
    │       └── 否 → 原生 IndexedDB
    ├── 网络响应缓存 → Cache Storage（配合 Service Worker）
    ├── 大文件/流处理 → OPFS
    └── 纯临时数据 → Memory
```

## 四、混合存储架构实战

实际项目中通常组合使用多种存储方案：

```javascript
// ========== 分层缓存架构 ==========

class HybridStorage {
  constructor() {
    this.memory = new MemoryCache(50);           // L1：内存缓存
    this.indexedDB = null;                        // L2：IndexedDB（懒加载）
  }

  async init() {
    this.indexedDB = await initDB();
  }

  // 读取：L1 -> L2 -> 网络
  async get(key, fetcher) {
    // 1. 查内存
    if (this.memory.has(key)) {
      return this.memory.get(key);
    }

    // 2. 查 IndexedDB
    if (this.indexedDB) {
      const cached = await this.getFromDB(key);
      if (cached && !this.isExpired(cached)) {
        this.memory.set(key, cached.value);
        return cached.value;
      }
    }

    // 3. 网络请求
    if (fetcher) {
      const value = await fetcher();
      await this.set(key, value);
      return value;
    }

    return null;
  }

  async set(key, value, options = {}) {
    const { ttl = 3600000, persist = true } = options;

    // 写入 L1
    this.memory.set(key, value, ttl);

    // 写入 L2
    if (persist && this.indexedDB) {
      await this.saveToDB(key, {
        value,
        storedAt: Date.now(),
        ttl
      });
    }
  }

  // 持久化存储配额管理
  async checkQuota() {
    const estimate = await navigator.storage.estimate();
    const usage = estimate.usage || 0;
    const quota = estimate.quota || Infinity;
    const ratio = usage / quota;

    if (ratio > 0.8) {
      // 触发清理策略
      await this.evictOldData();
    }

    return { usage, quota, ratio };
  }

  async evictOldData() {
    // 清理过期数据
    const trans = this.indexedDB.transaction(['cache'], 'readwrite');
    const store = trans.objectStore('cache');
    const all = await promisifyRequest(store.getAll());

    const now = Date.now();
    for (const item of all) {
      if (item.storedAt + item.ttl < now) {
        await promisifyRequest(store.delete(item.key));
      }
    }
  }
}

// ========== 实际使用场景 ==========

const storage = new HybridStorage();
await storage.init();

// 用户配置（localStorage 足够）
const config = JSON.parse(localStorage.getItem('appConfig') || '{}');

// 大量数据缓存（IndexedDB + Memory）
const users = await storage.get('users_list', async () => {
  const res = await fetch('/api/users');
  return res.json();
});

// 大文件缓存（IndexedDB 分片）
const imageBlob = await storage.get(`file_${imageUrl}`, async () => {
  const res = await fetch(imageUrl);
  return res.blob();
});

// Service Worker 静态资源缓存（Cache Storage）
// 在 sw.js 中处理

// 临时计算结果（纯内存）
const computed = new MemoryCache();
```

## 五、安全性对比

| 方案 | XSS 防护 | CSRF 防护 | 数据可见性 |
|------|---------|----------|-----------|
| Cookie (HttpOnly) | ✅ 不可被 JS 读取 | 需 SameSite | 服务端 + 客户端 |
| Cookie (无 HttpOnly) | ❌ 可被窃取 | 需 SameSite | 服务端 + 客户端 |
| localStorage | ❌ 可被 XSS 读取 | N/A | 仅客户端 |
| sessionStorage | ❌ 可被 XSS 读取 | N/A | 仅当前标签页 |
| IndexedDB | ❌ 可被 XSS 读取 | N/A | 仅客户端 |
| Cache Storage | ❌ 可被 SW 读取 | N/A | Service Worker |

**安全建议**：

```javascript
// 1. 敏感 Token 存 HttpOnly Cookie
document.cookie = 'auth=xxx; HttpOnly; Secure; SameSite=Strict';

// 2. 本地存储数据做签名防篡改
function storeWithIntegrity(key, data, secret) {
  const payload = JSON.stringify(data);
  const signature = hash(payload + secret); // HMAC
  localStorage.setItem(key, JSON.stringify({ payload, signature }));
}

// 3. 读取时验证签名
function readWithIntegrity(key, secret) {
  const stored = JSON.parse(localStorage.getItem(key) || '{}');
  const expected = hash(stored.payload + secret);
  if (stored.signature !== expected) {
    throw new Error('数据可能被篡改');
  }
  return JSON.parse(stored.payload);
}

// 4. 内容安全策略（CSP）
// 在 HTTP 响应头中设置：
// Content-Security-Policy: default-src 'self'; script-src 'self'
```

## 六、选型速查表

| 需求 | 推荐方案 | 备选方案 |
|------|---------|---------|
| 认证 Token | `HttpOnly Cookie` | - |
| 主题/语言设置 | `localStorage` | `Cookie` |
| 表单草稿 | `sessionStorage` | `localStorage` |
| 离线应用数据 | `IndexedDB` | `Cache Storage` |
| 图片/静态资源缓存 | `Cache Storage` | `IndexedDB` |
| 大文件处理 | `OPFS` | `IndexedDB` |
| API 响应缓存 | `IndexedDB` + 内存 | `Cache Storage` |
| 实时共享数据 | `BroadcastChannel` + `IndexedDB` | `localStorage` + `storage` 事件 |
| 视频/音频编辑 | `OPFS` | 内存 |
| 路由间数据传递 | 内存 / URL params | `sessionStorage` |
