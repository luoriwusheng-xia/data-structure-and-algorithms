# WebAssembly 基础：从字节码到浏览器高性能计算

WebAssembly（简称 WASM）是一种面向现代浏览器的二进制指令格式，旨在作为 JavaScript 的补充，让 C/C++/Rust/Go 等语言编写的代码以接近原生的速度在 Web 平台运行。

## 一、WASM 是什么

WebAssembly 是一种低层级的类汇编语言，具有如下特点：

- **二进制格式**：文件后缀通常为 `.wasm`，体积小巧，解析速度快。
- **栈式虚拟机**：基于结构化控制流和栈机模型设计。
- **沙箱安全**：运行在浏览器或宿主环境的独立线性内存中。
- **语言无关**：C/C++、Rust、Go、AssemblyScript、Zig 等都可编译为 WASM。
- **可移植**：一次编译，可在所有支持 WASM 的浏览器和运行时中运行。

WASM 不是取代 JavaScript，而是与 JavaScript 协同工作，负责计算密集型任务。

## 二、为什么需要 WASM

| 场景 | JavaScript 的问题 | WASM 的优势 |
|------|------------------|------------|
| 图像/视频编解码 | 纯 JS 性能不足 | 接近原生性能 |
| 游戏物理引擎 | GC 停顿、数值计算慢 | 无 GC、SIMD 加速 |
| 加密/哈希计算 | CPU 密集型任务卡顿 | 可复用成熟 C/Rust 库 |
| 科学计算/数据分析 | 大数据量处理慢 | 内存布局紧凑、并行友好 |
| 移植现有代码库 | 重写成本高 | 直接编译复用 |

核心收益：

- **性能**：计算密集型任务可提升数倍到数十倍。
- **复用**：复用现有 C/C++/Rust 生态的算法和库。
- **安全**：默认沙箱运行，内存隔离。

## 三、核心概念

### 3.1 Module（模块）

`.wasm` 文件编译后得到 `WebAssembly.Module`，代表静态的、无状态的代码。

```javascript
const bytes = await fetch('/hello.wasm').then((r) => r.arrayBuffer())
const module = await WebAssembly.compile(bytes)
```

### 3.2 Instance（实例）

`WebAssembly.Instance` 是 Module 的运行时实例，拥有独立的内存和状态。

```javascript
const instance = await WebAssembly.instantiate(module, importObject)
```

### 3.3 Memory（线性内存）

WASM 使用一块连续的、可增长的 ArrayBuffer 作为内存。JS 和 WASM 都可以通过 `WebAssembly.Memory` 读写这块内存。

```javascript
const memory = new WebAssembly.Memory({ initial: 1, maximum: 4 })
// initial 单位是 64KB 页
```

### 3.4 Table（函数表）

函数表用于存储函数引用，主要解决间接调用和动态链接问题。

```javascript
const table = new WebAssembly.Table({ initial: 1, element: 'anyfunc' })
```

### 3.5 Export / Import

- **Export**：WASM 模块暴露给 JS 调用的函数、内存、全局变量等。
- **Import**：WASM 模块从 JS 环境导入的函数或对象。

## 四、WASM 与 JavaScript 的关系

WASM 与 JS 是互补关系：

- **JS 负责**：DOM 操作、事件处理、网络请求、业务逻辑编排。
- **WASM 负责**：计算密集型任务、算法执行、 legacy 代码复用。

典型交互流程：

```
JS 准备输入数据
    ↓
写入 WASM Memory
    ↓
调用 WASM 导出函数
    ↓
从 Memory 读取结果
    ↓
JS 渲染到页面
```

## 五、浏览器中的最小示例

下面是一个手写 WAT（WebAssembly Text Format）并编译运行的最小示例。

### 5.1 WAT 源码

```wasm
;; hello.wat
(module
  ;; 从 JS 导入一个 log 函数
  (import "env" "log" (func $log (param i32)))

  ;; 导出一个 add 函数
  (func $add (param i32 i32) (result i32)
    local.get 0
    local.get 1
    i32.add
  )
  (export "add" (func $add))

  ;; 导出一个计算斐波那契的函数
  (func $fib (param $n i32) (result i32)
    local.get $n
    i32.const 2
    i32.lt_u
    if (result i32)
      local.get $n
    else
      local.get $n
      i32.const 1
      i32.sub
      call $fib

      local.get $n
      i32.const 2
      i32.sub
      call $fib

      i32.add
    end
  )
  (export "fib" (func $fib))
)
```

### 5.2 使用 wabt 编译为 wasm

```bash
wat2wasm hello.wat -o hello.wasm
```

### 5.3 浏览器中加载运行

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>WASM Demo</title>
</head>
<body>
  <script type="module">
    const importObject = {
      env: {
        log: (value) => console.log('WASM says:', value)
      }
    }

    const response = await fetch('hello.wasm')
    const bytes = await response.arrayBuffer()
    const { instance } = await WebAssembly.instantiate(bytes, importObject)

    console.log('add(2, 3) =', instance.exports.add(2, 3))
    console.log('fib(10) =', instance.exports.fib(10))
  </script>
</body>
</html>
```

## 六、WASM 工具链概览

| 工具链 | 语言 | 特点 |
|--------|------|------|
| **Emscripten** | C/C++ | 最成熟，自动处理内存、GL、文件系统 |
| **Rust + wasm-pack** | Rust | 现代、安全、生态活跃，有 wasm-bindgen |
| **AssemblyScript** | TypeScript 子集 | 前端开发者上手快 |
| **TinyGo** | Go | 编译产物小，适合嵌入式 |
| **JCO / ComponentizeJS** | JavaScript | 将 JS 组件化为 WASM |

## 七、常见误区

### 7.1 WASM 不能直接操作 DOM

WASM 没有 DOM API，所有 UI 操作都需要通过 JS 桥接。

### 7.2 WASM 不总是比 JS 快

对于 I/O 密集型、DOM 操作等任务，JS 反而更合适。WASM 的优势在纯计算场景。

### 7.3 WASM 体积不一定小

未优化的 WASM 可能很大。生产环境需要启用优化、压缩和 Streaming 编译。

### 7.4 WASM 有内存限制

线性内存默认 4GB 上限（32 位），超过需要特殊处理或使用 64 位内存提案。

## 八、总结

WebAssembly 为前端打开了高性能计算的大门：

- **基础单元**：Module、Instance、Memory、Table。
- **核心能力**：复用 native 代码、加速计算、保持安全沙箱。
- **适用场景**：编解码、加密、物理引擎、科学计算、游戏核心逻辑。
- **学习路径**：先理解 WAT 和 JS 交互，再用 Rust/C++ 工具链开发真实模块。
