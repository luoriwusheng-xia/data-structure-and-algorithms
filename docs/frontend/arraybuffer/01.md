# ArrayBuffer 深度讲解

在前端开发中，图片、音频、视频、文件、网络数据、加密结果等本质上都是二进制数据。JavaScript 最初并不擅长处理二进制，直到 ES6 引入 `ArrayBuffer` 和 `TypedArray`，才让前端具备了高效、灵活地操作原始二进制数据的能力。

---

## 1. 什么是 ArrayBuffer

`ArrayBuffer` 是一个**固定长度的原始二进制数据缓冲区**。它本身不能直接读写，只是一个内存块的引用。要操作这块内存，需要借助 `TypedArray`（类型化数组）或 `DataView`（数据视图）。

```javascript
// 创建一个 16 字节的内存缓冲区
const buffer = new ArrayBuffer(16)

console.log(buffer.byteLength) // 16
console.log(buffer)            // ArrayBuffer { [Uint8Contents]: <00 00 00 ...>, byteLength: 16 }
```

核心特点：

- **长度固定**：创建后不可扩容或缩容。
- **不能直接访问**：必须通过视图（View）来读写。
- **连续内存**：数据在内存中连续存储，访问效率高。
- **共享底层内存**：多个 TypedArray 可以共享同一个 ArrayBuffer 的不同区间。

## 2. ArrayBuffer、TypedArray 与 DataView 的关系

可以把 `ArrayBuffer` 理解成一块 raw memory（原始内存），而 `TypedArray` 和 `DataView` 是两种不同的"窗口"。

```
ArrayBuffer (16 bytes)
[00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00]

Uint8Array 视图   → 每 1 字节一个元素，共 16 个元素
Uint16Array 视图  → 每 2 字节一个元素，共 8 个元素
Uint32Array 视图  → 每 4 字节一个元素，共 4 个元素
Float64Array 视图 → 每 8 字节一个元素，共 2 个元素

DataView 视图     → 可按任意偏移量读写任意类型
```

### 2.1 TypedArray

TypedArray 是一组具体类型的统称，包括 `Uint8Array`、`Int16Array`、`Float32Array` 等。

```javascript
const buffer = new ArrayBuffer(8)

// 以无符号 8 位整数视图读取/写入
const u8 = new Uint8Array(buffer)
u8[0] = 255
console.log(u8[0]) // 255

// 以无符号 16 位整数视图读取同一缓冲区
const u16 = new Uint16Array(buffer)
console.log(u16[0]) // 255（小端序下）

// 以 32 位浮点数视图写入
const f32 = new Float32Array(buffer)
f32[0] = 3.14
console.log(f32[0]) // 3.140000104904175
```

### 2.2 DataView

`DataView` 更灵活，可以在任意偏移位置以指定字节序读写数据。

```javascript
const buffer = new ArrayBuffer(8)
const view = new DataView(buffer)

// 在偏移 0 处写入一个 16 位无符号整数，使用大端序
view.setUint16(0, 0x1234, false) // false 表示大端序（Big-Endian）

// 在偏移 2 处写入一个 32 位有符号整数，使用小端序
view.setInt32(2, -1000, true)    // true 表示小端序（Little-Endian）

// 读取
console.log(view.getUint16(0, false).toString(16)) // '1234'
console.log(view.getInt32(2, true))                // -1000
```

## 3. 字节序（Endianness）

字节序指多字节数据在内存中的存放顺序：

- **小端序（Little-Endian）**：低位字节放在低地址。x86、ARM 常用。
- **大端序（Big-Endian）**：高位字节放在低地址。网络协议常用。

```javascript
const buffer = new ArrayBuffer(2)
const u8 = new Uint8Array(buffer)
const u16 = new Uint16Array(buffer)

u16[0] = 0x1234
console.log(u8[0].toString(16), u8[1].toString(16))
// 大部分平台输出：'34 12'，即小端序
```

网络传输中通常使用大端序，因此在处理协议数据包时，`DataView` 是更稳妥的选择。

## 4. 应用场景

### 4.1 文件读取

`FileReader` 读取文件为 ArrayBuffer，可用于预览、上传、解析等。

```javascript
const input = document.getElementById('fileInput')

input.addEventListener('change', (event) => {
  const file = event.target.files[0]
  const reader = new FileReader()

  reader.onload = (e) => {
    const arrayBuffer = e.target.result
    const u8 = new Uint8Array(arrayBuffer)
    console.log('文件前 4 字节:', u8.slice(0, 4))
  }

  reader.readAsArrayBuffer(file)
})
```

### 4.2 网络请求

`fetch` 返回的 Response 可以解析为 ArrayBuffer，用于下载二进制资源。

```javascript
async function loadBinary(url) {
  const response = await fetch(url)
  const arrayBuffer = await response.arrayBuffer()
  return new Uint8Array(arrayBuffer)
}

loadBinary('/assets/data.bin').then((data) => {
  console.log('下载字节数:', data.byteLength)
})
```

### 4.3 Web Crypto 加解密

Web Crypto API 的加密结果、密钥材料、盐值等都以 ArrayBuffer 形式传递。

```javascript
const encoder = new TextEncoder()
const data = encoder.encode('hello')

const hashBuffer = await crypto.subtle.digest('SHA-256', data)
const hashArray = Array.from(new Uint8Array(hashBuffer))
const hashHex = hashArray.map((b) => b.toString(16).padStart(2, '0')).join('')
console.log(hashHex)
```

### 4.4 WebSocket 二进制通信

WebSocket 可以发送/接收 ArrayBuffer，常用于游戏、IM、实时音视频控制信令。

```javascript
const ws = new WebSocket('wss://example.com/socket')
ws.binaryType = 'arraybuffer'

ws.onmessage = (event) => {
  const buffer = event.data // ArrayBuffer
  const view = new DataView(buffer)
  const messageType = view.getUint8(0)
  const payloadLength = view.getUint32(1)
  console.log({ messageType, payloadLength })
}

// 发送二进制消息
const buffer = new ArrayBuffer(5)
const view = new DataView(buffer)
view.setUint8(0, 0x01)
view.setUint32(1, 1024)
ws.send(buffer)
```

### 4.5 WebGL / Canvas 图像数据

Canvas 的 `ImageData`、WebGL 的纹理数据本质上都是 TypedArray。

```javascript
const canvas = document.getElementById('canvas')
const ctx = canvas.getContext('2d')
const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height)

// imageData.data 是一个 Uint8ClampedArray
const pixels = imageData.data
pixels[0] = 255   // R
pixels[1] = 0     // G
pixels[2] = 0     // B
pixels[3] = 255   // A

ctx.putImageData(imageData, 0, 0)
```

### 4.6 Web Audio API

`AudioBuffer` 内部就是 ArrayBuffer 的封装，可以直接操作音频采样数据。

```javascript
const audioCtx = new AudioContext()
const buffer = audioCtx.createBuffer(1, 44100, 44100)
const data = buffer.getChannelData(0) // Float32Array

for (let i = 0; i < 44100; i++) {
  data[i] = Math.sin(i / 44100 * Math.PI * 2 * 440)
}
```

### 4.7 WebAssembly 内存交互

WASM 的线性内存（Linear Memory）就是一个 `WebAssembly.Memory`，其 `buffer` 属性是 `ArrayBuffer`。

```javascript
const memory = new WebAssembly.Memory({ initial: 1 }) // 64KB
const u8 = new Uint8Array(memory.buffer)

// 向 WASM 内存写入数据
const text = 'hello wasm'
const bytes = new TextEncoder().encode(text)
u8.set(bytes, 0)
```

### 4.8 大文件分片上传

```javascript
async function uploadFileInChunks(file, chunkSize = 1024 * 1024) {
  const totalChunks = Math.ceil(file.size / chunkSize)

  for (let i = 0; i < totalChunks; i++) {
    const start = i * chunkSize
    const end = Math.min(start + chunkSize, file.size)
    const chunk = file.slice(start, end)
    const arrayBuffer = await chunk.arrayBuffer()

    await fetch('/api/upload', {
      method: 'POST',
      headers: {
        'X-Chunk-Index': String(i),
        'X-Total-Chunks': String(totalChunks)
      },
      body: arrayBuffer
    })
  }
}
```

### 4.9 Worker 线程间传输数据

ArrayBuffer 可以作为 `Transferable Object` 在 Worker 之间转移所有权，避免复制大内存。

```javascript
// main.js
const worker = new Worker('worker.js')
const buffer = new ArrayBuffer(1024 * 1024) // 1MB

// 转移所有权，主线程不再能访问 buffer
worker.postMessage({ buffer }, [buffer])

// worker.js
self.onmessage = (event) => {
  const { buffer } = event.data
  const u8 = new Uint8Array(buffer)
  u8[0] = 42
  // 处理完后再转移回主线程
  self.postMessage({ buffer }, [buffer])
}
```

## 5. 性能与注意事项

### 5.1 Transferable Objects

通过 `postMessage` 的第二个参数转移 ArrayBuffer 所有权，可以零拷贝传输大数据。转移后原上下文不能再访问该 buffer。

```javascript
const buffer = new ArrayBuffer(100)
worker.postMessage({ buffer }, [buffer])
// 下面这行会报错，因为所有权已转移
// new Uint8Array(buffer)
```

### 5.2 Structured Clone

如果不使用 Transferable，`postMessage` 会通过 Structured Clone 复制数据。对于大 ArrayBuffer，复制成本高，应尽量使用 Transferable。

### 5.3 避免频繁创建

ArrayBuffer 分配的是连续内存，频繁创建和销毁会增加 GC 压力。对于高频场景，可以复用 buffer 或使用对象池。

```javascript
// 不推荐：每次处理都创建新 buffer
function process(data) {
  const buffer = new ArrayBuffer(data.length)
  // ...
}

// 推荐：复用 buffer
const pool = new ArrayBuffer(1024)
function process(data) {
  const view = new Uint8Array(pool, 0, data.length)
  // ...
}
```

### 5.4 类型选择

根据数据特征选择合适的 TypedArray：

| 类型 | 每个元素字节 | 取值范围 | 典型用途 |
| --- | --- | --- | --- |
| `Uint8Array` | 1 | 0 ~ 255 | 原始字节、图片像素 |
| `Uint16Array` | 2 | 0 ~ 65535 | 音频采样 |
| `Int32Array` | 4 | -2^31 ~ 2^31-1 | 整数坐标、索引 |
| `Float32Array` | 4 | 单精度浮点 | 3D 顶点、WebGL |
| `Float64Array` | 8 | 双精度浮点 | 科学计算 |

## 6. 总结

`ArrayBuffer` 是前端处理二进制数据的基石。它本身只是一块原始内存，配合 `TypedArray` 可以高效批量读写，配合 `DataView` 可以灵活处理字节序和任意偏移。

在前端工程化中，ArrayBuffer 广泛应用于文件处理、网络通信、加解密、音视频、图像处理、WebGL、WASM、Worker 间传输等场景。掌握它，是深入理解现代前端技术栈的重要一步。

| 场景 | 推荐视图/API |
| --- | --- |
| 原始字节流 | `Uint8Array` |
| 协议解析、字节序控制 | `DataView` |
| 数值计算、矩阵运算 | `Float32Array` / `Float64Array` |
| 大文件传输 | `Transferable ArrayBuffer` |
| 网络/文件/加密二进制 | `Response.arrayBuffer()` / `FileReader.readAsArrayBuffer()` |
