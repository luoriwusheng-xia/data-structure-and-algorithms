## 1. SQLite 是什么

SQLite 是一个**嵌入式关系型数据库引擎**，与传统数据库（MySQL、PostgreSQL）最大的区别是：

| 特性 | SQLite | MySQL |
|------|--------|-------|
| 架构 | 嵌入式，无独立进程 | C/S 架构，需要服务端进程 |
| 存储 | 单文件（.db 或 .sqlite） | 多文件 + 独立存储引擎 |
| 配置 | 零配置，开箱即用 | 需要安装、配置、启动服务 |
| 并发 | 读多写少场景友好 | 高并发读写优化 |
| 部署 | 直接嵌入应用程序 | 独立部署数据库服务 |
| 事务 | 支持 ACID | 支持 ACID |

**一句话理解**：SQLite 把整个数据库装在一个文件里，你的程序直接读写这个文件，不需要通过网络连接数据库服务器。

## 2. 安装与启动

### 2.1 安装

**Windows**：

```bash
# 通过 scoop 安装
scoop install sqlite

# 或通过官网下载：https://sqlite.org/download.html
```

**macOS**：

```bash
# 系统自带，也可通过 Homebrew 更新
brew install sqlite3
```

**Linux**：

```bash
sudo apt-get install sqlite3    # Debian/Ubuntu
sudo yum install sqlite         # CentOS/RHEL
```

**Node.js 环境**：

```bash
npm install better-sqlite3   # 推荐，同步 API，性能最好
# 或
npm install sqlite3          # 异步 API，基于 libsqlite3
```

> **Windows 用户注意**：`better-sqlite3` 需要编译原生模块。安装 Node.js 时请在"用于本地模块的工具"页面勾选**"自动安装所需工具"**；若已错过，可运行 `C:\Program Files\nodejs\install_tools.bat` 补装（需几分钟）。

### 2.2 启动交互式命令行

```bash
sqlite3                      # 进入交互模式
sqlite3 mydb.db              # 打开/创建数据库文件
sqlite3 :memory:             # 创建内存数据库（数据不持久化）
```

### 2.3 常用命令行命令

```sql
-- SQLite 的点命令（不是 SQL，是 CLI 工具命令）
.help              -- 查看所有点命令
.databases         -- 查看当前连接的数据库
.tables            -- 查看所有表
.schema users      -- 查看表结构
.indexes           -- 查看所有索引
.mode table        -- 设置输出格式：table/column/json/csv
.headers on        -- 显示列名
.timer on          -- 显示执行时间
.quit 或 .exit     -- 退出
```

## 3. 第一个 SQLite 数据库

### 3.1 通过命令行创建

```bash
sqlite3 first.db
```

```sql
-- 创建用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增主键
    username TEXT NOT NULL,
    email TEXT UNIQUE,
    age INTEGER CHECK (age >= 0),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 插入数据
INSERT INTO users (username, email, age) VALUES
    ('张三', 'zhangsan@example.com', 25),
    ('李四', 'lisi@example.com', 30),
    ('王五', 'wangwu@example.com', 28);

-- 查询
SELECT * FROM users;
SELECT username, email FROM users WHERE age > 26;

-- 更新
UPDATE users SET age = 31 WHERE username = '李四';

-- 删除
DELETE FROM users WHERE username = '王五';
```

### 3.2 通过 Node.js 操作

```javascript
// 使用 better-sqlite3（推荐）
const Database = require('better-sqlite3');

// 打开数据库（不存在则自动创建）
const db = new Database('first.db');

// 创建表
db.exec(`
    CREATE TABLE IF NOT EXISTS products (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        price REAL NOT NULL,
        stock INTEGER DEFAULT 0
    )
`);

// 插入（预编译语句，防止 SQL 注入）
const insert = db.prepare('INSERT INTO products (name, price, stock) VALUES (?, ?, ?)');
insert.run('iPhone 15', 5999.00, 100);
insert.run('MacBook Pro', 14999.00, 50);

// 查询单行
const product = db.prepare('SELECT * FROM products WHERE id = ?').get(1);
console.log(product);  // { id: 1, name: 'iPhone 15', price: 5999, stock: 100 }

// 查询多行
const allProducts = db.prepare('SELECT * FROM products').all();
console.log(allProducts);

// 事务
const insertMany = db.transaction((items) => {
    for (const item of items) {
        insert.run(item.name, item.price, item.stock);
    }
});

insertMany([
    { name: 'iPad', price: 3999, stock: 200 },
    { name: 'AirPods', price: 1299, stock: 500 }
]);

// 关闭连接
db.close();
```

## 4. 核心数据类型

SQLite 采用**动态类型系统**，与 MySQL 的严格类型不同：

### 4.1 存储类型（Storage Classes）

| 存储类型 | 说明 | 对应 MySQL 类型 |
|---------|------|----------------|
| `NULL` | 空值 | NULL |
| `INTEGER` | 整型（1-8 字节）| INT/BIGINT |
| `REAL` | 浮点数（8 字节 IEEE）| FLOAT/DOUBLE |
| `TEXT` | 文本字符串 | VARCHAR/TEXT |
| `BLOB` | 二进制数据 | BLOB |

### 4.2 类型亲和性（Type Affinity）

SQLite 有**类型亲和性**机制：你声明的类型名只是"建议"，实际存储会根据值的类型决定。

```sql
-- 以下声明都会被接受，但实际存储行为不同
CREATE TABLE t1 (
    a INTEGER,      -- 整数亲和性
    b REAL,         -- 实数亲和性
    c TEXT,         -- 文本亲和性
    d BLOB,         -- 无亲和性（原样存储）
    e VARCHAR(50),  -- 被识别为 TEXT 亲和性
    f FLOAT,        -- 被识别为 REAL 亲和性
    g INT,          -- 被识别为 INTEGER 亲和性
    h BOOLEAN       -- 被识别为 INTEGER 亲和性（0/1）
);

-- 甚至这样也能工作（但不推荐）
INSERT INTO t1 (a, b) VALUES ('hello', 'world');  -- a 存储为 TEXT
INSERT INTO t1 (a) VALUES (3.14);                  -- a 存储为 REAL
INSERT INTO t1 (a) VALUES (42);                     -- a 存储为 INTEGER
```

### 4.3 日期和时间处理

SQLite 没有专门的 DATE/TIME 类型，通常用 TEXT、REAL 或 INTEGER 存储：

```sql
-- 方式 1：TEXT，ISO8601 格式（推荐）
CREATE TABLE events (
    id INTEGER PRIMARY KEY,
    event_time TEXT DEFAULT (datetime('now'))  -- '2024-01-15 09:30:00'
);

-- 方式 2：INTEGER，Unix 时间戳
CREATE TABLE logs (
    id INTEGER PRIMARY KEY,
    created_at INTEGER DEFAULT (strftime('%s', 'now'))  -- 1705312200
);

-- 方式 3：REAL，Julian Day

-- 日期函数
SELECT date('now');                          -- '2024-01-15'
SELECT datetime('now');                       -- '2024-01-15 09:30:00'
SELECT strftime('%Y-%m-%d', 'now');          -- '2024-01-15'
SELECT strftime('%s', 'now');                -- Unix 时间戳
SELECT date('now', '+7 days');               -- 7 天后
SELECT date('now', '-1 month');              -- 1 个月前
SELECT datetime('2024-01-15 10:00:00', '+2 hours');
SELECT julianday('2024-01-15') - julianday('2024-01-01');  -- 天数差
```

### 4.4 Boolean 类型

SQLite 没有原生的 BOOLEAN 类型，使用 INTEGER：`0` = false，`1` = true。

```sql
CREATE TABLE tasks (
    id INTEGER PRIMARY KEY,
    title TEXT,
    is_completed INTEGER DEFAULT 0 CHECK (is_completed IN (0, 1))
);

-- 查询时可以用条件判断
SELECT * FROM tasks WHERE is_completed = 1;
```

## 5. 基础 SQL 操作

### 5.1 DDL - 数据定义

```sql
-- 创建表
CREATE TABLE students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    gender TEXT CHECK (gender IN ('男', '女')),
    score REAL DEFAULT 0,
    class_id INTEGER,
    FOREIGN KEY (class_id) REFERENCES classes(id)
        ON DELETE SET NULL
        ON UPDATE CASCADE
);

-- 修改表（SQLite 3.53+ 支持更多 ALTER TABLE 操作）
ALTER TABLE students ADD COLUMN phone TEXT;
ALTER TABLE students ADD COLUMN address TEXT DEFAULT '未知';

-- 3.53+ 新特性：可以添加 NOT NULL 和 CHECK 约束而不重建表
ALTER TABLE students ADD COLUMN email TEXT NOT NULL DEFAULT '';

-- 删除表
DROP TABLE IF EXISTS students;

-- 创建索引
CREATE INDEX idx_students_name ON students(name);
CREATE UNIQUE INDEX idx_students_phone ON students(phone);

-- 删除索引
DROP INDEX IF EXISTS idx_students_name;

-- 创建视图
CREATE VIEW v_student_scores AS
SELECT s.name, s.score, c.name AS class_name
FROM students s
JOIN classes c ON s.class_id = c.id;
```

### 5.2 DML - 数据操作

```sql
-- 插入
INSERT INTO students (name, gender, score, class_id) VALUES ('张三', '男', 85.5, 1);

-- 批量插入
INSERT INTO students (name, gender, score, class_id) VALUES
    ('李四', '女', 92.0, 1),
    ('王五', '男', 78.0, 2),
    ('赵六', '女', 88.5, 2);

-- 插入或替换（存在则替换，不存在则插入）
INSERT OR REPLACE INTO students (id, name, score) VALUES (1, '张三（更新）', 90.0);

-- 插入或忽略（存在则忽略）
INSERT OR IGNORE INTO students (id, name) VALUES (1, '重复数据');

-- Upsert（3.24.0+ 支持）
INSERT INTO students (id, name, score) VALUES (1, '张三', 95.0)
ON CONFLICT(id) DO UPDATE SET score = excluded.score;

-- 更新
UPDATE students SET score = score + 5 WHERE class_id = 1;

-- 删除
DELETE FROM students WHERE score < 60;

-- 清空表（比 DELETE 快，会重置自增 ID）
DELETE FROM students;
-- 或
DELETE FROM students; VACUUM;  -- 回收空间
```

### 5.3 DQL - 数据查询

```sql
-- 基础查询
SELECT * FROM students WHERE score >= 80;
SELECT name, score FROM students ORDER BY score DESC LIMIT 10;

-- 分页
SELECT * FROM students LIMIT 10 OFFSET 20;  -- 第 3 页，每页 10 条
SELECT * FROM students LIMIT 20, 10;        -- 同上（SQLite 特有语法）

-- 聚合
SELECT class_id, COUNT(*) AS cnt, AVG(score) AS avg_score,
       MAX(score) AS max_score, MIN(score) AS min_score
FROM students
GROUP BY class_id
HAVING COUNT(*) >= 5;

-- 子查询
SELECT * FROM students
WHERE score > (SELECT AVG(score) FROM students);

-- EXISTS
SELECT * FROM classes c
WHERE EXISTS (SELECT 1 FROM students s WHERE s.class_id = c.id);

-- 联接
SELECT s.name, c.name AS class_name
FROM students s
INNER JOIN classes c ON s.class_id = c.id;

-- 左联接（保留所有学生，即使没班级）
SELECT s.name, COALESCE(c.name, '未分配') AS class_name
FROM students s
LEFT JOIN classes c ON s.class_id = c.id;

-- 模糊查询
SELECT * FROM students WHERE name LIKE '%张%';
SELECT * FROM students WHERE name LIKE '张_';  -- 张 + 一个任意字符

-- CASE 表达式
SELECT name, score,
    CASE
        WHEN score >= 90 THEN '优秀'
        WHEN score >= 80 THEN '良好'
        WHEN score >= 60 THEN '及格'
        ELSE '不及格'
    END AS grade
FROM students;

-- WITH 子句（CTE，公用表表达式）
WITH top_students AS (
    SELECT * FROM students WHERE score >= 90
)
SELECT * FROM top_students ORDER BY score DESC;

-- 递归 CTE（查询树形结构）
WITH RECURSIVE subordinates AS (
    -- 锚定成员：找到起始员工
    SELECT id, name, manager_id, 0 AS level
    FROM employees WHERE id = 1

    UNION ALL

    -- 递归成员：找到下属
    SELECT e.id, e.name, e.manager_id, s.level + 1
    FROM employees e
    JOIN subordinates s ON e.manager_id = s.id
)
SELECT * FROM subordinates;
```

### 5.4 事务控制

```sql
-- 默认每个语句就是一个事务（自动提交模式）

-- 显式事务
BEGIN TRANSACTION;
    UPDATE accounts SET balance = balance - 100 WHERE id = 1;
    UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;

-- 出错回滚
BEGIN TRANSACTION;
    INSERT INTO accounts (name, balance) VALUES ('张三', 1000);
    -- 如果这里出错了...
    INSERT INTO accounts (name, balance) VALUES ('李四', 2000);  -- 比如违反唯一约束
ROLLBACK;  -- 两条插入都不会生效

-- 保存点（Savepoint）
BEGIN TRANSACTION;
    INSERT INTO logs (msg) VALUES ('开始处理');
    SAVEPOINT sp1;
        UPDATE accounts SET balance = balance - 100 WHERE id = 1;
        -- 如果这部分出错了，可以只回滚到这里
    RELEASE SAVEPOINT sp1;  -- 或 ROLLBACK TO SAVEPOINT sp1;
COMMIT;
```

## 6. 与 MySQL 的关键差异

### 6.1 语法差异

```sql
-- MySQL: AUTO_INCREMENT
-- SQLite: AUTOINCREMENT（注意大小写，且仅对 INTEGER PRIMARY KEY 有效）

-- MySQL: LIMIT offset, count
-- SQLite: LIMIT count OFFSET offset（也支持 LIMIT offset, count）

-- MySQL: CONCAT(str1, str2)
-- SQLite: str1 || str2  或  printf('%s%s', str1, str2)

-- MySQL: IFNULL(expr, alt)
-- SQLite: IFNULL(expr, alt) 或 COALESCE(expr, alt)  -- 两者都支持

-- MySQL: NOW()
-- SQLite: datetime('now') 或 CURRENT_TIMESTAMP

-- MySQL: RAND()
-- SQLite: random() 返回整数，random() % 100 或 abs(random()) % 100

-- MySQL: GROUP_CONCAT()
-- SQLite: group_concat() 或 string_agg()（3.44+）
```

### 6.2 数据类型差异

```sql
-- MySQL: VARCHAR(255) 严格限制长度
-- SQLite: VARCHAR(255) 只是提示，不限制长度

-- MySQL: DECIMAL(10,2) 精确小数
-- SQLite: 没有 DECIMAL，用 REAL 或 INTEGER（分单位存储）
-- 金额推荐用 INTEGER 存储"分"，避免浮点误差
CREATE TABLE orders (
    id INTEGER PRIMARY KEY,
    amount_cents INTEGER NOT NULL  -- 存储分，如 19999 表示 199.99 元
);

-- MySQL: ENUM('a', 'b', 'c')
-- SQLite: 没有 ENUM，用 CHECK 约束或 INTEGER 代码
```

### 6.3 函数差异速查

| 功能 | MySQL | SQLite |
|------|-------|--------|
| 当前时间 | `NOW()` | `datetime('now')` |
| 字符串拼接 | `CONCAT(a,b)` | `a \|\| b` |
| 字符串长度 | `LENGTH()` / `CHAR_LENGTH()` | `length()` |
| 子串 | `SUBSTRING()` | `substr()` |
| 随机数 | `RAND()` | `random()` |
| 四舍五入 | `ROUND(x, n)` | `round(x, n)` |
| 取整 | `FLOOR()` / `CEILING()` | `floor()` / `ceil()`（需加载 math 扩展）|
| MD5 | `MD5()` | 无内置，可用 hex(md5(x))（需扩展）|
| UUID | `UUID()` | `uuid()`（3.46+ 内置）或 `lower(hex(randomblob(16)))` |

## 7. SQLite 的应用场景

SQLite 的独特定位使其在以下场景中大放异彩：

### 7.1 桌面应用程序

几乎所有主流桌面软件都在用 SQLite：

```javascript
// Electron 应用中的典型用法
const Database = require('better-sqlite3');
const path = require('path');

// 将数据库放在用户数据目录
const dbPath = path.join(app.getPath('userData'), 'app-data.db');
const db = new Database(dbPath);

// 存储应用配置、用户偏好、本地缓存数据
db.exec(`
    CREATE TABLE IF NOT EXISTS user_settings (
        key TEXT PRIMARY KEY,
        value TEXT,
        updated_at TEXT DEFAULT (datetime('now'))
    )
`);

// 增删改查...
```

**实际案例**：
- **Chrome/Firefox/Safari**：存储浏览历史、Cookie、缓存
- **微信 PC 版**：存储聊天记录（`MicroMsg.db`）
- **Skype/Slack**：本地消息存储
- **Photoshop**：存储图层信息、元数据

### 7.2 移动应用

SQLite 是 iOS 和 Android 的原生数据库：

```javascript
// React Native 使用 SQLite
import * as SQLite from 'expo-sqlite';

const db = SQLite.openDatabaseSync('myapp.db');

// 创建表并操作
db.withTransactionSync(() => {
    db.runSync(`
        CREATE TABLE IF NOT EXISTS todos (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            title TEXT NOT NULL,
            done INTEGER DEFAULT 0
        )
    `);
});
```

**实际案例**：
- **所有 iOS App**：CoreData 底层就是 SQLite
- **所有 Android App**：Room 框架底层就是 SQLite
- **微信/支付宝/抖音**：本地消息、缓存、用户数据

### 7.3 物联网（IoT）和嵌入式设备

```c
// C 语言嵌入式用法（如树莓派、ESP32）
#include <sqlite3.h>

sqlite3 *db;
int rc = sqlite3_open("/data/sensor.db", &db);

// 存储传感器读数
const char *sql = "INSERT INTO readings (device_id, temperature, humidity) VALUES (?, ?, ?)";
sqlite3_stmt *stmt;
sqlite3_prepare_v2(db, sql, -1, &stmt, NULL);

sqlite3_bind_text(stmt, 1, "dev-001", -1, SQLITE_STATIC);
sqlite3_bind_double(stmt, 2, 25.6);
sqlite3_bind_double(stmt, 3, 60.5);

sqlite3_step(stmt);
sqlite3_finalize(stmt);
sqlite3_close(db);
```

**实际案例**：
- **智能汽车**：特斯拉存储车辆配置、日志
- **智能家居**：设备配置和状态存储
- **工业设备**：数据采集和本地缓存

### 7.4 Web 应用的本地缓存/会话存储

```javascript
// Node.js 小型 Web 服务使用 SQLite 替代 Redis（单机场景）
const express = require('express');
const Database = require('better-sqlite3');

const app = express();
const db = new Database('app.db');

// 存储会话（比文件 session 更高效）
db.exec(`
    CREATE TABLE IF NOT EXISTS sessions (
        id TEXT PRIMARY KEY,
        data TEXT,
        expires_at INTEGER
    )
`);

// 清理过期会话
db.prepare("DELETE FROM sessions WHERE expires_at < ?")
  .run(Math.floor(Date.now() / 1000));

// Express 中间件：从 SQLite 获取会话
app.use((req, res, next) => {
    const sessionId = req.cookies.sessionId;
    if (sessionId) {
        const row = db.prepare(
            "SELECT data FROM sessions WHERE id = ? AND expires_at > ?"
        ).get(sessionId, Math.floor(Date.now() / 1000));
        req.session = row ? JSON.parse(row.data) : {};
    }
    next();
});
```

### 7.5 数据分析和科学计算

```python
# Python + SQLite 数据分析
import sqlite3
import pandas as pd

conn = sqlite3.connect('sales.db')

# 直接用 Pandas 读写
df = pd.read_sql_query("""
    SELECT product_name, SUM(quantity) as total_qty, SUM(amount) as total_amount
    FROM sales
    WHERE sale_date >= date('now', '-30 days')
    GROUP BY product_name
    ORDER BY total_amount DESC
    LIMIT 20
""", conn)

# 结果导出
df.to_csv('monthly_report.csv', index=False)
conn.close()
```

**实际案例**：
- **Jupyter Notebook**：作为轻量级数据存储
- **数据科学流水线**：中间结果缓存
- **机器学习**：特征存储、实验记录

### 7.6 测试环境

```javascript
// Node.js 测试中使用内存数据库
const Database = require('better-sqlite3');

// :memory: 数据库，每次测试都是干净的
let db;

beforeEach(() => {
    db = new Database(':memory:');
    db.exec(`
        CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT);
        INSERT INTO users (name) VALUES ('Alice'), ('Bob');
    `);
});

afterEach(() => {
    db.close();
});

test('find user by name', () => {
    const user = db.prepare('SELECT * FROM users WHERE name = ?').get('Alice');
    expect(user.id).toBe(1);
});
```

## 8. SQLite 的局限性

了解 SQLite 的边界，才能在正确的地方使用它：

| 局限性 | 说明 | 替代方案 |
|-------|------|---------|
| **并发写入** | 数据库级锁定，同时只有一个写操作 | 使用 WAL 模式改善，或换 PostgreSQL/MySQL |
| **网络访问** | 没有内置的网络协议 | 用 litestream 同步，或换 C/S 数据库 |
| **用户管理** | 没有用户/权限系统 | 用操作系统文件权限控制 |
| **存储上限** | 理论 281TB，但实际受文件系统限制 | 分区、归档旧数据 |
| **数据类型严格性** | 动态类型可能导致意外 | 用 CHECK 约束、应用层校验 |
| **ALTER TABLE 限制** | 3.53 之前修改列比较麻烦 | 升级到 3.53+，或使用重建表方案 |
| ** RIGHT/FULL JOIN** | 不支持 | 用 LEFT JOIN + 子查询变通 |
| **存储过程** | 不支持 | 用应用层代码实现 |

---

## 小结

SQLite 的核心哲学是**简单、可靠、嵌入式**。它不是 MySQL 的替代品，而是填补了"不需要数据库服务器"这个生态位。

> **什么时候用 SQLite**：单机应用、本地存储、原型开发、测试、缓存、物联网、数据量不大的网站
>
> **什么时候不用 SQLite**：高并发写入、需要网络访问、分布式部署、TB 级数据、复杂权限管理
