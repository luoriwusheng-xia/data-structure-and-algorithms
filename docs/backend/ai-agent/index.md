## 主流 AI Agent 框架对比

在深入学习之前，了解当前主流框架的定位和差异，有助于做出正确的技术选型。

### 框架概览

| 框架 | 定位 | 适用场景 | 核心优势 | 主要局限 |
|------|------|----------|----------|----------|
| **LangChain / LangGraph** | 通用编排基础设施 | 复杂生产级 Agent | 生态最全、集成最多、可观测性强 | 学习曲线陡峭、抽象层级多 |
| **LlamaIndex** | 数据/RAG 层 | 企业知识库、文档智能 | RAG 能力业界最强、检索策略丰富 | Agent 编排能力较弱 |
| **CrewAI** | 角色化多 Agent | 团队协作模拟、内容生成 | API 极简、代码可读性强 | Token 消耗高、生产观测弱 |
| **AutoGen / AG2** | 对话式多 Agent | 代码生成、研究型任务 | 对话灵活、异步吞吐强 | 状态追踪困难、调试复杂 |
| **Semantic Kernel** | 企业级 SDK | .NET/Azure 企业应用 | 多语言支持、合规安全 | 生态较小、更新保守 |
| **Dify** | LLMOps 平台 | 企业 AI 应用搭建 | 功能最全、可私有部署 | 资源占用高、上手慢 |
| **FastGPT** | 知识库平台 | 中文 RAG 问答 | QA 拆分、中文优化、轻量 | 功能范围窄 |
| **Coze / 扣子** | 零代码平台 | 快速原型、个人 Bot | 上手最快、渠道最全 | 不可私有部署 |
| **OpenAI Agents SDK** | 轻量 SDK | OpenAI 生态快速原型 | 极简、hosted 工具丰富 | 强绑定 OpenAI 模型 |
| **Pydantic AI** | 类型安全框架 | 生产 API 后端 | 类型安全、模型无关 | 无内置状态管理 |
| **n8n** | 自动化+Agent | 业务系统集成 | 400+ 集成、可自托管 | AI 能力深度有限 |

### 为什么选 LangChain？

本系列选择 **LangChain + LangGraph** 作为核心框架，原因如下：

1. **生态最完善**：130k+ Stars，集成模型和工具最多，遇到问题容易找到解决方案
2. **从原型到生产**：LCEL 快速验证想法，LangGraph 构建生产级工作流
3. **可观测性**：LangSmith 提供完整的调用链追踪和调试能力
4. **不绑定厂商**：支持 DeepSeek、OpenAI、Claude、本地模型等，便于切换和成本控制
5. **Node.js 支持**：与现有后端技术栈无缝衔接
