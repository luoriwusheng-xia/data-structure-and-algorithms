## 1. RAG（检索增强生成）

RAG 是解决大模型"幻觉"和"知识过时"问题的核心方案。

### 1.1 核心流程

```
用户提问 → 向量化 → 向量检索 → 拼接上下文 → LLM 生成答案
```

### 1.2 代码实现

```typescript
// src/03-rag.ts
import { ChatOpenAI, OpenAIEmbeddings } from '@langchain/openai'
import { MemoryVectorStore } from 'langchain/vectorstores/memory'
import { RecursiveCharacterTextSplitter } from 'langchain/text_splitter'
import { createStuffDocumentsChain } from 'langchain/chains/combine_documents'
import { createRetrievalChain } from 'langchain/chains/retrieval'
import { ChatPromptTemplate } from '@langchain/core/prompts'
import 'dotenv/config'

const model = new ChatOpenAI({
  modelName: 'deepseek-chat',
  temperature: 0,
  openAIApiKey: process.env.DEEPSEEK_API_KEY,
  configuration: { baseURL: process.env.DEEPSEEK_BASE_URL },
})

// 文档向量化（生产环境应使用持久化向量数据库）
async function buildVectorStore(docs: string[]) {
  const splitter = new RecursiveCharacterTextSplitter({
    chunkSize: 500,
    chunkOverlap: 50,
  })

  const documents = await splitter.createDocuments(docs)

  const embeddings = new OpenAIEmbeddings({
    modelName: 'text-embedding-3-small',
    openAIApiKey: process.env.DEEPSEEK_API_KEY,
    configuration: { baseURL: process.env.DEEPSEEK_BASE_URL },
  })

  return MemoryVectorStore.fromDocuments(documents, embeddings)
}

// RAG 问答链
async function createRagChain(vectorStore: MemoryVectorStore) {
  const prompt = ChatPromptTemplate.fromTemplate(`
基于以下上下文回答问题。如果上下文中没有相关信息，请明确说明。

上下文：
{context}

问题：{input}

请用中文回答：
`)

  const combineDocsChain = await createStuffDocumentsChain({
    llm: model,
    prompt,
  })

  return createRetrievalChain({
    retriever: vectorStore.asRetriever({ k: 3 }),
    combineDocsChain,
  })
}

// 使用
async function main() {
  const docs = [
    '公司考勤制度：上班时间为 9:00-18:00，弹性 1 小时。',
    '请假流程：提前在 OA 系统提交申请，直属领导审批。',
    '报销规范：差旅费需提供发票和行程单，限额 500 元/天。',
  ]

  const vectorStore = await buildVectorStore(docs)
  const chain = await createRagChain(vectorStore)

  const result = await chain.invoke({ input: '请假需要什么流程？' })
  console.log(result.answer)
}

main()
```

### 1.3 向量数据库选型

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| Chroma | 轻量、本地优先 | 开发测试、小规模应用 |
| Pinecone | 全托管、Serverless | 生产环境、快速启动 |
| Milvus/Zilliz | 高性能、分布式 | 海量数据、企业级 |
| PGVector | PostgreSQL 扩展 | 已有 PG 基础设施 |
| Qdrant | Rust 实现、高性能 | 自托管、低延迟要求 |

## 2. Memory（记忆）

### 2.1 基础 Memory

```typescript
import { BufferMemory } from 'langchain/memory'
import { ConversationChain } from 'langchain/chains'

const memory = new BufferMemory({

const chain = new ConversationChain({ llm: model, memory })
```

### 2.3 自定义 Memory（结合数据库）

生产环境应将对话历史持久化到 Redis/数据库：

```typescript
import { BaseChatMemory, InputValues, OutputValues } from 'langchain/memory'
import { BaseMessage, HumanMessage, AIMessage } from '@langchain/core/messages'

class RedisChatMemory extends BaseChatMemory {
  private redis: any
  private sessionId: string

  constructor(redis: any, sessionId: string) {
    super({ returnMessages: true, inputKey: 'input', outputKey: 'response' })
    this.redis = redis
    this.sessionId = sessionId
  }

  async loadMemoryVariables(): Promise<Record<string, any>> {
    const history = await this.redis.lrange(`chat:${this.sessionId}`, 0, -1)
    const messages = history.map((h: string) => {
      const parsed = JSON.parse(h)
      return parsed.role === 'user'
        ? new HumanMessage(parsed.content)
        : new AIMessage(parsed.content)
    })
    return { history: messages }
  }

  async saveContext(
    inputValues: InputValues,
    outputValues: OutputValues
  ): Promise<void> {
    await this.redis.rpush(
      `chat:${this.sessionId}`,
      JSON.stringify({ role: 'user', content: inputValues.input })
    )
    await this.redis.rpush(
      `chat:${this.sessionId}`,
      JSON.stringify({ role: 'assistant', content: outputValues.response })
    )
  }
}
```

## 3. Agent（智能体）

Agent 让 LLM 自主决策，调用工具完成任务。

### 3.1 工具定义

```typescript
import { tool } from '@langchain/core/tools'
import { z } from 'zod'

const searchTool = tool(
  async ({ query }) => {
    // 实际调用搜索引擎 API
    return `搜索结果：关于 "${query}" 的最新资讯...`
  },
  {
    name: 'web_search',
    description: '当需要查询实时信息时使用',
    schema: z.object({
      query: z.string().describe('搜索关键词'),
    }),
  }
)

const calculatorTool = tool(
  async ({ expression }) => {
    return String(eval(expression)) // 生产环境请使用安全的计算库
  },
  {
    name: 'calculator',
    description: '进行数学计算',
    schema: z.object({
      expression: z.string().describe('数学表达式，如 2+2'),
    }),
  }
)
```

### 3.2 ReAct Agent

```typescript
import { createReactAgent } from '@langchain/langgraph/prebuilt'

const agent = createReactAgent({
  llm: model,
  tools: [searchTool, calculatorTool],
})

const result = await agent.invoke({
  messages: [
    new HumanMessage('2024 年 Nobel 物理学奖得主是谁？他今年多大年龄？'),
  ],
})
```

## 4. LangGraph（工作流编排）

LangGraph 是 LangChain 官方推出的状态机工作流框架，适合构建复杂的多步骤 Agent。

### 4.1 基础状态图

```typescript
import { StateGraph, END } from '@langchain/langgraph'
import { BaseMessage, HumanMessage, AIMessage } from '@langchain/core/messages'

// 定义状态
interface AgentState {
  messages: BaseMessage[]
  nextStep: string
}

// 定义节点
const workflow = new StateGraph<AgentState>({
  channels: {
    messages: {
      value: (x: BaseMessage[], y: BaseMessage[]) => x.concat(y),
      default: () => [],
    },
    nextStep: {
      value: (x: string, y: string) => y ?? x,
      default: () => 'start',
    },
  },
})

// 添加节点
workflow.addNode('classify', async (state) => {
  const prompt = `对用户意图进行分类：技术支持 / 销售咨询 / 投诉 / 其他

用户消息：${state.messages.at(-1)?.content}

只输出分类标签：`

  const result = await model.invoke([new HumanMessage(prompt)])
  return { nextStep: result.content as string }
})

workflow.addNode('support', async (state) => {
  const result = await model.invoke([
    new SystemMessage('你是技术支持专家。'),
    ...state.messages,
  ])
  return { messages: [result] }
})

workflow.addNode('sales', async (state) => {
  const result = await model.invoke([
    new SystemMessage('你是销售顾问。'),
    ...state.messages,
  ])
  return { messages: [result] }
})

// 添加条件边
workflow.addConditionalEdges('classify', (state) => {
  const category = state.nextStep
  if (category.includes('技术支持')) return 'support'
  if (category.includes('销售')) return 'sales'
  return END
})

workflow.setEntryPoint('classify')
workflow.addEdge('support', END)
workflow.addEdge('sales', END)

const app = workflow.compile()

// 运行
const result = await app.invoke({
  messages: [new HumanMessage('我的订单什么时候发货？')],
})
```

## 小结

| 能力 | 解决的问题 | 企业应用 |
|------|-----------|---------|
| RAG | 知识库问答 | 智能客服、内部知识检索 |
| Memory | 多轮对话 | 客服机器人、个人助理 |
| Agent | 自主任务执行 | 自动化运维、数据处理 |
| LangGraph | 复杂流程编排 | 审批流、多步骤业务 |
| 多 Agent | 专业分工 | 复杂项目协作、智能团队 |
