# WebSocket 实战指南

## 什么是 WebSocket

WebSocket 是一种在单个 TCP 连接上进行**全双工通信**的协议。它使得客户端和服务器之间可以建立持久连接，双方都可以随时发送数据，而不需要像 HTTP 那样每次都建立新的连接。

### 与 HTTP 的对比

| 特性 | HTTP | WebSocket |
|------|------|-----------|
| 连接方式 | 短连接，每次请求需建立新连接 | 长连接，一次握手后保持连接 |
| 通信模式 | 单向：客户端请求，服务器响应 | 全双工：双方可同时发送 |
| 头部开销 | 大，每次请求都带完整头部 | 小，握手后数据帧头部仅 2-14 字节 |
| 实时性 | 依赖轮询，延迟高 | 真正实时，服务器主动推送 |
| 适用场景 | REST API、静态资源 | 实时通信、推送、聊天、游戏 |

### 握手过程

```
客户端请求：
GET /chat HTTP/1.1
Host: server.example.com
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
Sec-WebSocket-Version: 13

服务器响应：
HTTP/1.1 101 Switching Protocols
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Accept: s3pPLMBiTxaQ9kYGzzhZRbK+xOo=
```

101 状态码表示协议切换成功，之后的数据传输不再使用 HTTP 协议。

## 常见应用场景

1. **即时通讯**：聊天室、客服系统、IM 工具
2. **实时数据推送**：股票行情、比特币价格、体育比分
3. **协同编辑**：在线文档、白板、代码协作
4. **实时游戏**：多人对战、棋牌游戏
5. **IoT 设备监控**：传感器数据实时上报
6. **直播弹幕**：实时评论、礼物消息
7. **系统通知**：订单状态变更、告警推送
