# Web3 背景介绍：从互联网产权革命说起

## 一、Web1 → Web2 → Web3

互联网的发展历程可以概括为三个阶段，理解它们是进入 Web3 的前提。

| 阶段 | 特点 | 代表 | 核心问题 |
|------|------|------|----------|
| Web1 | 只读（Read-only） | 门户网站、静态网页 | 内容由平台单向发布，用户只能浏览 |
| Web2 | 读+写（Read+Write） | 微博、微信、抖音 | 用户生产内容，但数据归平台所有 |
| Web3 | 读+写+拥有（Read+Write+Own） | 以太坊、Uniswap、OpenSea | 用户真正拥有自己的数据和数字资产 |

### 1.1 Web2 的问题

在 Web2 时代，你注册一个账号，本质上是在平台的数据库里"借"了一个身份：

- 你的微信聊天记录保存在腾讯服务器
- 你的微博粉丝列表属于新浪
- 你在游戏中的装备、皮肤归游戏公司所有
- 平台可以随时封号、删内容、改规则

如果你是一名开发者，你肯定经历过：调用某个平台的 API，突然被告知接口下线、涨价、或者需要申请更高级别的权限。这是**中心化平台**的固有缺陷。

### 1.2 Web3 的核心主张

Web3 试图通过技术手段解决这些问题：

**「Not your keys, not your coins.」**

Web3 的核心是让用户持有自己的**私钥（Private Key）**，从而真正控制自己的数字身份和资产。私钥对应的公钥就是你的"账号"，不需要任何人批准就能使用。

**去中心化（Decentralization）**

Web3 应用通常运行在区块链网络上，数据由全球成千上万的节点共同维护，而不是某一家公司的服务器。这意味着：

- 没有单点故障
- 没有人能单方面关停整个网络
- 代码一旦部署，规则公开透明

**可组合性（Composability）**

区块链上的智能合约就像乐高积木。一个项目构建的金融协议，可以被另一个项目直接调用。这种"无需许可的创新"是 Web3 最迷人的特性之一。

**代币激励（Tokenomics）**

Web3 项目通过发行代币（Token）来协调各方利益。用户、开发者、投资人可以通过持有代币参与项目治理，分享网络增长的红利。

## 二、区块链基础概念

### 2.1 区块链是什么

可以把区块链理解为一个**全球共享的、不可篡改的账本**。每个"区块"里记录了若干交易，新区块通过密码学与上一个区块链接，形成一条链。

```
区块 N-1        区块 N          区块 N+1
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 交易列表 │ -> │ 交易列表 │ -> │ 交易列表 │
│ 前一哈希 │    │ 前一哈希 │    │ 前一哈希 │
│ 当前哈希 │    │ 当前哈希 │    │ 当前哈希 │
└─────────┘    └─────────┘    └─────────┘
```

### 2.2 公链与联盟链

| 类型 | 特点 | 代表 | 适用场景 |
|------|------|------|----------|
| 公链 | 完全开放，任何人可参与 | 以太坊、Solana、BNB Chain | DeFi、NFT、DAO |
| 联盟链 | 由多个机构共同维护 | Hyperledger Fabric、蚂蚁链 | 供应链金融、政务 |
| 私链 | 单一机构控制 | 企业内部链 | 内部数据存证 |

对于开发者入门，**以太坊（Ethereum）** 是首选，因为它的生态最成熟、文档最丰富。

### 2.3 账户模型

以太坊有两种账户：

**外部账户（EOA，Externally Owned Account）**

- 由私钥控制
- 由人类用户持有
- 地址以 `0x` 开头，例如：`0xdAC17F958D2ee523a2206206994597C13D831ec7`
- 可以发起交易

**合约账户（Contract Account）**

- 由代码控制
- 没有私钥
- 只有在收到交易时才会执行代码
- 存储着合约的状态和数据

```javascript
// 一个典型的以太坊地址
const myAddress = '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb';
```

### 2.4 交易与 Gas

在区块链上执行任何操作都需要**交易（Transaction）**，而交易需要支付**Gas 费**。

Gas 费 = Gas 用量 × Gas 单价

- **Gas 用量**：执行操作消耗的计算资源
- **Gas 单价（Gas Price）**：你愿意为每单位 Gas 支付的价格

当网络拥堵时，Gas 价格会飙升。这也是为什么 Layer 2（二层网络）如此重要的原因。

### 2.5 Layer 1 与 Layer 2

| 层级 | 作用 | 代表 |
|------|------|------|
| Layer 1 | 主链，负责最终结算和安全 | 以太坊主网、Solana |
| Layer 2 | 扩展方案，提升吞吐量、降低 Gas | Arbitrum、Optimism、Base、Polygon |

对于普通用户和开发者来说，Layer 2 的体验更好、成本更低，但安全性最终由 Layer 1 保证。

## 三、Web3 主要赛道

### 3.1 DeFi（去中心化金融）

DeFi 用智能合约重建传统金融服务：借贷、交易、保险、衍生品等。

- **DEX（去中心化交易所）**：Uniswap、Curve
- **借贷协议**：Aave、Compound
- **稳定币**：USDT、USDC、DAI

### 3.2 NFT（非同质化代币）

NFT 让数字内容具备唯一性和所有权，适用于艺术品、游戏道具、会员卡、学历证书等。

- **交易平台**：OpenSea、Blur
- **标准**：ERC-721（唯一）、ERC-1155（半同质化）

### 3.3 DAO（去中心化自治组织）

DAO 是通过智能合约运行的组织，成员用代币投票决定资金用途和规则变更。

- **工具**：Snapshot（投票）、Gnosis Safe（多签钱包）
- **代表**：MakerDAO、Uniswap DAO

### 3.4 GameFi 与 SocialFi

- **GameFi**：游戏+金融，玩家可以通过游戏赚取代币或 NFT
- **SocialFi**：社交网络+金融，用户数据归自己所有，创作可获得代币奖励

## 四、为什么开发者要关注 Web3

如果你已经是一名 Web2 开发者，Web3 给你带来的是全新的机会：

1. **更高的收入潜力**：智能合约开发者的薪资普遍高于普通前端/后端
2. **全新的产品形态**：可组合性让你能做出 Web2 无法实现的金融和协作工具
3. **全球化的工作方式**：Web3 项目大多是远程、分布式、按贡献付费
4. **技术前沿**：零知识证明、MEV、账户抽象等方向正在快速发展

## 五、学习路径总览

```
第一阶段：理解概念
    ↓ 区块链原理、账户、交易、Gas
第二阶段：动手实践
    ↓ 安装钱包、领取测试币、完成一次转账
第三阶段：学习开发
    ↓ Solidity 语法、Hardhat 框架、Ethers.js
第四阶段：参与实战
    ↓ 部署自己的 Token/NFT、参与开源协议
```

Web3 不是玄学，它是一系列成熟技术的组合：密码学、分布式系统、博弈论、经济学。作为有编程基础的你，最大的门槛不是技术本身，而是理解这个新世界的设计哲学。
