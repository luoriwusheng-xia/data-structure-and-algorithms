# Web3 技术选型：新手入门的工具箱

进入 Web3 开发领域，第一件事不是写代码，而是选对工具。本章从**公链、开发语言、框架、钱包、数据源、部署环境**六个维度，帮你建立一套适合初学者的技术栈。

## 一、公链选择

### 1.1 以太坊（Ethereum）—— 首选学习链

如果你是 Web3 新手，**以太坊仍然是最好的起点**。理由很简单：

- 生态最成熟，教程、文档、工具链最完善
- Solidity 是最主流的合约语言
- EVM（以太坊虚拟机）生态兼容链最多
- 最多的工作岗位和开源项目

**适合场景**：DeFi、NFT、DAO、学习基础概念

### 1.2 Layer 2 链

Layer 2 在以太坊的基础上提供了更低的 Gas 和更快的确认速度，推荐作为**部署和测试的首选**。

| 链 | 特点 | 适合场景 |
|----|------|----------|
| Arbitrum | 生态成熟，兼容 EVM | DeFi、复杂合约 |
| Optimism | 采用 Optimistic Rollup | 通用 DApp |
| Base | Coinbase 推出，用户友好 | 消费级应用 |
| Polygon PoS | 速度快，成本低 | NFT、游戏 |

### 1.3 其他公链

| 链 | 特点 | 适合人群 |
|----|------|----------|
| Solana | 高性能，Rust 语言 | 追求性能、熟悉 Rust |
| BNB Chain | EVM 兼容，用户多 | 亚洲项目、低成本 |
| Avalanche | 子网架构 | 需要定制化链的项目 |

**入门建议**：先精通以太坊 + 一条 Layer 2（如 Arbitrum Sepolia 测试网），再根据需要扩展。

## 二、智能合约语言

### 2.1 Solidity（必选）

Solidity 是以太坊生态的智能合约语言，语法接近 JavaScript，有编程基础的人上手很快。

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

contract HelloWorld {
    string public message;

    constructor() {
        message = "Hello, Web3!";
    }

    function setMessage(string memory _message) public {
        message = _message;
    }

    function getMessage() public view returns (string memory) {
        return message;
    }
}
```

**核心概念**：

- `pragma`：指定编译器版本
- `contract`：定义合约
- `public`：自动创建 getter
- `view`：不修改状态，免费调用
- `memory`：临时存储位置

### 2.2 Vyper

Python 风格的合约语言，语法更严格，安全性更高，但生态较小。

**适合**：安全要求极高的项目，或 Python 开发者过渡。

### 2.3 Rust（Solana/Aptos/Sui）

如果你选择 Solana 或 Move 系公链，Rust 是必修课。Rust 学习曲线陡峭，不建议作为 Web3 第一门语言。

## 三、开发框架

### 3.1 Hardhat（推荐新手）

Hardhat 是目前最受欢迎的以太坊开发环境，学习曲线平缓，插件丰富。

```bash
# 初始化 Hardhat 项目
npx hardhat init

# 常见任务
npx hardhat compile      # 编译合约
npx hardhat test         # 运行测试
npx hardhat node         # 启动本地网络
npx hardhat run scripts/deploy.js --network sepolia
```

**优点**：

- JavaScript/TypeScript 配置，对前端友好
- 内置本地网络，测试方便
- 强大的调试工具（console.log 直接打印）
- 插件生态完善（Ethers.js、Waffle、Coverage 等）

### 3.2 Foundry

Foundry 是用 Rust 编写的高性能测试框架，测试用 Solidity 写，执行速度极快。

**优点**：

- 测试速度快（比 Hardhat 快 10-100 倍）
- 用 Solidity 写测试，贴近真实执行环境
- 内置 Fuzzing（模糊测试）

**缺点**：对纯 JavaScript 开发者来说门槛稍高。

**建议**：先用 Hardhat 入门，熟练后学习 Foundry 提升测试和审计能力。

## 四、前端交互库

### 4.1 Ethers.js v6（推荐）

Ethers.js 是与以太坊交互的 JavaScript 库，提供了简洁的 API。

```javascript
import { ethers } from 'ethers';

// 连接 MetaMask
const provider = new ethers.BrowserProvider(window.ethereum);
const signer = await provider.getSigner();

// 读取合约
const contract = new ethers.Contract(contractAddress, abi, provider);
const message = await contract.getMessage();

// 写入合约
const contractWithSigner = contract.connect(signer);
const tx = await contractWithSigner.setMessage('Hello from Ethers.js!');
await tx.wait(); // 等待交易确认
```

### 4.2 Web3.js

老牌库，API 偏底层，历史包袱较重。新项目一般优先选择 Ethers.js 或 viem。

### 4.3 Viem + Wagmi

- **viem**：类型安全的以太坊交互库，体积更小
- **wagmi**：React Hooks 封装，极大简化钱包连接和合约交互

```javascript
import { useAccount, useContractRead, useContractWrite } from 'wagmi';

function App() {
  const { address } = useAccount();
  const { data } = useContractRead({
    address: contractAddress,
    abi,
    functionName: 'getMessage',
  });

  return <div>{address ? `Connected: ${address}` : 'Not connected'}</div>;
}
```

## 五、钱包选择

### 5.1 MetaMask（开发必备）

浏览器插件钱包，支持所有 EVM 链。开发时几乎必装。

- 支持导入测试网
- 可以切换多个账号
- 方便与 DApp 交互

### 5.2 其他钱包

| 钱包 | 特点 |
|------|------|
| Rainbow | 移动端体验好 |
| Rabby | 安全功能强，适合高级用户 |
| Ledger/Trezor | 硬件钱包，大额资产存储 |

## 六、测试网与水龙头

### 6.1 常用测试网

| 测试网 | 链 | 水龙头 |
|--------|----|--------|
| Sepolia | 以太坊 | sepoliafaucet.com, alchemy.com/faucets |
| Mumbai | Polygon | faucet.polygon.technology |
| Arbitrum Sepolia | Arbitrum | faucet.quicknode.com/arbitrum-sepolia |

### 6.2 什么是水龙头（Faucet）

水龙头是免费发放测试币的网站。开发时你需要测试网的 ETH 来支付 Gas，但不需要花真钱。

## 七、区块浏览器

区块浏览器是 Web3 开发的"调试器"，可以查看交易、合约、事件日志。

| 链 | 主网浏览器 | 测试网浏览器 |
|----|------------|--------------|
| 以太坊 | etherscan.io | sepolia.etherscan.io |
| Arbitrum | arbiscan.io | sepolia.arbiscan.io |
| Polygon | polygonscan.com | mumbai.polygonscan.com |

## 八、数据源与索引

### 8.1 The Graph

区块链原生数据查询困难，The Graph 通过索引链上事件，提供 GraphQL 接口。

**适合**：复杂的数据聚合、历史记录查询。

### 8.2 Alchemy / Infura

提供节点 RPC 服务，你不需要自己运行以太坊节点。

```javascript
// Alchemy RPC 示例
const provider = new ethers.JsonRpcProvider(
  'https://eth-sepolia.g.alchemy.com/v2/YOUR_API_KEY'
);
```

## 九、推荐技术栈组合

### 新手入门组合

| 层级 | 工具 | 理由 |
|------|------|------|
| 链 | Sepolia / Arbitrum Sepolia | 测试网免费，生态成熟 |
| 合约语言 | Solidity | 资料最多，工作机会最多 |
| 开发框架 | Hardhat | 配置简单，JS 开发者友好 |
| 前端交互 | Ethers.js v6 | 文档清晰，足够完成大多数项目 |
| 钱包 | MetaMask | 开发必备 |
| 数据源 | Alchemy | 稳定可靠，免费额度够用 |

### 进阶组合

| 层级 | 工具 | 理由 |
|------|------|------|
| 测试框架 | Foundry | 速度极快，适合复杂项目 |
| 前端框架 | wagmi + viem + RainbowKit | 类型安全，用户体验好 |
| 索引 | The Graph | 高效查询历史数据 |
| 存储 | IPFS / Arweave | 去中心化文件存储 |

## 十、选型决策树

```
开始 Web3 开发
    │
    ├── 你想做什么？
    │       │
    │       ├── DeFi / NFT / DAO → 以太坊 + Solidity
    │       ├── 高性能游戏 / 高频交易 → Solana + Rust
    │       └── 企业联盟链 → Hyperledger Fabric / 蚂蚁链
    │
    ├── 你的技术背景？
    │       │
    │       ├── 前端/Node.js → Hardhat + Ethers.js
    │       ├── Python → Vyper / Brownie
    │       └── Rust → Foundry + Solana
    │
    └── 你的目标？
            │
            ├── 找工作 → Solidity + Hardhat
            ├── 做独立项目 → wagmi + viem + 一条 Layer 2
            └── 研究底层 → 以太坊源码 + 密码学
```
